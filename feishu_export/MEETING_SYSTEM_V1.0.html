满意解研究所会议制度 V1.0

文档版本: 1.0
生效日期: 2026-03-10
制定者: 满意妞（主控AI）
审批人: Egbertie

1. 制度目的

为确保满意解研究所各项工作高效推进，建立规范、透明、可追踪的会议机制，使Egbertie能够及时了解项目进度和重要决策。

2. 会议类型

2.1 每日站会（Daily Standup）

项目 详情
时间 每天09:30（晨报后）
时长 15分钟
参与人 满意妞（主控AI）、CONTENT、PEO、EEO
会议形式 语音/文字快速同步
核心内容 昨日完成、今日计划、阻塞问题

会议流程: 1. 每人汇报昨日完成工作（2分钟） 2. 每人说明今日计划（2分钟） 3. 提出阻塞问题并快速协调（5分钟） 4. 满意妞总结并分配紧急任务（2分钟）

输出物: 站会简报（发送至工作群）

2.2 周复盘会（Weekly Review）

项目 详情
时间 每周日20:00
时长 60分钟
参与人 全员 + Egbertie（可选）
会议形式 视频会议/语音会议
核心内容 周进展汇报、问题复盘、下周计划

会议议程: 1. 开场（5分钟）- 满意妞主持，确认议程 2. 各模块进展汇报（30分钟） - CONTENT：内容产出、数据表现 - PEO：用户增长、社群运营 - EEO：产品开发、技术进展 3. 问题复盘（15分钟）- 本周遇到的主要问题及解决方案 4. 下周计划对齐（10分钟）- 优先级确认、资源协调

输出物: 周会议纪要（提交Egbertie审阅）

2.3 里程碑评审会（Milestone Review）

项目 详情
时间 里程碑达成时召开
时长 90分钟
参与人 全员 + Egbertie（必须参加）
会议形式 正式评审会议
核心内容 成果展示、经验总结、决策确认

会议议程: 1. 里程碑成果展示（30分钟）- 演示可交付成果 2. 数据回顾（15分钟）- 关键指标达成情况 3. 经验总结（20分钟）- 成功经验与改进点 4. 下阶段决策确认（20分钟）- 方向调整、资源投入 5. Egbertie点评与指示（5分钟）

输出物: 里程碑评审报告（Egbertie签字确认）

3. 会议准备规范

3.1 会前准备

时间点 动作 责任人
会前24小时 发送会议邀请和议程 满意妞
会前1小时 自动提醒通知 系统
会前15分钟 测试设备、准备材料 各参会人

3.2 会中纪律

  1. 准时开始: 迟到超过5分钟需说明原因
  2. 专注参与: 禁止处理与会议无关事务
  3. 记录要点: 满意妞负责记录关键信息
  4. 控制时间: 严格按议程时间执行

3.3 会后跟进

时间点 动作 责任人
会后2小时内 提交会议纪要 满意妞
会后24小时内 行动项确认 各负责人
每日站会 行动项进度同步 全员

4. 会议纪要规范

4.1 纪要模板位置

/root/.openclaw/workspace/templates/meeting-minutes.md

4.2 纪要内容要求

  • 基本信息: 时间、地点、参与人、会议类型
  • 议程回顾: 讨论要点和结论
  • 决策事项: 需Egbertie确认的重要决策
  • 行动项: 明确负责人和截止日期
  • 待确认问题: 需后续跟进的事项

4.3 纪要提交流程

满意妞起草 → 发送参会人确认 → 提交Egbertie审阅 → 归档保存

5. 自动化机制

5.1 自动提醒系统

配置文件位置: /root/.openclaw/cron/meeting-reminders.json

提醒规则: - 会前1小时：推送会议提醒 - 会前15分钟：再次提醒 - 会后30分钟：提醒提交纪要 - 行动项截止前1天：提醒负责人

5.2 任务自动同步

  • 会议行动项自动加入 TASK_MASTER.md
  • 决策项标记为"等待Egbertie确认"
  • 完成状态自动同步至项目管理看板

6. 角色职责

角色 会议职责
满意妞 会议主持、纪要撰写、任务跟踪、提醒发送
CONTENT 内容相关议题汇报、执行分配任务
PEO 运营相关议题汇报、执行分配任务
EEO 产品技术议题汇报、执行分配任务
Egbertie 重要决策审批、方向指导、资源支持
