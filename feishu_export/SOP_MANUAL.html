1.6 详细步骤

步骤1: 需求识别

  • 明确文档目的和目标读者
  • 确定文档类型和范围
  • 分配文档负责人

步骤2: 模板选择

  • 从模板库选择合适模板
  • 复制模板到目标位置
  • 填写文档头部信息

步骤3: 内容编写

  • 遵循"金字塔原理"：结论先行
  • 使用清晰、简洁的语言
  • 多用表格、列表、图表
  • 关键信息加粗或高亮

步骤4: 自检审查

检查清单： - [ ] 文档头部信息完整 - [ ] 版本号正确 - [ ] 章节编号连续 - [ ] 无拼写和语法错误 - [ ] 链接可正常访问 - [ ] 表格格式正确

步骤5: 交叉审查

  • 提交给相关角色审查
  • 收集反馈意见
  • 修改完善

步骤6: 审批确认

  • 提交给Egbertie或授权审批人
  • 获得书面或电子批准
  • 记录审批意见

步骤7: 归档发布

  • 保存到指定目录
  • 更新文档索引
  • 通知相关人员
  • 同步到Notion/GitHub

1.7 更新流程

触发条件 更新类型 审批要求
错别字修正 补丁版本+1 无需审批
内容补充 次版本+1 原审批人确认
重大变更 主版本+1 重新审批
架构调整 主版本+1 Egbertie审批

3. 会议召开SOP

3.1 目的

规范各类会议的组织、召开和跟进流程，提高会议效率，确保会议成果落地。

3.2 会议类型

会议类型 频率 时长 参与人 目的
每日站会 每日 15分钟 核心团队 进度同步、阻塞识别
周例会 每周一 60分钟 全团队 周复盘、计划对齐
里程碑评审 里程碑 90分钟 核心+顾问 阶段成果、决策
专题讨论 按需 45分钟 相关人员 问题解决、决策
1对1沟通 按需 30分钟 相关两人 深度交流、反馈

3.5 会中执行

主持人职责

  1. 开场: 简述议程、确认时间
  2. 控场: 控制每个议题时间
  3. 引导: 确保人人参与、聚焦主题

4. 备份恢复SOP

4.1 目的

规范数据备份和灾难恢复操作流程，确保在发生数据丢失或系统故障时，能够快速、准确地恢复业务运营。

4.2 备份策略

备份频率

数据类型 备份频率 保留时长 存储位置
核心配置文件 每日+变更触发 30天 本地+加密云
Markdown文档 每日自动 90天 本地+Git
记忆文件 每日自动 180天 本地+加密云
Skill文件 每周+变更触发 60天 本地+Git

备份时间

  • 每日备份: 每天 02:00 (Asia/Shanghai)
  • 每周备份: 每周日 03:00 (Asia/Shanghai)

4.5 灾难分级

级别 定义 示例 RTO
P0-紧急 完全不可用 系统崩溃、全部数据丢失 1小时
P1-严重 核心功能受损 关键配置丢失 2小时
P2-一般 非核心功能受损 部分文档丢失 4小时
P3-轻微 轻微影响 单个文件丢失 48小时

4.6 恢复流程

P0级灾难恢复 (系统完全不可用)

步骤1: 灾难确认 (0-15分钟) - 确认灾难范围和影响 - 启动应急响应 - 通知Egbertie

步骤2: 恢复准备 (15-30分钟)

# 查看可用备份
python3 /root/.openclaw/workspace/scripts/backup-manager.py list

# 选择最新完整备份
BACKUP_DATE=$(ls -lt /backups/weekly/ | head -2 | tail -1 | awk '{print $9}')
echo "恢复点: $BACKUP_DATE"

步骤3: 执行恢复 (30-90分钟)

# 执行自动恢复
python3 /root/.openclaw/workspace/scripts/backup-manager.py restore \
  --date $BACKUP_DATE \
  --type full

# 验证关键文件
cat /root/.openclaw/workspace/.env
cat /root/.openclaw/openclaw.json

步骤4: 服务恢复 (90-120分钟)

# 重启服务
openclaw gateway restart

# 验证定时任务
openclaw cron list

# 运行验证脚本
python3 /root/.openclaw/workspace/scripts/backup-manager.py verify

步骤5: 恢复报告 - 生成恢复报告 - 记录故障原因 - 提出改进建议

单文件恢复

# 恢复单个文件
python3 /root/.openclaw/workspace/scripts/backup-manager.py restore-file \
  --backup-date 2026-03-10 \
  --source-path docs/PROJECT_CHARTER.md \
  --target-path /root/.openclaw/workspace/docs/PROJECT_CHARTER.md
