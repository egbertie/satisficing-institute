# 日常模式优化方案 V2.0

> **参照**: 完全静默模式优化（2026-03-22）  
> **优化目标**: 在保证功能完整的前提下，减少Token浪费，提高效率  
003e **当前问题**: 日常模式任务过多、频率过高、缺乏优先级分层

---

## 一、日常模式问题诊断

### 1.1 当前问题

```
问题1: 任务过多
├── 每日运行任务: 15+
├── 每小时触发: 3-4个
├── 结果: Token消耗过快，用户疲劳
└── 症状: 89%消耗发生在第5天（正常应为71%）

问题2: 频率不合理
├── 心跳检查: 每10分钟一次
├── 信息情报: 每4小时一次
├── 结果: 很多检查无实质内容，空转消耗Token
└── 症状: 80%的心跳检查报告"无异常"

问题3: 缺乏优先级分层
├── 所有任务同等频率
├── P0和P3任务混排
├── 结果: 重要任务被稀释，次要任务过度执行
└── 症状: 用户抱怨"太多提醒"

问题4: 无动态调整
├── 固定频率，不随负载变化
├── 忙碌时和空闲时同样频率
├── 结果: 用户忙碌时被打扰，空闲时无服务
└── 症状: "适时提醒"变成"过度打扰"
```

### 1.2 优化原则

```
参照完全静默模式的核心思路：
┌─────────────────────────────────────────────────────────┐
│ 1. 分层管理 - P0/P1/P2/P3任务不同频率                    │
│ 2. 动态调整 - 根据用户活跃度调整频率                     │
│ 3. 批量处理 - 非紧急任务批量执行                         │
│ 4. 智能触发 - 基于事件触发，而非固定时间                 │
│ 5. 用户控制 - 让用户选择接收什么、何时接收               │
└─────────────────────────────────────────────────────────┘
```

## 三、动态频率调整机制

### 3.1 基于用户活跃度的调整

```python
# 动态频率调整逻辑
def get_heartbeat_frequency():
    last_message_time = get_last_user_message()
    time_since_last = now() - last_message_time
    
    # 用户活跃时（1小时内有互动）
    if time_since_last < 1 hour:
        return "每30分钟"  # 保持正常频率
    
    # 用户离开（1-4小时无互动）
    if 1 hour <= time_since_last < 4 hours:
        return "每小时"  # 降频
    
    # 用户长时间离开（4-8小时）
    if 4 hours <= time_since_last < 8 hours:
        return "每2小时"  # 继续降频
    
    # 用户可能睡觉（8小时+）
    if time_since_last >= 8 hours:
        return "暂停"  # 暂停非紧急检查
```

### 3.2 基于Token余量的调整

```python
# Token余量自适应
def get_task_frequency(task_priority, token_remaining):
    """
    根据Token余量动态调整任务频率
    """
    if token_remaining > 70%:
        # 充足: 正常频率
        return NORMAL_FREQUENCY
    
    elif 30% <= token_remaining <= 70%:
        # 中等: P2/P3降频50%
        if task_priority in ["P2", "P3"]:
            return HALF_FREQUENCY
        return NORMAL_FREQUENCY
    
    elif 15% <= token_remaining < 30%:
        # 预警: 仅保留P0/P1
        if task_priority == "P3":
            return "暂停"
        elif task_priority == "P2":
            return QUARTER_FREQUENCY
        return HALF_FREQUENCY
    
    else:
        # 紧急: 进入休眠或完全静默
        return "进入休眠模式"
```

### 3.3 基于任务结果的调整

```python
# 智能频率调整：连续N次无异常则降频
def adaptive_frequency(task_name, current_frequency):
    last_n_results = get_last_results(task_name, n=5)
    
    # 连续5次无异常，降频一级
    if all(result == "无异常" for result in last_n_results):
        return decrease_frequency(current_frequency)
    
    # 发现异常，恢复高频
    if any(result != "无异常" for result in last_n_results):
        return NORMAL_FREQUENCY
    
    return current_frequency
```
