# 休眠模式优化方案 V2.0

> **参照**: 完全静默模式优化（2026-03-22）  
> **优化目标**: 在保持极低Token消耗的同时，确保核心功能不丢失  
> **当前问题**: 休眠模式定义不清晰，与完全静默模式边界模糊

---

## 一、模式定义澄清

### 1.1 三种静默模式的边界

```
┌─────────────────────────────────────────────────────────────┐
│                    静默模式光谱                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   【日常模式】        【休眠模式】        【完全静默】       │
│   ───────────      ─────────────      ───────────────       │
│   Token: 正常       Token: 5-10%       Token: <5%           │
│   功能: 100%        功能: 核心20%      功能: 底线5%         │
│   触发: 默认        触发: 低Token      触发: 紧急           │
│   唤醒: 即时        唤醒: 1分钟内      唤醒: 5分钟内        │
│                                                             │
│   【休眠模式定位】                                          │
│   - 介于日常和完全静默之间                                  │
│   - Token余量15-30%时激活                                   │
│   - 保留核心监控，暂停非紧急任务                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 休眠模式 vs 完全静默模式

| 维度 | 休眠模式 | 完全静默模式 |
|------|----------|--------------|
| **Token余量** | 15-30% | <15% |
| **触发条件** | 预警级 | 紧急级 |
| **任务保留** | 核心监控+周期性检查 | 仅生存必需 |
| **唤醒延迟** | 1分钟内 | 5分钟内 |
| **数据安全** | 每日备份 | 每周备份 |
| **状态保存** | 每小时 | 每6小时 |

## 二、休眠模式配置 V2.0

### 2.1 触发条件（自动）

```python
# 休眠模式触发逻辑
def should_enter_hibernation():
    token_remaining = get_token_remaining()
    token_total = get_token_total()
    remaining_pct = token_remaining / token_total
    
    # 触发条件1: Token余量15-30%
    if 0.15 <= remaining_pct <= 0.30:
        return True
    
    # 触发条件2: 用户主动指令
    if user_command == "/hibernate":
        return True
    
    # 触发条件3: 连续错误率>50%
    if error_rate_last_hour > 0.5:
        return True
    
    return False
```

### 2.2 保留任务（核心20%）

| 任务 | 频率 | Token/次 | 日总计 | 必要性 |
|------|------|----------|--------|--------|
| Token熔断监控 | 每6小时 | 500 | 2K | 自我保护 |
| 黄昏图腾归位 | 每日18:00 | 1.5K | 1.5K | 仪式底线 |
| 状态快照保存 | 每2小时 | 200 | 2.4K | 数据安全 |
| 心跳检查 | 每30分钟 | 100 | 4.8K | 系统存活 |
| **日总计** | | | **~10.7K** | |

### 2.3 暂停任务（非核心80%）

| 任务 | 原频率 | 节省Token/日 | 恢复策略 |
|------|--------|--------------|----------|
| 晨间图腾 | 每日09:00 | 1.5K | 退出休眠后恢复 |
| 7浪费追踪 | 每日23:00 | 3K | 退出后补录 |
| 蓝军审计 | 周二/四/六 | 8K | 退出后执行 |
| 信息情报 | 每4小时 | 12K | 退出后批量处理 |
| 外部学习 | 每日22:00 | 5K | 退出后恢复 |
| **日节省** | | **~29.5K** | |

### 2.4 与完全静默模式的对比

```
休眠模式（10.7K/日） vs 完全静默（~7K/日）

差异任务：
├─ 心跳检查（4.8K）→ 完全静默下暂停
├─ 状态快照（2.4K）→ 完全静默下改为每6小时
└─ Token监控频率（节省~1K）

关键区别：
• 休眠模式保持"心跳" - 系统可感知
• 完全静默切断"心跳" - 系统最低限度存活
```

## 三、进入/退出机制

### 3.1 进入流程

```
用户指令: "/hibernate" 或 自动触发
         ↓
[Step 1] 保存当前状态快照
         - 当前任务列表
         - 未完成的对话上下文
         - 重要待办事项
         ↓
[Step 2] 暂停非核心Cron任务
         - 禁用12个非核心任务
         - 保留5个核心任务
         ↓
[Step 3] 生成休眠报告
         - 进入时间
         - 预计持续时间
         - 唤醒方式
         ↓
[Step 4] 进入休眠
         - 仅响应唤醒指令
         - 每30分钟一次心跳（可选）
```

### 3.2 退出流程

```
用户指令: 任意消息 或 "/wake"
         ↓
[Step 1] 立即响应
         - 休眠模式响应时间: <1分钟
         - 完全静默响应时间: <5分钟
         ↓
[Step 2] 加载休眠前状态
         - 恢复任务列表
         - 提示休眠期间错过的事项
         ↓
[Step 3] 询问恢复策略
         "休眠期间有X条信息未处理，是否批量处理？"
         ↓
[Step 4] 恢复非核心任务
         - 重新启用暂停的Cron任务
         - 恢复到日常模式
```

**休眠中**:
