## 任务二：顶会论文"对齐-互补"框架融入方法论

### 核心发现

**论文标题**：Align When They Want, Complement When They Need!  
**来源**：AAAI 2026，普渡大学研究团队

### "对齐-互补"框架本质

```
传统AI辅助决策的问题：
┌─────────────────────────────────────────────────────────┐
│  单模型AI无法同时兼顾「建立信任」和「提升性能」            │
│                                                           │
│  • 对齐性AI：在人类高自信区与人类保持一致 → 建立信任       │
│    但代价：浪费AI纠错能力，团队性能上不去                 │
│                                                           │
│  • 互补性AI：在人类低自信区精准纠错 → 提升性能             │
│    但代价：与人类高自信判断冲突时，摧毁信任               │
│                                                           │
│  → 单模型的「互补性-对齐性权衡」困境                     │
└─────────────────────────────────────────────────────────┘

破局方案：自适应AI集成
┌─────────────────────────────────────────────────────────┐
│  训练两个专家AI + 智能路由机制                             │
│                                                           │
│  ┌─────────────┐    ┌─────────────┐                      │
│  │  对齐专家    │    │  互补专家    │                      │
│  │  (Alignment)│    │ (Complement)│                      │

│  │ 专注人类高   │    │ 专注人类低   │                      │

│  │ 信任关系     │    │ 纠错         │                      │

│           ↓                    ↓                          │
│           └────────┬───────────┘                          │
│                    ↓                                      │
│            ┌───────────────┐                              │
│            │  理性路由捷径   │                              │
│            │  (RRS)         │                              │
│            │                │                              │
│            │ 哪个专家更自信  │                              │
│            │ 就用哪个        │                              │

│            人机团队准确率提升9%                            │

```

### 关键洞察：人的直觉力是核心

**论文验证的观点**：
1. **人类自信度是关键信号** - 高自信时信人，低自信时信AI
2. **信任是门槛** - 没有信任，人类不会采纳AI建议
3. **互补需要时机** - 只有在人类愿意接受时，AI的纠错才有价值

### 融入满意解研究所方法论

**核心原则：AI辅助，人为主导**

```
合伙人决策中的"对齐-互补"框架：

┌─────────────────────────────────────────────────────────┐
│                    Egbertie（人）是主角                   │
│                      ↑ 直觉力核心                         │
├─────────────────────────────────────────────────────────┤
```

### 与"左脑×右脑"方法的融合

| 维度 | 左脑（鼎） | 右脑（玉） | 论文对应 |
|------|------------|------------|----------|
| **对齐** | 规则、契约、风控验证 | 感知信任度、气场匹配 | 对齐专家 |
| **互补** | 风险量化、案例分析 | 直觉纠错、模式识别 | 互补专家 |
| **路由** | 历史数据支持 | **直觉判断当前状态** | RRS机制 |

**关键认知**：
- 论文的RRS（理性路由捷径）对应我们的"直觉路由"
- 论文用"模型自信度"判断，我们用"人的直觉感知"判断
- **这是升级，不是替代** - 直觉力在AI时代更珍贵

**文档位置**：`memory/methodology/align-complement-framework.md`

**使用场景**：
1. 合伙人咨询时，判断当前用对齐还是互补模式
2. 培训其他决策教练时，讲解人机协作原理
3. 对外分享时，用学术框架支撑实践方法

### Python化评估标准

```
评估维度：
1. 【规则复杂度】简单规则(if/else) → 高Python化价值
              复杂判断(需语义理解) → 保持MD
              
2. 【执行频率】高频执行 → 高Python化价值（省Token）
             低频执行 → 保持MD（开发成本）
             
3. 【确定性】确定性输出 → 高Python化价值
           开放性输出 → 保持MD
           
4. 【维护成本】Python代码维护成本 vs MD维护成本
```
