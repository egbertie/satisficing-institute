## 三、常见场景处理

### 场景1: 紧急更新（时间不足）

**策略**: 仅更新P0级文件，P1/P2延后

```markdown
**紧急更新模式**
- 仅执行P0级文件更新
- 在检查清单中标记P1/P2为"延后处理"
- 设置提醒，24小时内补全P1级
```

### 场景2: 大范围变更（影响>10个文件）

**策略**: 分批更新，分批验证

```markdown
**分批更新策略**
第一批: P0级文件（必须立即更新）
第二批: P1级文件（当天内更新）
第三批: P2级文件（本周内更新）

每批完成后执行验证
```

### 场景3: 发现遗漏文件（更新后发现）

**策略**: 立即补更，记录教训

```markdown
**遗漏处理流程**
1. 立即更新遗漏文件
2. 更新知识图谱，添加新关联
3. 记录"遗漏原因"到流程改进日志
4. 更新检查清单模板，避免再次遗漏
```
