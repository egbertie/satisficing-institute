## 数据库结构

### 属性字段

| 字段名 | 类型 | 选项/说明 |
|--------|------|----------|
| 会议标题 | Title | 自动填充 |
| 会议类型 | Select | 晨会 / 周会 / 月度复盘 / 专题会 |
| 日期 | Date | 会议日期 |
| 时间 | Text | 如：09:00-09:15 |
| 状态 | Select | 计划中 / 进行中 / 已完成 / 已归档 |
| 主持人 | Person | 满意妞/CONTENT/EEO/PEO |
| 记录人 | Person | |
| 参与人 | Multi-select | |
| 相关项目 | Relation | 关联项目数据库 |
| 会议链接 | URL | 飞书/腾讯会议链接 |

## 视图配置

### 1. 日历视图
- 按日期展示所有会议
- 颜色区分：晨会(绿)/周会(蓝)/月度(紫)/专题(橙)

### 2. 看板视图
- 按状态分组：计划中 → 进行中 → 已完成 → 已归档

### 3. 列表视图
- 按时间倒序排列
- 显示：标题、类型、日期、状态

### 4. 筛选视图
- **本周会议**: 日期 within this week
- **我参与的**: 参与人 contains @我
- **待归档**: 状态 = 已完成

---

## 自动化规则（可选）

| 触发条件 | 动作 |
|----------|------|
| 状态 = 已完成 | 发送通知到群 |
| 日期 = 今天 | 标记状态为进行中 |
| 会议类型 = 周会 | 自动添加标签 #周会 |

---

## 使用流程

1. **会前**: 创建页面，选择对应模板，填写基本信息
2. **会中**: 实时记录要点，标记行动项
3. **会后**: 24小时内完善纪要，更新状态为已完成
4. **归档**: 一周后自动归档或手动归档
