## 四、经验固化（承诺洗澡经验提炼）

### 4.1 核心经验

**有效做法**:
1. **诚实标注先行** - 先暴露问题，再修复
2. **脚本化批量处理** - 144个Skill批量提取，高效
3. **验证驱动** - 每个修复都有量化验证
4. **渐进式修复** - 不求快，求扎实

**避免的做法**:
1. ❌ 追求完成度数字
2. ❌ 框架冒充系统
3. ❌ 监控冒充执行
4. ❌ 隐瞒未完成部分

### 4.2 固化机制

**防复发机制**:
1. 每次汇报必须包含"未完成部分"
2. 以"可用/不可用"代替"完成度百分比"
3. 以"验证通过"代替"声称完成"
4. 用户可随时验证任何声称

**快速检查脚本**:
```bash
# 检查知识管理状态
python3 scripts/validate_knowledge_graph.py

# 检查自动化管道
python3 scripts/automation_pipeline_v2.py

# 检查任务和规则
```
