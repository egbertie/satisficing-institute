### 📋 具体使用场景

#### 场景1：工作文件自动备份
```bash
# Cron任务：每日 03:00 备份工作空间
0 3 * * * python3 plugins/feishu_drive_uploader.py upload /backup/daily_$(date +%Y%m%d).zip
```

**备份内容**:
- MEMORY.md, SOUL.md, USER.md
- SKILL文档和代码
- 项目配置文件
- 重要日志文件

#### 场景2：大文件存储
- 上传大型文档、PDF报告
- 自动压缩（99.9%压缩率）
- 生成分享链接（权限开通后）

#### 场景3：飞书通道消息备份
- 飞书IM消息 → 自动同步到飞书云盘
- 生成对话记录存档

## 🔄 备份策略

### 文档双备份
```
当前页面生成文档
       │
       ├──► 飞书云盘（备份1）
       │
       └──► 企微文档（备份2）
```

### 消息单通道备份
```
飞书IM消息 ──► 飞书云盘存档（仅飞书通道）
企微消息 ────► 企微会话存档（仅企微通道）
当前对话 ────► 不跨通道同步（减少Token）
```

## 📝 Token优化策略

| 操作 | 频率 | Token消耗 | 优化措施 |
|------|------|-----------|---------|
| 飞书文件上传 | 每日1次 | ~500 | 压缩后上传 |
| 企微日程查询 | 按需 | ~200 | 缓存结果 |
| 企微待办同步 | 每6小时 | ~300 | 增量同步 |
| 文档备份 | 每日1次 | ~800 | 仅变更文件 |
| 消息备份 | 实时 | ~100/条 | 仅关键消息 |

**预估每日Token消耗**: ~3,000（当前预算内）
