## 二、Token最优化方案

### 核心原则

```
P0 (不可削减): 备份+安全+核心监控
P1 (可优化): 仪式+审计+知识管理  
P2 (可暂停): 学习+研究+非核心报告
P3 (立即禁用): 重复任务+高频检查
```

### 优化措施

#### 1. 备份合并（节省20K/天）

| 原任务 | 频率 | Token | 优化后 |
|--------|------|-------|--------|
| 每日备份(01:53) | 每日 | 10K | **禁用** |
| 灾备复刻(03:07) | 每日 | 20K | **保留** |
| 备份清理(03:42) | 每日 | 2K | **改为每周** |

**理由**: 灾备复刻已包含备份内容，无需重复

#### 2. 监控降频（节省12K/天）

| 原任务 | 原频率 | Token | 优化后 |
|--------|--------|-------|--------|
| Token熔断监控 | 每2小时 | 12K/天 | **每6小时** → 4K/天 |
| 动态权威切换 | 每2小时 | 6K/天 | **每4小时** → 3K/天 |

**理由**: Token变化缓慢，6小时检查足够

#### 3. 仪式精简（节省5K/天）

| 原任务 | 频率 | Token | 优化后 |
|--------|------|-------|--------|
| 晨间图腾 | 每日 | 2K | **保留** |
| 黄昏仪式 | 每日 | 2K | **保留** |
| 3D情景规划 | 每周一 | 15K | **改为双周** |
| 4D-WhatIf | 每周五 | 15K | **改为双周** |
| 5D元认知 | 每周六 | 15K | **改为双周** |

#### 4. 审计按需（节省8K/天）

| 原任务 | 原频率 | Token | 优化后 |
|--------|--------|-------|--------|
| 蓝军审计 | 每日 | 8K | **每周3次** → 3.4K/天 |
| 7浪费追踪 | 每日 | 3K | **保留**（低Token） |
| 知识图谱 | 每日 | 5K | **每周2次** → 1.4K/天 |

#### 5. 禁用非核心任务（节省30K/天）

**立即禁用**:
- 每日站会(10:00) - 20K
- 每日安全检查(09:30) - 改为每周2次
- milestone检查(09:45) - 15K
- 每周组织罗盘(周一06:37) - 保留但精简
- 每日知识萃取(23:47) - 10K
- 每日进度报告(21:47) - 5K
