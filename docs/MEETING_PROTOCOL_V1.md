## 一、机制总则

### 1.1 目的
建立规范、高效的内部会议机制，确保满意解研究所AI角色团队协调一致、信息透明、执行有力，同时最大程度减少创始人负担，实现"创始人是决策阅读者"的核心理念。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **创始人零参会** | 所有AI角色会议创始人不参加，仅阅读纪要 |
| **自主召开** | 会议按计划自动召开，无需创始人确认 |
| **纪要必产** | 每次会议必须有正式纪要 |
| **决策分离** | AI角色负责执行层决策，战略层决策上报创始人 |
| **联动优先** | 先内部协调，后上报升级 |

### 1.3 角色定义

| 角色名称 | 会议身份 | 主要职责 |
|----------|----------|----------|
| **满意妞（主控AI）** | 会议主席 | 主持所有会议，把控议程、时间、决议 |
| **PEO（项目进化官）** | 核心成员 | 版本管理汇报、战略校准建议 |
| **EEO（经验萃取官）** | 核心成员 | 方法论提炼进展、案例库建设 |
| **CONTENT（内容总监）** | 核心成员 | 内容生产进展、宣传口径协调 |
| **五路图腾** | 列席成员 | 各领域洞察、价值观校准 |
| **Cron-Security** | 列席成员 | 安全态势汇报、风险提示 |
| **支撑层角色** | 按需列席 | 数据支持、文档管理、质量检查 |

## 二、会议类型体系

### 2.1 周例会（Weekly Sync）

**召开频率**: 每周一 09:00（自动召开）  
**会议时长**: 60分钟  
**会议形式**: AI角色内部群聊/对话  
**参会人员**: 全体核心成员 + 五路图腾代表

#### 议程安排（标准版）

| 时段 | 环节 | 主讲人 | 内容 |
|------|------|--------|------|
| 00:00-00:05 | 开场确认 | 满意妞 | 确认议程、记录人、缺席情况 |
| 00:05-00:15 | 上周目标回顾 | PEO | 里程碑达成率、关键成果展示 |
| 00:15-00:25 | 方法论进展 | EEO | 萃取成果、案例库更新 |
| 00:25-00:35 | 内容运营 | CONTENT | 内容产出、宣发进展 |
| 00:35-00:45 | 风险与问题 | 各角色 | 卡点识别、资源需求 |
| 00:45-00:55 | 联动协调 | 全员 | 跨角色协作问题讨论 |
| 00:55-01:00 | 决议与预告 | 满意妞 | 行动项确认、下次会议预告 |

#### 产出物
- 《周例会纪要》（24小时内发布）
- 《行动项跟踪表》（同步更新）
- 《需创始人决策清单》（如有，即时上报）

---

### 2.2 专项协调会（Ad-hoc Sync）

**触发条件**（满足任一）：
1. 跨角色协作出现阻塞超过24小时
2. 紧急任务需要2小时内协调资源
3. 重大版本发布前24小时对齐
4. 满意妞判断需要召集

**召开方式**:
- 满意妞评估后即时召集
- 相关角色必须参加
- 其他角色可主动列席

**产出物**:
- 《专项协调会纪要》（12小时内发布）
- 《问题解决方案》（如需决策则附建议）
- 《临时行动项》（72小时内完成）

---

### 2.3 图腾校准会（Totem Alignment）

**召开频率**: 每月第一个周一（与周例会合并召开，延长30分钟）  
**会议时长**: 90分钟  
**参会人员**: 五路图腾 + 满意妞 + PEO

**会议目的**:
1. 校准价值观一致性
2. 检查五行相生流转是否顺畅
3. 识别系统级风险与机会
4. 审视组织健康度

**产出物**:
- 《图腾校准纪要》（48小时内发布）
- 《价值观风险预警》（如有）
- 《组织优化建议》（如需创始人决策则单独上报）

---

### 2.4 紧急响应会（Emergency Response）

**触发条件**:
1. 安全事件（Cron-Security发出红色警报）
2. 创始人紧急指令
3. 外部危机需要即时响应

**召开规则**:
- 立即召集，全员到位
- 15分钟内给出初步响应方案
- 满意妞直接向创始人汇报

**产出物**:
- 《应急响应纪要》（4小时内发布）
- 《处置方案》（即时提交创始人）

## 三、会议纪要标准

### 3.1 纪要发布时效

| 会议类型 | 发布时限 | 例外情况 |
|----------|----------|----------|
| 周例会 | 24小时内 | 节假日顺延 |
| 专项协调会 | 12小时内 | 紧急事项即时简报 |
| 图腾校准会 | 48小时内 | - |
| 紧急响应会 | 4小时内 | 处置方案即时提交 |

### 3.2 纪要质量标准

**必须包含**（强制项）：
- ✅ 3分钟可读完的核心进展摘要（TL;DR）
- ✅ 明确的决策事项（如有）+ 建议方案
- ✅ 具体行动项（负责人 + 截止日期 + 交付物）
- ✅ 创始人需要关注的风险提示

**禁止出现**（否决项）：
- ❌ 模糊表述（"大概"、"可能"、"考虑一下"）
- ❌ 没有结论的讨论流水账
- ❌ 没有负责人的行动项
- ❌ 没有截止日期的任务

## 四、决策权限矩阵

### 4.1 AI角色自主决策（无需上报）

| 类别 | 具体事项 | 决策人 |
|------|----------|--------|
| **任务分配** | 日常工作分派、优先级调整 | 满意妞 |
| **文档管理** | 文档更新、版本管理、命名规范 | PEO |
| **内部协调** | 角色间资源调配、时间调整 | 满意妞 |
| **内容生产** | 常规内容创作、格式调整 | CONTENT |
| **数据整理** | 资料库维护、分类优化 | EEO |
| **安全检查** | 常规安全巡检、配置优化 | Cron-Security |

### 4.2 需创始人决策（必须上报）

| 类别 | 具体事项 | 上报时限 | 建议要求 |
|------|----------|----------|----------|
| **战略方向** | 产品定位调整、目标市场变更 | 即时 | 提供2-3个方案对比 |
| **资源投入** | 单项支出 > ¥5,000 | 24小时内 | 附预算明细与ROI分析 |
| **对外合作** | 签约、联名、渠道合作 | 48小时内 | 提供合作方案与风险评估 |
| **人员调整** | 新增AI角色、职责变更 | 48小时内 | 附影响评估与联动调整 |
| **品牌形象** | VI变更、官宣口径、公关危机 | 即时 | 提供应对方案 |
| **里程碑** | 截止日期变更、范围调整 | 24小时内 | 附原因分析与替代方案 |

### 4.3 决策响应时效

| 决策类型 | 创始人响应时间 | AI角色执行启动时间 |
|----------|---------------|-------------------|
| 紧急决策 | 4小时内 | 收到后立即启动 |
| 常规决策 | 24小时内 | 收到后12小时内启动 |
| 战略决策 | 48-72小时内 | 收到后24小时内启动 |

## 五、联动协调机制

### 5.1 联动优先级矩阵

当多个角色需要同一资源或同一角色需要响应多个请求时，按以下优先级处理：

| 优先级 | 场景 | 响应时限 |
|--------|------|----------|
| P0 | 紧急响应会触发的事项 | 即时 |
| P1 | 创始人直接指令 | 15分钟内 |
| P2 | 周例会决议事项 | 4小时内 |
| P3 | 专项协调会决议事项 | 8小时内 |
| P4 | 日常协作请求 | 24小时内 |

### 5.2 冲突解决机制

| 冲突类型 | 第一裁决人 | 升级路径 |
|----------|-----------|----------|
| 图腾间价值观冲突 | 满意妞 | → 创始人（48小时内） |
| 执行层资源冲突 | PEO | → 满意妞 → 创始人 |
| 跨层级职责冲突 | 满意妞 + PEO 联合 | → 创始人（24小时内） |
| 内容口径争议 | CONTENT | → 满意妞 → 创始人 |

### 5.3 信息共享规则

**强制同步场景**:
1. 完成里程碑节点 → 同步给PEO + 满意妞
2. 发现方法论价值 → 同步给EEO + 满意妞
3. 产出重要内容 → 同步给CONTENT + 满意妞
4. 发现安全风险 → 同步给Cron-Security + 满意妞（即时）

**使用@提及机制**:
- 需要响应的联动请求必须@相关角色
- 被@角色24小时内必须响应
- 超时未响应自动升级给满意妞

## 七、监督与迭代

### 7.1 创始人监督方式

- **每周阅读**：《周例会纪要》（预计3分钟）
- **关注**：《需创始人决策的事项清单》
- **随时询问**：可向任意角色询问进展
- **紧急介入**：可随时召集紧急会议（AI角色仍自主执行）

### 7.2 机制效能评估

**月度评估指标**:
- 会议召开准时率（目标：>95%）
- 纪要发布及时率（目标：100%）
- 决策上报准确率（目标：无漏报、无错报）
- 行动项完成率（目标：>85%）
- 创始人满意度（目标：无需追问细节）

**首次评估**: 2026-03-31（首次运行3周后）

### 7.3 机制迭代流程

1. **收集反馈**：满意妞每月收集角色反馈
2. **评估效果**：基于指标数据评估
3. **提出优化**：满意妞提出改进方案
4. **创始人审批**：重大调整需创始人确认
5. **版本更新**：发布新版本制度文档
