## 🔴 高危风险（立即修复）

### R1: API密钥明文存储 + 权限过宽
**发现**：
- 6个.env文件存储API密钥
- 权限644（其他用户可读）
- .gitignore未排除.env文件

**风险**：
- 密钥泄露可导致账户被盗
- 被恶意利用产生高额费用
- 数据被窃取

**修复措施**：
```bash
# 1. 立即收紧权限
chmod 600 /root/.openclaw/workspace/.env
chmod 600 /root/.openclaw/workspace/.config/*.env

# 2. 更新.gitignore
echo ".env" >> /root/.openclaw/workspace/.gitignore
echo ".config/*.env" >> /root/.openclaw/workspace/.gitignore

# 3. 验证Git历史无密钥泄露
git log --all --full-history -- .env
```

### R2: 扩展插件供应链风险
**发现**：
- 4个第三方扩展：feishu、kimi-claw、kimi-search、wecom-openclaw-plugin
- wecom插件含37个node_modules依赖包
- 无版本锁定/签名验证机制

**风险**：
- 恶意npm包注入
- 供应链攻击
- 数据外泄

**修复措施**：
1. 建立扩展白名单机制
2. 审查所有npm包来源
3. 定期扫描依赖漏洞
4. 敏感操作二次确认

---

### R3: 网络请求代码散布广泛
**发现**：
- 836个Python文件中大量含网络请求
- 49个文件使用requests/urllib
- 缺乏统一出站流量审计

**风险**：
- 恶意代码外泄数据
- 被植入C2后门
- 异常流量难以发现

**修复措施**：
1. 建立网络请求白名单
2. 敏感域名需用户确认
3. 出站流量日志记录
4. 异常流量告警

## 🟡 中危风险（本周修复）

### R4: 备份文件残留
**发现**：
- extensions-backups目录含多版本备份
- 备份中可能包含旧密钥/配置
- 无清理策略

**影响**：
- 历史密钥泄露风险
- 磁盘空间浪费
- 配置碎片化

**修复**：
```bash
# 清理过期备份
find /root/.openclaw/extensions-backups -mtime +30 -delete
```
