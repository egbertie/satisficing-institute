# 不掉链子保障方案 V1.0

> 针对"完成即停、忘记迭代、效率不持续"问题的解决方案

---

## 🎯 核心问题

| 问题 | 表现 | 后果 |
|------|------|------|
| 完成即停 | 子代理完成→报告→等待用户→再启动 | 时间断层、效率损失 |
| 忘记迭代 | 忙于执行→忘记优化→重复低效 | 长期停滞、无法进化 |
| 被动响应 | 用户推动→才行动→不主动思考 | 依赖性强、自主性弱 |

---

## 🛡️ 三重保障机制

### 保障1：自动触发机制（技术层）

**规则**：任何任务完成，自动触发检查

```
任务完成 → 立即检查：
  ├─ 是否有依赖此任务的后继？→ 自动启动
  ├─ 是否有可并行的其他任务？→ 自动启动
  ├─ 是否达到整点/半点？→ 生成进度快照
  └─ 用户是否在等待？→ 立即汇报
```

**实施**：
- [ ] 修改 `task_coordinator_v2.py` 添加 `on_task_complete` 钩子
- [ ] 每个子代理完成时，自动调用协调器检查后续
- [ ] 建立任务依赖图，自动识别可启动任务

**验证标准**：子代理完成后5分钟内，自动启动下一项或汇报，无需等待用户指令

---

### 保障2：强制反思机制（流程层）

**规则**：每小时必须反思，每日必须总结

| 触发条件 | 强制动作 | 输出物 |
|---------|---------|--------|
| 每完成3个任务 | 反思：哪里可以更快？ | 优化建议1条 |
| 每小时整点 | 检查：是否有遗漏？ | 状态快照 |
| 每日22:00 | 总结：今日效率如何？ | 效率报告 |
| 每周日 | 复盘：本周迭代了什么？ | 迭代日志 |

**实施**：
- [ ] Skill添加 `mandatory_reflection` 模块
- [ ] 使用 cron 定时强制触发反思
- [ ] 反思结果写入 `memory/迭代日志.md`

**验证标准**：每日至少1条优化建议被记录并实施

---

### 保障3：用户契约机制（承诺层）

**规则**：对用户承诺必须兑现，否则自动触发补救

```
对用户承诺 → 自动记录到契约清单 → 倒计时监控
                ↓
      到期前2小时预警 → 到期前30分钟紧急 → 到期自动触发补救
                ↓
      未完成 → 自动生成补救方案 → 并行执行补救
```

**实施**：
- [ ] 创建 `docs/用户契约清单.md`
- [ ] 任何承诺自动记录，带截止时间
- [ ] 到期前自动预警，超期自动补救

**验证标准**：用户承诺兑现率 > 95%
