## 一、问题诊断

### 1.1 多文档重复存储问题
| 内容 | 存储位置 | 数量 | 状态 |
|------|----------|------|------|
| 战略定位 | V1.3/战略定位1.1/V2.0 | 3处 | 🔴 不一致 |
| 专家档案 | 专家替身档案/知识图谱 | 2处 | 🟡 部分同步 |
| 任务状态 | TASK_MASTER/个人看板 | 2处 | 🟡 延迟同步 |
| 方法论 | 百科全书/各实施手册 | 多处 | 🔴 版本不一 |

### 1.2 根本原因
1. **缺乏单一事实源意识**
2. **修改后未同步关联文档**
3. **无变更追踪机制**
4. **无审核机制**

## 三、变更同步机制

### 3.1 修改触发规则
```
修改单一源 → 触发关联更新 → 记录变更日志
```

### 3.2 自动同步类型
| 类型 | 触发条件 | 自动动作 |
|------|----------|----------|
| 战略变更 | 战略定位修改 | 提醒更新PitchDeck |
| 专家信息 | 专家档案修改 | 更新知识图谱 |
| 任务状态 | 任务完成 | 更新相关报告 |

### 3.3 手动审核类型
| 类型 | 原因 | 审核人 |
|------|------|--------|
| 官宣文案 | 影响对外形象 | Egbertie |
| 核心方法论 | 影响业务逻辑 | Egbertie |
| 定价信息 | 影响商业策略 | Egbertie |

## 四、引用统一管理

### 4.1 引用规范
```markdown
# 内部引用
[A满意哥专属文件夹/XX/XX.md]

# 专家引用  
[专家:黎红雷教授](experts/li_honglei.md)

# 文档引用
[任务总清单](docs/TASK_MASTER.md)

# 记忆引用
[2026-03-20记录](memory/2026-03-20.md#章节)
```

### 4.2 失效引用检测
每周六自动扫描：
1. 所有.md文件中的引用
2. 验证目标文件是否存在
3. 标记失效引用
4. 生成修复清单
