# 完全静默 vs 休眠 概念区分指南

> **文档目的**: 明确区分"完全静默"和"休眠"两个概念，避免执行混淆  
> **创建时间**: 2026-03-22  
> **适用对象**: 主控AI、Cron子代理、用户

---

## 核心区别

| 维度 | 完全静默 (Complete Silence) | 休眠 (Hibernation) |
|------|----------------------------|-------------------|
| **本质** | 紧急止损状态 | 正常休息状态 |
| **Token目标** | **严格为0** | 极低消耗 (<100/天) |
| **备份任务** | ❌ **禁用** | ✅ **保留** |
| **Cron任务** | 全部禁用 | 禁用大部分，保留核心 |
| **触发条件** | 用户指令 / Token<10% | 10分钟无交互 / 夜间 |
| **数据风险** | 有（无自动备份） | 无 |
| **适用场景** | Token危机、紧急止损 | 正常夜间休息 |

## 指令映射表

### 明确触发完全静默
| 指令 | 系统响应 |
|------|----------|
| "完全静默" | ✅ 进入模式A，Token=0，全部停止 |
| "绝对静默" | ✅ 进入模式A |
| "停止一切" | ✅ 进入模式A |
| "静默模式" | ✅ 进入模式A |

### 明确触发休眠
| 指令 | 系统响应 |
|------|----------|
| "休眠" | ✅ 进入模式B，保留备份 |
| "睡觉" | ✅ 进入模式B |
| "暂停" | ✅ 进入模式B |
| "休息" | ✅ 进入模式B |

### 唤醒指令
| 指令 | 系统响应 |
|------|----------|
| "唤醒" | ✅ 退出任何休眠/静默状态 |
| "开始" | ✅ 退出任何休眠/静默状态 |
| "继续" | ✅ 退出任何休眠/静默状态 |
| 任意消息 | ✅ 自动唤醒 |

## 执行检查清单

### 进入完全静默前检查
- [ ] 用户明确说了"完全静默"/"绝对静默"/"停止一切"
- [ ] 或 Token < 10%
- [ ] 禁用所有Cron任务（包括备份）
- [ ] 记录当前待办到 `pending-confirmation-queue.md`
- [ ] 回复: "已进入完全静默模式，Token严格为0，备份已暂停。发送任意消息唤醒。"

### 进入休眠前检查
- [ ] 10分钟无交互 / 用户说"休眠"
- [ ] 禁用非核心Cron任务
- [ ] 保留: 每日备份、灾备同步
- [ ] 回复: "已进入标准休眠模式，保留每日备份，其他任务已暂停。发送任意消息唤醒。"

### 唤醒后检查
- [ ] 恢复所有Cron任务（根据模式）
- [ ] 如从完全静默唤醒，检查 `pending-confirmation-queue.md`
- [ ] 报告休眠期间是否有异常

## 常见错误

### ❌ 错误1: 用户说"休眠"但执行了完全静默
**后果**: 备份中断，数据风险
**正确**: "休眠" = 模式B，保留备份

### ❌ 错误2: 完全静默期间仍有Cron运行
**后果**: Token消耗，违反零Token保证
**正确**: 完全静默 = 全部禁用

### ❌ 错误3: 混淆指令识别
**后果**: 用户意图与执行不匹配
**正确**: 严格按指令映射表执行
