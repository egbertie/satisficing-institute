#### 监控脚本

```python
# token-guardian.py
import json
import os
from datetime import datetime

class TokenGuardian:
    def __init__(self):
        self.thresholds = {
            'warning': 0.50,
            'critical': 0.30,
            'emergency': 0.15
        }
        self.state_file = 'memory/token-state.json'
    
    def check_token_level(self, current, total):
        ratio = current / total
        
        if ratio > self.thresholds['warning']:
            return 'healthy', '🟢'
        elif ratio > self.thresholds['critical']:
            return 'warning', '🟡'
        elif ratio > self.thresholds['emergency']:
            return 'critical', '🟠'
        else:
            return 'emergency', '🔴'
    
    def take_action(self, level):
        actions = {
            'healthy': self._action_healthy,
            'warning': self._action_warning,
            'critical': self._action_critical,
            'emergency': self._action_emergency
        }
        return actions[level]()
    
    def _action_warning(self):
        # 增加Token分析到晨报
        # 缩短确认窗口
        return {"action": "enable_token_tracking", "notify": True}
    
    def _action_critical(self):
        # 停用非核心Skill
        # 切换至备选模型
        return {"action": "enable_fallback_mode", "notify": True}
    
    def _action_emergency(self):
        # 进入冬眠模式
        # 必须人工确认恢复
        return {"action": "enable_hibernation", "notify": True, "require_confirm": True}
```

**量化收益：**
- Token耗尽导致的系统停摆：从"可能"降至"不可能"
- 预警提前量：从"突然耗尽"到"多级预警"

### 2.2 外部依赖处理流程：内部优先70%

#### 当前问题
- 蓝军任务等待外部附件5天
- 未启用内部替代方案

#### 优化方案：双路径执行模型

```
┌─────────────────────────────────────────────────────────────┐
│                  外部依赖双路径执行模型                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  触发条件: 任务依赖外部输入                                 │
│                                                             │
│  决策点1: 内部可完成度评估（< 2小时）                       │
│  ├── 内部资源清单检查                                       │
│  ├── 可完成度百分比估算                                     │
│  └── 质量差距评估                                           │
│                                                             │
│      ├─ 内部可完成度 ≥ 70% ──→ 路径B: 内部优先              │
│      │                                                      │
│      │  路径B执行:                                          │
│      │  ├── 立即启动内部版本                                │
│      │  ├── 标记"待外部输入优化"                            │
│      │  ├── 72小时无外部输入，自动完成                      │
│      │  └── 收到外部输入后增量更新                          │
│      │                                                      │
│      └─ 内部可完成度 < 70% ──→ 路径A: 等待外部            │
│                                                              │
│         路径A执行:                                          │
│         ├── 明确外部依赖的截止时间                          │
│         ├── 24小时无进展，主动询问                          │
│         ├── 72小时无输入，升级至路径B                       │
│         └── 阻塞超过3天，必须上报                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 自动化实现

```python
# dependency-handler.py
class DependencyHandler:
    def __init__(self):
        self.blocking_threshold = 72  # 小时
        self.internal_completion_threshold = 0.70
    
    def handle_external_dependency(self, task, dependency):
        # 检查依赖状态
        if dependency.is_available():
            return self.execute_path_a(task, dependency)
        
        # 评估内部可完成度
        internal_completeness = self.assess_internal_completeness(task)
        
        if internal_completeness >= self.internal_completion_threshold:
            # 路径B: 内部优先
            return self.execute_path_b(task, internal_completeness)
        else:
            # 路径A: 等待外部，但设置超时
            return self.execute_path_a_with_timeout(task, dependency)
    
    def execute_path_b(self, task, completeness):
        result = {
            'path': 'B - Internal First',
            'action': 'execute_with_internal_resources',
            'completeness': completeness,
            'status': 'completed_pending_enhancement',
            'note': f'This result is {completeness*100:.0f}% complete. '
                    f'External input can enhance it further.'
        }
        return result
```

**量化收益：**
- 任务阻塞时间：从"平均2-5天"降至"< 24小时"
- 外部依赖导致的延期：减少80%

#### 健康度评分算法

```python
# health-scanner.py
class WorkspaceHealthScanner:
    def __init__(self):
        self.dimensions = {
            'structure': 0.25,
            'freshness': 0.25,
            'reference': 0.25,
            'security': 0.25
        }
    
    def scan(self):
        scores = {}
        scores['structure'] = self.scan_structure()
        scores['freshness'] = self.scan_freshness()
        scores['reference'] = self.scan_references()
        scores['security'] = self.scan_security()
        
        total_score = sum(
            scores[dim] * weight 
            for dim, weight in self.dimensions.items()
        )
        
        return {
            'total_score': round(total_score, 1),
            'dimension_scores': scores,
            'status': self.get_status(total_score),
            'issues': self.issues,
            'recommendations': self.generate_recommendations()
        }
    
    def get_status(self, score):
        if score >= 90:
            return {'level': 'healthy', 'emoji': '🟢', 'action': '保持'}
        elif score >= 70:
            return {'level': 'warning', 'emoji': '🟡', 'action': '关注'}
        elif score >= 50:
            return {'level': 'critical', 'emoji': '🟠', 'action': '处理'}
        else:
            return {'level': 'emergency', 'emoji': '🔴', 'action': '立即处理'}
```

**量化收益：**
- 问题发现时间：从"被动发现"到"主动预警"
- 系统健康度透明度：从"黑盒"到"可视"

class FirstPrincipleAuditor:

            'results': results,
            'overall_compliance': self.calculate_compliance(results),
