## 一、第一性原理：边界为何重要

### 1.1 OPC形态的硬约束

第一年满意解是一个**One Person Company（一人公司）**：

| 资源维度 | 约束条件 | 影响 |
|---------|---------|------|
| 人力 | Egbertie + AI助手 | 无法承载团队化业务 |
| 时间 | 需兼顾其他业务 | 满意解时间有限且碎片化 |
| 资金 | 启动期现金流优先 | 不能投入大成本试错 |
| 注意力 | 单线程处理 | 无法并行服务多个复杂项目 |

**结论**：第一年不是"能做什么"，而是"只能做什么"。

### 1.2 边界决策的三原则

**原则1：核心能力内聚**
- 必须做：只有Egbertie能做的事（面谈、决策建议、专家调度）
- 合作做：Egbertie能做但效率低的事（设计、研究、尽调）
- 不做：Egbertie不该做的事（开发、运营、培训）

**原则2：验证优先于扩张**
- 第一年唯一目标：验证"合伙人评估服务"是否有人买单
- 非目标：品牌建设、规模扩张、团队搭建

**原则3：极简交付最大化价值**
- 客户要的是"不后悔的合伙人选择"，不是工具/报告/体系
- 交付越简单，价值越清晰

## 二、要做（Must Do）——核心服务

### 2.1 唯一核心产品：合伙人决策评估服务

| 要素 | 定义 |
|-----|------|
| **服务名称** | 合伙人决策评估服务（Partner Decision Assessment） |
| **服务对象** | 获得政府补贴的初创期智能硬件企业家（详见 [YEAR1_PLAN 客户画像](SATISFICING_YEAR1_PLAN_V1.0.md#三唯一客户画像)） |
| **服务目标** | 帮助客户做出不后悔的合伙人选择 |
| **服务边界** | 仅覆盖"合伙人选择决策"这一单一决策点 |
| **服务周期** | 2-4周 |
| **服务定价** | 待定价（建议区间：3-8万元/项目） |

### 2.2 服务交付内容

**交付物（极简版）**：

1. **3次深度面谈**
   - 第1次：客户背景与需求澄清（60分钟）
   - 第2次：候选人评估与风险识别（90分钟）
   - 第3次：决策建议与后续规划（60分钟）

2. **1份评估报告**
   - 格式：飞书/Notion文档（不做PDF设计）
   - 内容：候选人画像、风险清单、决策建议、后续行动
   - 页数：控制在5页以内

3. **1次专家会诊**（可选）
   - 针对复杂案例，调度专家网络进行会诊
   - 单独收费：1-2万元/次

### 2.3 必须由Egbertie亲自做的环节

| 环节 | 原因 | 不可替代性 |
|-----|------|-----------|
| 客户面谈 | 22年体感的直觉判断 | 经验无法外包 |
| 评估结论输出 | 综合判断与决策建议 | 责任无法外包 |
| 专家网络调度 | 人脉资源与信任关系 | 关系无法外包 |
| 客户关系维护 | 信任建立与口碑传播 | 人格无法外包 |

## 三、合作做（Partner）——外包服务

### 3.1 合作领域清单

| 领域 | 合作方类型 | 满意解角色 | 合作模式 |
|-----|-----------|-----------|---------|
| **技术尽调** | 技术专家/第三方机构 | 引荐+监督 | 按项目引荐费 |
| **法律风控** | 律师事务所 | 引荐+监督 | 按项目引荐费 |
| **视觉设计** | 外包设计师 | 需求方+验收方 | 按件付费 |
| **行业研究** | 外包研究员/AI工具 | 需求方+审核方 | 按件付费 |
| **内容分发** | KOL/媒体/渠道 | 内容提供方 | 合作分成 |
| **后续陪跑** | 教练/顾问 | 转介绍 | 转介绍费 |

### 3.2 合作的边界原则

**必须合作的情况**：
1. 非Egbertie核心能力范围
2. 自建成本高于合作成本
3. 时间投入产出比过低

**合作中的满意解角色**：
- **不做执行方**：不负责具体执行工作
- **做连接方**：引荐资源、协调沟通
- **做监督方**：把控质量、确保交付
- **做背书方**：为合作方提供信任背书

### 3.3 合作方画像（理想状态）

| 合作方 | 理想画像 | 寻找渠道 |
|-------|---------|---------|
| 技术尽调专家 | 硬科技领域10年+经验，有投资背景 | 专家网络/投资机构推荐 |
| 法律顾问 | 专注股权设计，服务过早期企业 | 律师协会/创业者推荐 |
| 设计师 | 简洁风格，理解B2B专业感 | 设计平台/朋友推荐 |
| 研究员 | AI工具熟练，信息检索能力强 | 兼职平台/学生兼职 |

## 四、不做（Not-To-Do）——明确排除

### 4.1 第一年绝对不做的事

| 不做的事 | 不做原因 | 诱惑来源 | 替代方案 | 详细说明 |
|---------|---------|---------|---------|---------|
| **技术产品类** | OPC资源约束 | 觉得"系统"才有壁垒 | 使用现有工具 | 详见 [NOT_TODO V1.0 第二章](SATISFICING_NOT_TODO_V1.0.md#二技术产品类绝对不做) |
| **培训内容类** | 分散核心注意力 | 觉得培训"规模更大" | 案例自然溢出 | 详见 [NOT_TODO V1.0 第三章](SATISFICING_NOT_TODO_V1.0.md#三培训内容类绝对不做) |
| **运营社群类** | 运营负担重 | 觉得社群"粘性更强" | 专家网络替代 | 详见 [NOT_TODO V1.0 第四章](SATISFICING_NOT_TODO_V1.0.md#四运营社群类绝对不做) |
| **咨询服务类** | 时间占用大 | 觉得"陪伴更有价值" | 合作方转介绍 | 详见 [NOT_TODO V1.0 第五章](SATISFICING_NOT_TODO_V1.0.md#五咨询服务类绝对不做) |
| **客户市场类** | 能力边界外 | 觉得"市场更大" | 聚焦深耕 | 详见 [NOT_TODO V1.0 第六章](SATISFICING_NOT_TODO_V1.0.md#六客户市场类绝对不做) |
| **团队组织类** | OPC形态约束 | 觉得"公司化更正规" | 轻资产运营 | 详见 [NOT_TODO V1.0 第七章](SATISFICING_NOT_TODO_V1.0.md#七团队组织类绝对不做) |
| **品牌营销类** | 验证期非必要 | 觉得"广告能快速获客" | 口碑传播 | 详见 [NOT_TODO V1.0 第八章](SATISFICING_NOT_TODO_V1.0.md#八品牌营销类绝对不做) |

### 4.2 "不做"的决策逻辑

**第一层：资源不匹配**
- 需要全职团队的事 → 不做
- 需要大额资金投入的事 → 不做
- 需要长期运营维护的事 → 不做

**第二层：阶段不匹配**
- 扩张期才需要做的事 → 不做
- 验证期非必要的事 → 不做
- 提前优化的投入 → 不做

**第三层：能力不匹配**
- Egbertie不擅长的事 → 不做
- 无法建立壁垒的事 → 不做
- 可被标准化替代的事 → 不做

### 4.3 应对"诱惑"的话术

当有人建议满意解"还可以做XXX"时，用以下话术回应：

> "第一年我们只专注一件事：帮助初创期硬件企业家做出不后悔的合伙人选择。其他事情，明年再看。"

当自己想"是不是也该做XXX"时，问自己：

1. 这件事现在不做，会影响核心目标吗？
2. 这件事Egbertie必须亲自做吗？
3. 这件事的投入产出比高于核心服务吗？

## 五、边界决策流程图

```
新业务/新机会出现
       ↓
是否属于"合伙人选择决策"相关？
       ↓
   否 → 不做
       ↓
   是 → 必须由Egbertie亲自做？
           ↓
       否 → 能否找到合作方？
               ↓
           否 → 不做
               ↓
           是 → 合作做
               ↓
       是 → OPC资源能否支撑？
               ↓
           否 → 不做
               ↓
           是 → 纳入核心服务
```

## 六、边界守护原则

### 6.1 月度边界审视

每月最后一个工作日，花30分钟回答：

1. 本月是否有超出边界的行为？
2. 本月是否有新的"诱惑"出现？
3. 边界是否需要调整（收紧/放宽）？

### 6.2 边界调整规则

**可以调整边界的情况**：
- 核心服务已验证成功（有付费客户+正向口碑）
- 资源约束发生变化（如获得融资、有全职加入）
- 市场反馈明确指向新需求

**不可以调整边界的情况**：
- 只是觉得"也可以试试"
- 看到别人在做
- 担心"错过机会"

### 6.3 边界宣言

> 满意解第一年的边界，不是为了限制可能性，而是为了保护专注力。
>
> 在OPC的约束下，我们只能做好一件事。把这件事做到极致，比做十件平庸的事更有价值。
>
> 边界是护城河，不是牢笼。
