# 满意解研究所质量检查流程 V1.0
> **创建时间**: 2026-03-20  
> **执行频率**: 每次交付前必做  
> **执行原则**: 不检查不交付

---

## 一、交付前自检（必须）

### 1.1 文档类检查
```bash
检查清单:
□ 目的明确（为什么做）
□ 范围清晰（给谁用）
□ 步骤完整（怎么做）
□ 标准量化（怎样算完成）
□ 无错别字
□ 链接可访问
□ 示例可运行
```

### 1.2 代码类检查
```bash
检查清单:
□ 功能说明完整
□ 使用方法清晰
□ 错误处理完备
□ 语法无错误
□ 可独立运行
□ 测试用例通过
```

### 1.3 任务类检查
```bash
检查清单:
□ 完成状态真实
□ 验证方式明确
□ 交付物路径正确
□ 无虚报成分
□ 无隐瞒问题
```

---

## 二、交付中他检（抽查）

### 2.1 功能验证
- 运行交付物，验证功能正常
- 测试边界条件
- 确认错误处理有效

### 2.2 文档验证
- 检查必含项完整
- 验证示例可复现
- 确认逻辑自洽

### 2.3 质量验证
- 对照质量标准检查
- 标记不符合项
- 要求返工或接受

---

## 三、交付后追踪（持续）

### 3.1 问题记录
- 记录发现的问题
- 记录用户反馈
- 记录返工情况

### 3.2 根因分析
- 分析问题原因
- 识别模式问题
- 定位标准缺陷

### 3.3 标准改进
- 更新质量标准
- 优化检查流程
- 培训执行人员

---

## 四、检查工具

### 4.1 自动化检查脚本
```bash
# 运行质量检查
python3 scripts/quality_check.py --type=doc --path=docs/
python3 scripts/quality_check.py --type=code --path=scripts/
python3 scripts/quality_check.py --type=task --path=TASK_MASTER.md
```

### 4.2 手动检查模板
- 文档检查模板: `templates/doc_check.md`
- 代码检查模板: `templates/code_check.md`
- 任务检查模板: `templates/task_check.md`

---

## 五、质量门

```
交付物 → 自检 → 他检 → 质量门 → 交付
         ↓      ↓      ↓
       不通过  不通过  不通过
         ↓      ↓      ↓
       返工   返工   返工
```

**质量门标准**:
- 门1: 必含项完整 ✅
- 门2: 功能可用 ✅
- 门3: 测试通过 ✅
- 门4: 用户验收 ✅

---

## 六、问题升级

| 问题级别 | 处理方式 | 升级时限 |
|----------|----------|----------|
| 轻微 | 现场修正 | 立即 |
| 中等 | 返工重做 | 24小时内 |
| 严重 | 暂停交付 | 立即升级 |
