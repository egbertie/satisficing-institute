## 一、定时任务管理（Cron）

### 1.1 当前定时任务清单
| 任务名称 | 频率 | 时间 | 产出位置 |
|---------|------|------|----------|
| 每日自动备份 | 每日 | 01:53 | `/data/backups/daily/` |
| 每日晨报生成 | 每日 | 08:47 | `memory/YYYY-MM-DD.md` |
| Token周度监控 | 每周 | 周日12:00 | `memory/token-weekly-monitor.json` |
| 系统安全检查 | 每日 | 14:00 | `diary/security-check.md` |
| 小时级任务协调 | 每小时 | :24 | `memory/小时协调检查报告.md` |

### 1.2 Cron配置规范
```yaml
# config/cron.yml
cron_jobs:
  backup_daily:
    schedule: "53 1 * * *"
    command: "scripts/backup_daily.sh"
    timeout: 300
    
  morning_report:
    schedule: "47 8 * * *"
    command: "scripts/generate_morning_report.py"
    timeout: 120
    
  token_monitor:
    schedule: "0 12 * * 0"
    command: "scripts/check_token_usage.py"
    timeout: 60
```

### 1.3 任务执行保障
- **失败自动重试**: 最多3次，间隔5分钟
- **失败告警**: 写入 `diary/cron-failures.md`
- **执行日志**: 保留最近30天

## 三、灾备恢复

### 3.1 灾难场景与恢复时间
| 场景 | RTO | RPO | 恢复步骤 |
|------|-----|-----|----------|
| 文件误删 | 5分钟 | 0 | 从每日备份恢复 |
| 系统崩溃 | 30分钟 | 1天 | 重建容器+恢复备份 |
| 数据损坏 | 1小时 | 1天 | 校验后恢复 |
| 完全丢失 | 2小时 | 1天 | 全新部署+恢复 |

### 3.2 恢复操作手册
```bash
# 1. 查看最新备份
ls -la /data/backups/daily/ | tail -5

# 2. 恢复指定日期备份
./scripts/restore_backup.sh backup_20260320_015317

# 3. 验证恢复完整性
./scripts/verify_restore.sh
```

### 3.3 灾备演练
- **频率**: 每季度1次
- **记录**: `docs/DISASTER_DRILL_YYYY-MM-DD.md`
- **改进**: 演练后24小时内更新恢复手册

## 四、安全管理

### 4.1 每日安全检查清单
| 检查项 | 方法 | 标准 | 失败处理 |
|--------|------|------|----------|
| 异常登录 | 检查系统日志 | 无陌生IP | 立即告警 |
| 文件变更 | git diff | 无未授权变更 | 调查来源 |
| API使用 | 检查调用日志 | 无异常调用 | 审查调用者 |
| Token消耗 | 检查使用量 | 无异常峰值 | 分析原因 |

### 4.2 安全事件响应
| 级别 | 事件 | 响应时间 | 行动 |
|------|------|----------|------|
| P0 | 数据泄露风险 | 立即 | 暂停服务+调查+报告 |
| P1 | 异常访问 | 2小时内 | 调查+加固+记录 |
| P2 | 配置变更 | 当日 | 审查+确认+记录 |

### 4.3 安全基线
- **敏感文件**: MEMORY.md 仅在主会话读取
- **API密钥**: 环境变量存储，不硬编码
- **访问控制**: 最小权限原则
- **审计日志**: 所有敏感操作记录

## 五、Token管理

### 5.1 预算控制
| 场景 | 日预算 | 月预算 | 超限处理 |
|------|--------|--------|----------|
| 正常运营 | 1,500 | 45,000 | 预警 |
| 紧急任务 | 3,000 | 90,000 | 需确认 |
| 极限模式 | 5,000 | 150,000 | 仅P0任务 |

### 5.2 优化策略
- **轻量检查**: 只查时间戳，不读全文
- **缓存复用**: 重复内容不重新生成
- **批量处理**: 每小时批量检查，不实时
- **分层检查**: 快速筛选→深度检查

### 5.3 监控告警
- **日用量>80%**: 黄色预警
- **日用量>100%**: 红色预警+暂停非紧急任务
- **周趋势上升>50%**: 分析原因+优化
