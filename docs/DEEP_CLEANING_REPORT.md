## 五、教训记录

### 5.1 为什么忘记了？

| 原因 | 说明 | 预防措施 |
|------|------|----------|
| **概念混淆** | 把"有文档"当成"有机制" | 强制5标准检查 |
| **进度虚报** | 声称90%实际35% | 诚实标注系统 |
| **缺少复盘** | 建立后未检查执行 | 每周机制审计 |
| **标准不清** | 没有统一的5标准 | 现在已确立 |

### 5.2 今后如何避免？

1. **任何新机制必须5标准 checklist**
2. **每周机制审计**: 检查所有机制是否skill化
3. **诚实标注**: 不再声称"已建立"，只报告"已完成/进行中/X%"
4. **深度洗澡**: 每两周全面扫描所有机制
