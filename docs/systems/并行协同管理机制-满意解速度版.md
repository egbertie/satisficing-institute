## 一、并行工作流设计

### 1.1 项目并行原则

**打破串行，能并行的绝不等待：**

```
❌ 传统串行：
研究 → 设计 → 开发 → 测试 → 发布
（每个环节等上一个完成）

✅ 满意解并行：
研究 ─┬→ 设计 ─┬→ 开发 ─┬→ 测试 ─┬→ 发布
      │         │         │         │
      └→ 内容 ─┘         └→ 运营 ─┘
      └→ 游戏 ─┘         └→ 数据 ─┘

（研究30%时，设计、内容、游戏同时启动）
```

### 1.2 任务依赖矩阵

明确哪些任务可以并行，哪些必须串行：

| 任务A | 任务B | 关系 | 说明 |
|-------|-------|------|------|
| 主题研究 | 工具设计 | **可并行** | 研究出框架即可设计工具 |
| 工具设计 | 游戏开发 | **可并行** | 游戏可基于框架先行设计 |
| 服务设计 | 定价审核 | **可并行** | FIN可同时审核成本模型 |
| 产品设计 | 内容创作 | **可并行** | NARR/SOUL可同步创作 |
| 原型开发 | 客户测试 | **可并行** | Persona-Sim可模拟测试 |
| 研究报告 | 专家审核 | **必须串行** | 需等报告完成才能审核 |
| 合同签署 | 服务交付 | **必须串行** | 法律要求 |

### 1.3 启动触发机制

**不等100%，达到30%即可启动下游：**

```
研究完成30%（框架确定）
    ↓ 触发
设计启动（基于框架）
内容启动（基于核心观点）
游戏启动（基于方法论）
    ↓
研究完成60%（核心内容确定）
    ↓ 触发
开发启动（MVP开发）
运营准备启动（物料准备）
    ↓
研究完成100%
    ↓ 触发
正式发布
```

## 二、敏捷迭代机制

### 2.1 小步快跑节奏

| 周期 | 节奏 | 动作 |
|------|------|------|
| **日** | 每日站会（异步） | 各角色更新进度、阻塞问题 |
| **周** | 周迭代 | 交付可用版本、快速验证 |
| **双周** | 双周发布 | 正式发布、客户反馈 |
| **月** | 月度复盘 | 系统优化、架构调整 |

### 2.2 MVP优先原则

**先交付最小可用版本，再持续优化：**

```
第一轮（Week 1）：MVP
- 研究：核心框架
- 工具：基础功能
- 内容：核心观点
- 目标：能跑通核心流程

第二轮（Week 2）：V1.0
- 研究：完整内容
- 工具：完整功能
- 内容：完整包装
- 目标：可对外交付

第三轮（Week 3-4）：V1.x
- 根据反馈优化
- 持续迭代
- 目标：越来越好
```

### 2.3 快速验证闭环

```
设计 → 模拟测试（Persona-Sim）→ 反馈 → 优化 → 小范围真实测试 → 反馈 → 优化 → 正式发布
（每个循环1-3天）
```

## 三、协同调度机制

### 3.1 指挥官调度（每日）

**每日异步站会格式**（各角色在固定频道更新）：

```
【角色名】日站会 2026-03-07

✅ 昨日完成：
- 任务1
- 任务2

🔄 今日进行：
- 任务3（预计今日完成）
- 任务4（需GAME协作）

⚠️ 阻塞/需支持：
- 问题1（需指挥官决策）
- 资源需求（需TECH支持）

📊 进度：XX%（按计划/提前/延迟）
```

### 3.2 PMO资源协调（实时）

**资源冲突时，PMO立即协调：**

```
场景：
GAME需要Persona-Sim测试游戏
同时PROD也需要Persona-Sim测试工具

PMO协调：
- 评估优先级（GAME当前更紧急，15天里程碑）
- 分配时间（上午GAME，下午PROD）
- 同步信息（让Persona-Sim了解上下文）
- 跟踪执行（确保两边都不耽误）
```

### 3.3 跨角色协作协议

**需要协作时，主动发起，不等安排：**

```
发起协作请求：
"@PROD 我需要满意解决策的核心方法论，用于游戏机制设计。
需要内容：决策流程图、关键工具逻辑、典型决策场景。
用途：《创业合伙人选择大富翁》游戏机制。
紧急程度：高（Day 2需要）。
"

响应要求：
- 24小时内必须响应
- 能提供则立即提供
- 不能提供则说明原因和替代方案
- 紧急事项立即升级指挥官
```

## 七、核心原则

### 满意解速度三原则

1. **不等不靠**：达到30%就启动下游，边做边完善
2. **能并则并**：无强依赖的任务100%并行
3. **快错快改**：小步快跑，快速验证，快速迭代

### 协同三原则

1. **主动协同**：需要协作时主动发起，不等安排
2. **24小时响应**：协作请求24小时内必须响应
3. **资源动态调配**：PMO根据优先级动态调配资源

### 质量三原则

1. **MVP不妥协核心**：最小版本，但核心体验不打折
2. **风控不妥协**：法律、财务、安全底线不因速度降低
3. **持续优化**：发布不是结束，是迭代的开始
