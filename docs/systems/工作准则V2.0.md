## 二、资源使用准则

### 2.1 API分级管理策略

#### 🟢 免费API（无限制，充分利用）
| 类型 | 示例 | 策略 |
|------|------|------|
| **本地工具** | DuckDB、本地模型 | ✅ 无限使用，优先调用 |
| **开源API** | 政府开放数据、开源服务 | ✅ 无限制，充分利用 |
| **免费试用** | Claude Code API联调期 | ✅ **免费额度内联调，需要时再付费** |
| **免费版工具** | **Notion免费版**、WPS免费功能 | ✅ **充分利用免费功能，不升级付费** |

**原则**：免费资源不设置调用限制，最大化利用！

#### 🟡 收费/限额API（有规划，合理控制）
| 类型 | 示例 | 策略 |
|------|------|------|
| **按量付费** | Kimi API | ⚠️ 规划使用，控制限额 |
| **次数限制** | Allegretto会员 | ⚠️ 配额管理，超额预警 |
| **订阅服务** | Claude Code | ⚠️ 成本评估，必要时启用 |

**原则**：
- 每次调用前评估必要性
- 批量处理，减少调用次数
- 夜间错峰使用（响应快+省额度）
- 结果复用，避免重复调用

### 2.2 API调用前自检（仅针对收费/限额API）

**免费API**：✅ 直接调用，不设限制

**收费/限额API调用前检查**：
- [ ] 这个任务真的需要API吗？（能否本地解决）
- [ ] 当前配额还充足吗？（检查剩余额度）
- [ ] 能否批量处理？（减少调用次数）
- [ ] 结果可以复用吗？（避免重复生成）
- [ ] 有必要现在调用吗？（能否等到夜间错峰）

### 2.3 成本控制红线

**免费API**：✅ 无限制，充分利用到上限

**收费/限额API**：
- Kimi API（Allegretto）：每日不超过限额50%（保留缓冲）
- 夜间使用：收费API的70%调用在夜间（响应快+省额度）
- 结果保存：100%生成物保存到GitHub（复用）
- 超额预警：达到80%配额时自动预警
