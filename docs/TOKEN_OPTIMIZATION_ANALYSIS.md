### 方案A: 立即止血（Token>80%时执行）

- [ ] 灾备复刻每日同步 → 改为每周1次
- [ ] 全方位灾备同步 → 与灾备复刻合并
- [ ] 厂商API能力监控 → 暂停，手动按需执行
- [ ] 全局文件治理 → 改为每周1次
- [ ] 飞书帮助中心信息迭代 → 已有2次超时，建议暂停

**预期节省**: 每日~40K-60K Token

### 方案B: 深度优化（本周内完成）

**1. 合并重复任务**
```
当前: 灾备复刻(03:07) + 全方位灾备(04:17) = 2个任务
优化: 统一为"每日灾备同步"(03:30) = 1个任务
节省: ~15K-20K Token/天
```

**2. 降低思考级别**
```
每周组织罗盘报告: thinking=high → thinking=medium
每周六项目复盘: thinking=high → thinking=medium
每日站会: thinking=medium → thinking=low
节省: ~20K-30K Token/天
```

**3. 调整执行频率**
```
全局文件治理: 每日 → 每周2次
引用一致性检查: 每周1次 → 每月2次
API双周审查: 双周 → 每月1次
节省: ~10K-15K Token/天
```

**4. 精简任务内容**
```
厂商API能力监控: 监控5个厂商 → 只监控飞书/企微2个
每日安全检查: 5项检查 → 3项核心检查
承诺保障检查: 全量扫描 → 只扫描P0/P1
节省: ~10K-15K Token/天
```

**总计优化后节省**: ~55K-80K Token/天
