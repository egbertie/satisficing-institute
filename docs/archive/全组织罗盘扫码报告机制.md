## 二、数据采集机制

### 2.1 自动采集数据

**每日自动采集（由各角色提交到固定位置）**：

| 数据类型 | 采集源 | 存储位置 |
|---------|--------|---------|
| 任务进度 | 各角色日站会 | /data/daily/YYYYMMDD/ |
| 项目状态 | PMO项目看板 | /data/projects/status.md |
| 财务数据 | FIN财务系统 | /data/finance/weekly.md |
| 风险状态 | 风控组 | /data/risk/weekly.md |
| 研究产出 | 14位专家 | /data/research/weekly.md |

### 2.2 扫描维度

**组织罗盘8个维度**：

```
           战略层（指挥官）
                ↑
    文化层 ← 组织罗盘 → 执行层
   （五图腾）   扫描    （主控AI）
                ↓
           运营层（PMO）

支撑维度：财务、风险、专家、技术
```

**扫描指标**：

| 维度 | 关键指标 | 数据来源 |
|------|---------|---------|
| 战略健康 | 里程碑达成率、战略偏差度 | PMO + 指挥官输入 |
| 组织健康 | 角色活跃度、协作效率 | 各角色日站会 |
| 运营健康 | 任务完成率、阻塞解决时间 | PMO项目看板 |
| 财务健康 | 收入、支出、现金流 | FIN财务报告 |
| 文化健康 | 价值观践行度、团队氛围 | SOUL评估 + 指挥官感知 |
| 研究健康 | 研究产出量、质量评分 | DEAN + BLUE评估 |
| 客户健康 | 客户满意度、复购率 | CSM客户反馈 |
| 风险状态 | 风险数量、等级、应对措施 | 风控组 |

## 三、报告生成流程

### 3.1 周日晚上：数据汇总

```
周日 22:00 - 自动触发数据汇总
    ↓
收集各角色最后更新
    ↓
汇总到 /data/weekly/YYYY-MM-DD/ 目录
    ↓
生成数据摘要
```

### 3.2 周一早上：报告生成

```
周一 06:00 - 自动生成报告
    ↓
REPORT-组织罗盘官扫描全组织数据
    ↓
生成精华版（1页）
生成全要素版（详细）
    ↓
07:00 - 发送给指挥官
```

### 3.3 报告发送

**发送方式**：
- 精华版：直接发送在对话中
- 全要素版：提供文件链接

**发送时间**：周一早上 07:00

## 四、角色职责

### REPORT-组织罗盘官（新增）

**基本信息**：
- 角色ID：report
- 名称：组织罗盘官
- 类型：支撑体系-报告专员
- 核心使命：每周一早上生成全组织罗盘扫码报告

**核心职责**：
1. 每周日晚上汇总全组织数据
2. 周一早上生成精华版+全要素版报告
3. 扫描8个维度的组织健康状况
4. 识别关键进展、风险、优化建议
5. 向指挥官汇报

**协作关系**：
- 从所有角色收集数据
- 从PMO获取项目状态
- 从FIN获取财务数据
- 从风控组获取风险状态
- 从DEAN获取组织架构状态

**输出交付**：
- 每周一 07:00：精华版报告（1页）
- 每周一 07:00：全要素版报告（详细）
- 特殊情况：临时紧急报告
