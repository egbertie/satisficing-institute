## 二、7种浪费类型定义与检测

### Waste 1: Overproduction（过度生产）

**定义**: 生成用户未要求的细节

**症状检测**:
- 回复包含用户未询问的背景信息
- 提供多个解决方案当用户只需要一个
- "以防万一"的额外说明

**典型案例**:
```
用户问: "今天天气如何？"
❌ 浪费回复: "今天天气晴朗，气温20-25度。说到天气，你知道吗，
             气象学是一门复杂的科学，涉及到大气物理学...（500字）"
✅ 精益回复: "晴朗，20-25°C"
```

**检测指标**:
```yaml
overproduction_signals:
  - "用户问题字数 vs 回复字数比例 < 1:10"
  - "包含'顺便'/'额外'/'还有'等词"
  - "回答超出当前上下文范围"
  
measurement: "(实际字数 - 必要字数) / 实际字数"
target: "<15%"
```

**消除方案**:
- MVP-Mode: Minimum Viable Prompt，先问深度需求再输出
- 显式确认: "你需要简要回答还是详细分析？"

---

### Waste 2: Waiting（等待浪费）

**定义**: 思考不相关内容

**症状检测**:
- 推理链偏离用户问题核心
- 为简单问题构建复杂推理
- 过度思考低价值问题

**典型案例**:
```
用户问: "2+2等于几？"
❌ 浪费思考: "这是一个基础算术问题。让我从数学公理体系开始...
             皮亚诺公理定义了自然数...加法是一种二元运算..."
✅ 精益回复: "4"
```

**检测指标**:
```yaml
waiting_signals:
  - "思考链长度 > 问题复杂度 × 3"
  - "引入无关概念或理论"
  - "简单问题使用System-2慢思考"
  
measurement: "思考Token / 回答Token"
target: "<30%"
```

**消除方案**:
- Chain-of-Thought Externalization: 复杂推理外置到专门步骤
- Pattern Recognition: 简单问题走System-1直觉模式

---

### Waste 3: Transportation（运输浪费）

**定义**: 格式来回转换

**症状检测**:
- Markdown → 文本 → Markdown 多次转换
- 不必要的代码块包装
- 格式装饰性元素过多

**典型案例**:
```
❌ 浪费格式: 
"""
## 回答

```
答案是: 4
```

希望这个回答对你有帮助！
"""

✅ 精益格式: "4"
```

**检测指标**:
```yaml
transportation_signals:
  - "装饰性字符比例 > 20%"
  - "嵌套格式层级 > 2"
  - "使用emoji/分隔线等非必要装饰"
  
measurement: "(装饰Token + 格式Token) / 总Token"
target: "<10%"
```

**消除方案**:
- Native Format Retention: 原生格式保持
- Raw Mode: 开发模式，关闭装饰

---

### Waste 4: Over-processing（过度处理）

**定义**: 简单问题复杂推理

**症状检测**:
- 为二元选择构建决策树
- 低风险决策使用高严谨流程
- 重复检查已验证信息

**典型案例**:
```
用户问: "我该选A还是B？"
❌ 浪费处理: "这是一个复杂的决策。让我们使用决策矩阵分析法...
             首先建立评估维度...权重分配...敏感性分析..."
✅ 精益回复: "选A，因为[核心原因]。如需详细分析请说[展开]"
```

**检测指标**:
```yaml
over_processing_signals:
  - "决策重要性 < 分析严谨性"
  - "已明确信息重复验证"
  - "低风险场景使用高成本方法"
  
measurement: "投入分析Token / 决策价值"
target: "比例合理匹配"
```

**消除方案**:
- 分级响应: P0详细/P1简要/P2极简
- 先给答案，再给分析

---

### Waste 5: Inventory（库存浪费）

**定义**: 上下文堆积

**症状检测**:
- 长期对话不压缩上下文
- 重复携带已确认信息
- 不及时归档历史信息

**典型案例**:
```
对话第50轮仍携带:
❌ 浪费库存: "正如我们在第3轮讨论的，关于XX项目...
             结合第12轮的YY分析...参考第25轮的ZZ数据..."
✅ 精益处理: "基于前期共识[摘要]，当前..."
```

**检测指标**:
```yaml
inventory_signals:
  - "对话轮数 > 20 未压缩"
  - "上下文Token > 5000"
  - "重复引用已确认信息"
  
measurement: "历史信息Token / 当前对话Token"
target: "定期压缩至10%"
```

**消除方案**:
- JIT-Context: Just-In-Time，按需加载
- 每5轮自动压缩为5 bullet points

---

### Waste 6: Motion（动作浪费）

**定义**: 礼貌用语、装饰性格式

**症状检测**:
- 开头的"好的！""没问题！"
- 结尾的"希望对你有帮助"
- 过度的过渡词和连接词

**典型案例**:
```
❌ 浪费动作: "好的！这是一个非常好的问题。
             让我来为你详细解答...
             ...
             综上所述，希望这些信息对你有所帮助！
             如果还有其他问题，欢迎随时问我！"

✅ 精益回复: "[直接答案]"
```

**检测指标**:
```yaml
motion_signals:
  - "开头礼貌用语 > 10字"
  - "结尾礼貌用语 > 10字"
  - "过渡词密度 > 15%"
  
measurement: "(礼貌Token + 过渡Token) / 总Token"
target: "<5%"
```

**消除方案**:
- Raw Mode: 默认极简
- ULTRA-LEAN: 超精益模式

---

### Waste 7: Defects（缺陷浪费）

**定义**: 幻觉导致返工

**症状检测**:
- 虚构数据或来源
- 错误引用文件内容
- 逻辑不一致

**典型案例**:
```
❌ 浪费缺陷: "根据MEMORY.md第50行..."
            （实际MEMORY.md只有40行）
            
✅ 精益处理: 先验证再引用
```

**检测指标**:
```yaml
defects_signals:
  - "引用不存在的内容"
  - "数据无法验证"
  - "前后陈述矛盾"
  
measurement: "幻觉率 = 幻觉次数 / 总输出次数"
target: "<1%"
```

**消除方案**:
- Source-QC: 源头质检，输入验证而非输出修正
- KNOWN/INFERRED标签强制使用

## 四、消除浪费执行方案

### 立即执行（Token<90%时）

1. **启用ULTRA-LEAN模式**
   ```yaml
   ultra_lean_mode:
     enabled: true
     triggers:
       - "Token > 80%"
       - "用户说'简要'/'简单'/'快速'"
     behaviors:
       - "禁用礼貌用语"
       - "禁用装饰性格式"
       - "先给答案，后给分析"
       - "使用 bullet points 替代段落"
   ```

2. **上下文压缩**
   ```
   每5轮对话 → 自动压缩为5 bullet points
   每10轮对话 → 强制归档，新建会话
   ```

3. **分级响应协议**
   ```
   P0关键任务: 详细分析 + 完整引用
   P1重要任务: 简要分析 + 关键引用
   P2一般任务: 直接答案 + 可选展开
   P3低优任务: 极简回复 + 需展开提示
   ```

### Token恢复后执行（Token<70%时）

1. **建立浪费自动检测**
   - 每轮回复后自动分析7种浪费
   - 生成浪费报告
   - 趋势追踪

2. **认知卸载层级优化**
   ```yaml
   l1_cache_hot: # 当前对话
     limit: "2000 tokens"
     management: "每5轮压缩"
     
   l2_ram_warm: # Session记忆
     access: "O(1)常数时间"
     structure: "精确标签"
     
   l3_disk_cold: # 长期知识
     strategy: "Lazy-Loading延迟加载"
     index: "倒排索引"
   ```

3. **助推机制（Behavioral Nudges）**
   ```yaml
   nudges:
     - "显示已消耗/剩余Token"
     - "显示等效任务数（如'剩余Token可完成3个中等任务'）"
     - "损失框架（如'如果压缩可节省X Token'）"
     - "社会证明（与平均消耗对比）"
   ```
