## 🎯 产品定位矩阵

### 按客户决策阶段划分

| 决策阶段 | 核心痛点 | 对应产品 | 服务目标 |
|----------|----------|----------|----------|
| **决策前** | 不了解自己的决策风格 | 产品1：体检 | 自我认知 |
| **决策中** | 重大选择缺乏科学依据 | 产品2：诊断 | 科学匹配 |
| **决策后** | 执行过程缺乏支持 | 产品3：陪跑 | 持续陪伴 |
| **长期** | 组织能力需要提升 | 产品4：顾问 | 系统升级 |

### 按客户成熟度划分

| 客户类型 | 特征 | 主力产品 | 辅助产品 |
|----------|------|----------|----------|
| **新手创业者** | 首次创业，缺乏经验 | 产品1 | 产品2（合伙人招募时） |
| **成长期创始人** | 已有团队，面临扩张 | 产品2+3 | 产品1（团队扩展时） |
| **成熟期CEO** | 企业稳定，寻求突破 | 产品4 | 产品3（重大决策时） |
| **连续创业者** | 多次创业，资源充足 | 产品2+4 | 产品3（关键项目） |

## 🔄 产品间协作机制

### 1. 数据流动

```
产品1（体检）
    ↓ 输出：决策基因图谱
    ↓ 包含：6维度评分、决策类型、盲区识别
    ↓
产品2（诊断）
    ↓ 输入：双方决策基因 → PFI计算
    ↓ 输出：匹配度评分、风险评估、契约建议
    ↓
产品3（陪跑）
    ↓ 输入：历史决策模式 → 决策DNA
    ↓ 输出：决策日志、质量评分、改进建议
    ↓
产品4（顾问）
    ↓ 输入：全年决策数据 → 仪表盘
    ↓ 输出：组织能力评估、系统优化建议
```

### 2. 服务衔接

| 触发条件 | 上游产品 | 下游产品 | 衔接动作 |
|----------|----------|----------|----------|
| 体检完成+高决策风险盲区 | 产品1 | 产品2 | 自动推荐合伙人诊断 |
| 诊断完成+低匹配度预警 | 产品2 | 产品3 | 推荐陪跑服务修复关系 |
| 陪跑3个月+持续高压力 | 产品3 | 产品4 | 推荐年度顾问深度支持 |
| 顾问服务+新合伙人加入 | 产品4 | 产品2 | 启动新合伙人诊断 |

### 3. 客户旅程映射

```
【认知阶段】
内容营销 → 免费体检 → 基础报告
              ↓
【评估阶段】
深度解读 → 合伙人招募需求识别
              ↓
【购买阶段】
选择：诊断服务 / 陪跑服务 / 年度顾问
              ↓
【使用阶段】
持续服务 + 数据沉淀 + 能力提升
              ↓
【续约阶段】
服务升级 / 产品切换 / 增值服务
              ↓
【推荐阶段】
成功案例 → 口碑传播 → 新客获取
```

## 📊 统一数据模型

### 核心实体

```yaml
# 客户实体
Customer:
  id: 唯一标识
  name: 姓名
  company: 公司
  stage: 创业阶段 [种子/A轮/B轮/C轮/上市]
  industry: 行业 [AI/芯片/生物医药/新能源/其他]
  join_date: 首次接触日期
  lifetime_value: 生命周期价值
  
# 决策档案
DecisionProfile:
  customer_id: 关联客户
  dna_scores:  # 产品1输出
    information: 0-100
    risk: 0-100
    time: 0-100
    interpersonal: 0-100
    stress: 0-100
    values: 0-100
  decision_type: [数据型冒险家/直觉型稳健派/平衡型...]
  blind_spots: [盲区列表]
  updated_at: 更新时间
  
# 合伙关系
Partnership:
  founder_id: 创始人ID
  partner_id: 合伙人ID
  pfi_score: 匹配度评分
  assessment_date: 评估日期
  risk_level: [高/中/低]
  contract_type: [均分/控股/项目制...]
  status: [评估中/已合伙/已分手]
  
# 决策记录
DecisionRecord:
  id: 唯一标识
  customer_id: 关联客户
  decision_type: [技术/商业/团队/融资/战略]
  complexity: [简单/中等/复杂]
  urgency: [低/中/高/紧急]
  five_totems:  # 五路图腾校准
    LIU: 1-5
    SIMON: 1-5
    GUANYIN: 1-5
    CONFUCIUS: 1-5
    HUINENG: 1-5
  outcome: [成功/部分成功/失败/待定]
  quality_score: 1-5
  created_at: 决策日期
```

## 🎛️ 统一服务标准

### 响应时间标准

| 服务级别 | 产品1 | 产品2 | 产品3 | 产品4 |
|----------|-------|-------|-------|-------|
| **紧急响应** | N/A | N/A | 5分钟 | 即时 |
| **标准响应** | 即时 | 2小时 | 2小时 | 4小时 |
| **报告交付** | 24小时 | 72小时 | 月度 | 季度 |
| **预约安排** | 即时 | 3天内 | 1周内 | 协商 |

### 质量标准

| 维度 | 标准定义 | 测量方法 |
|------|----------|----------|
| **准确性** | 数据准确，引用有据 | 内部审核+客户确认 |
| **完整性** | 覆盖承诺全部内容 | 交付清单核对 |
| **及时性** | 按时交付，不延误 | 时间追踪 |
| **满意度** | NPS > 50 | 客户调研 |
| **转化率** | 达到目标值 | 数据分析 |
