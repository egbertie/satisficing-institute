### 2.3 典型方案详解

#### 方案A：愿景重构工作坊 (Vision Reconstruction Workshop)

**适用场景：** 战略方向严重分歧，但双方仍有合作意愿

**流程设计：**
```
Day 1 上午：分离式回顾
├── 各自书写：创业初心回顾 (2小时)
├── 独立反思：个人愿景演变 (1小时)
└── 情绪释放：未被满足的需求表达 (1小时)

Day 1 下午：共同探索
├── 价值锚定：不可妥协的核心价值 (2小时)
├── 场景想象：5年后的理想画面 (1.5小时)
└── 差异识别：明确分歧点与共识区 (1.5小时)

Day 2 上午：融合共创
├── 愿景整合：寻找共同愿景表述 (3小时)
├── 路径设计：实现愿景的多条路径 (2小时)
└── 决策机制：未来战略决策规则 (2小时)

Day 2 下午：承诺固化
├── 行动承诺：各自承担的职责 (2小时)
├── 检查机制：定期回顾机制 (1小时)
└── 仪式庆祝：重新确认合伙关系 (1小时)
```

**配套工具包：**
- 《愿景探索手册》
- 《战略分歧决策框架》
- 《个人需求表达模板》
- 《共同愿景宣言模板》

#### 方案B：信任重建训练 (Trust Rebuilding Program)

**适用场景：** 因某事件导致信任受损，需要系统性修复

**四周渐进式训练：**

| 周次 | 主题 | 核心活动 | 目标 |
|-----|------|---------|------|
| **Week 1** | 创伤梳理 | 事件回顾、情绪表达、责任厘清 | 释放情绪、厘清事实 |
| **Week 2** | 脆弱展示 | 弱点分享、恐惧表达、需求坦诚 | 重建情感连接 |
| **Week 3** | 承诺实践 | 小承诺兑现、透明度提升、一致性强化 | 积累信任资本 |
| **Week 4** | 巩固深化 | 大承诺设计、监督机制建立、未来约定 | 制度化信任 |

**关键原则：**
1. **过错方需真诚道歉**（具体行为+影响认知+改变承诺）
2. **受害方需给予机会**（表达伤害+期望明确+观察窗口）
3. **建立可验证的信任行为**（具体、可观察、可评估）

### 3.2 模块化训练营

**模块1：非暴力沟通 (NVC) 合伙人版**

```
┌─────────────────────────────────────────────────────────┐
│  四要素框架                                              │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  观察(Observation)                                       │
│  └── "本周三次会议你都迟到了10分钟以上"                  │
│      而非 "你总是迟到，不负责任"                         │
│                                                          │
│  感受(Feeling)                                           │
│  └── "这让我感到焦虑和不被尊重"                          │
│      而非 "你让我很生气"                                 │
│                                                          │
│  需要(Need)                                              │
│  └── "因为我需要团队的协作效率和互相尊重"                │
│      而非 "你应该守时"                                   │
│                                                          │
│  请求(Request)                                           │
│  └── "你能否确保下周会议准时出席，或提前告知？"          │
│      而非 "你必须改正"                                   │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**实战演练场景：**
1. 股权分配谈判中的NVC应用
2. 拒绝合伙人提议时的表达方式
3. 表达不满而不伤害关系

**模块2：冲突转化技术 (Conflict Transformation)**

| 技术名称 | 核心方法 | 适用场景 |
|---------|---------|---------|
| **利益冰山** | 从立场深入挖掘底层利益 | 任何谈判僵局 |
| **第三方视角** | 引入"墙上的苍蝇"视角 | 双方情绪激烈时 |
| **是的，而且...** | 即兴喜剧技巧，建设性叠加 | 创意分歧 |
| **时间机器** | 跳到5年后回望现在 | 眼前利益争执 |
| **角色互换** | 站在对方立场辩护 | 理解障碍 |

**模块3：反馈与赞美艺术**

```python
# SBI-I 反馈模型 (Situation-Behavior-Impact-Intention)

def give_feedback():
    situation = "在今天的投资人会议上"  # 具体情境
    behavior = "你打断了我三次"          # 可观察行为
    impact = "这让我感到不被尊重，也影响了我们的专业形象"  # 对我/我们的影响
    intention = "我希望我们能有轮流发言的默契"  # 期望/意图
    
    return f"{situation}，当{behavior}时，{impact}。{intention}，你觉得呢？"
```

### 4.2 调解流程标准化

```
Phase 1:  intake (摄入)
├── 冲突背景收集
├── 双方意愿确认
├── 调解形式选择(单独/联合/混合)
└── 调解协议签署

Phase 2:  preparation (准备)
├── 双方单独访谈
├── 利益与需求分析
├── 情绪状态评估
└── 调解策略制定

Phase 3:  session (调解)
├── 开场与规则设定
├── 故事讲述(双方)
├── 利益澄清与确认
├── 方案头脑风暴
├── 方案评估与选择
└── 行动计划制定

Phase 4:  follow-up (跟进)
├── 协议执行监督
├── 定期回访(1周/1月/3月)
├── 协议调整(如需)
└── 案例归档与学习
```
