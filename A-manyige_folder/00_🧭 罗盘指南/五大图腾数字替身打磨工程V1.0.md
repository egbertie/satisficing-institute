## 二、数字替身打磨流程（标准化）

### 2.1 四阶段打磨法

```
第一阶段：素材搜集（2-4周/替身）
├── 核心著作阅读与缩写
├── 学术论文/文献搜集
├── 交叉验证资料获取
└── 素材质量评级与筛选

第二阶段：核心提炼（1-2周/替身）
├── 核心思想体系梳理
├── 关键概念定义
├── 方法论框架构建
├── 经典语录/案例萃取
└── 与用户（Egbertie）的契合点挖掘

第三阶段：档案撰写（1周/替身）
├── 数字替身档案（8000-10000字）
├── 快速参考卡片（1页A4）
├── 决策场景应用指南
└── 互动话术库（50+场景）

第四阶段：验证迭代（持续）
├── 沙盘推演实测
├── 用户反馈收集
├── 档案修订更新
└── 版本迭代（V1.0 → V2.0 → ...）
```

### 2.2 素材质量标准

**每尊替身必须满足：**
- [ ] 核心著作阅读 ≥ 3本（或等效学术论文）
- [ ] 总素材字数 ≥ 2万字
- [ ] 关键观点有出处标注
- [ ] 至少有3个可引用的经典案例
- [ ] 与用户决策风格的契合点 ≥ 5个

## 五、质量控制

### 5.1 素材质量评级

| 评级 | 标准 | 使用方式 |
|:---:|:---|:---|
| ⭐⭐⭐⭐⭐ | 一手资料、学术权威、与替身高度相关 | 核心素材 |
| ⭐⭐⭐⭐ | 可靠来源、重要参考、有启发性 | 主要素材 |
| ⭐⭐⭐ | 一般来源、辅助参考、特定场景 | 补充素材 |
| ⭐⭐ | 待验证、观点有趣、需交叉确认 | 待审核 |
| ⭐ | 不可靠、偏离主题 | 弃用 |

### 5.2 交叉验证要求

**每个核心观点必须有：**
- 至少1个一手资料出处
- 或至少2个二手资料交叉印证
- 或与用户（Egbertie）的契合点明确

### 5.3 档案质量标准

**数字替身档案必须包含：**
1. **身份定义**（500字）：姓名、时代、身份、核心标签
2. **思想体系**（2000字）：核心思想、方法论、理论框架
3. **经典语录**（20条）：有出处的代表性言论
4. **经典案例**（5个）：具体情境下的决策/行为
5. **与用户契合点**（5个）：与Egbertie决策风格的共鸣
6. **应用场景**（10个）：在合伙人决策中的具体应用
7. **互动话术**（50+场景）：不同情境下的回应方式
