## 三、评估执行流程

```mermaid
flowchart LR
    subgraph 问卷评估
        A1[问卷发放] --> A2[填写指导]

    end
    
    subgraph 深度访谈
        B1[访谈准备] --> B2[结构化访谈]

    end
    
    subgraph 背景调查
        C1[资料审核] --> C2[交叉验证]
        C2 --> C3[信息补充]
    end

    B4 --> D
    C3 --> D
```

### 3.2 7维度评分流程

```mermaid
flowchart TB
    Input[原始数据] --> Values[维度1: 价值观契合度<br/>权重20%]
    Input --> Capability[维度2: 能力互补性<br/>权重20%]
    Input --> Commitment[维度3: 承诺可信度<br/>权重15%]
    Input --> Communication[维度4: 沟通效率<br/>权重15%]
    Input --> Interest[维度5: 利益一致性<br/>权重15%]
    Input --> Exit[维度6: 退出可接受性<br/>权重10%]
    Input --> Growth[维度7: 成长匹配度<br/>权重5%]
    
    Values --> WeightedScore[加权计算]
    Capability --> WeightedScore
    Commitment --> WeightedScore
    Communication --> WeightedScore
    Interest --> WeightedScore
    Exit --> WeightedScore
    Growth --> WeightedScore
    
    WeightedScore --> Overall[综合得分]
    Overall --> RiskLevel[风险等级判定]
    
    RiskLevel --> Excellent[>=8.0 EXCELLENT]
    RiskLevel --> Good[7.0-7.9 LOW]
    RiskLevel --> Medium[6.0-6.9 MEDIUM]
    RiskLevel --> High[5.5-5.9 HIGH]
    RiskLevel --> Critical[<5.5 CRITICAL]
    
    style WeightedScore fill:#5B8DB8,stroke:#3D5A73,stroke-width:2px,color:#fff
    style RiskLevel fill:#5B8DB8,stroke:#3D5A73,stroke-width:2px,color:#fff
    style Excellent fill:#7BAE7F,stroke:#5A8A5E,stroke-width:2px,color:#fff
    style Critical fill:#ffccbc,stroke:#bf360c,stroke-width:2px,color:#000
```

## 六、质量管控流程

### 6.1 报告审核流程

```mermaid
flowchart TB
    Draft[报告初稿] --> SelfReview[自审]
    SelfReview --> PeerReview[同行审核]
    PeerReview --> ExpertReview[专家审核]
    
    ExpertReview --> Check{审核通过?}
    Check -->|否| Revise[修改完善]
    Revise --> SelfReview
    
    Check -->|是| Final[定稿]
    Final --> Delivery[交付]
    
    style ExpertReview fill:#5B8DB8,stroke:#3D5A73,stroke-width:2px,color:#fff
    style Check fill:#5B8DB8,stroke:#3D5A73,stroke-width:2px,color:#fff
    style Delivery fill:#7BAE7F,stroke:#5A8A5E,stroke-width:2px,color:#fff
```

### 6.2 质量检查清单

| 检查项 | 检查内容 | 责任人 |
|--------|----------|--------|
| 数据准确性 | 评分计算是否正确 | 分析师 |
| 逻辑一致性 | 分析与结论是否一致 | 同行 |
| 专业性 | 术语使用、分析深度 | 专家 |
| 完整性 | 所有承诺内容是否交付 | PM |
| 时效性 | 是否在72小时内交付 | PM |

## 七、异常处理流程

### 7.1 延期处理流程

```mermaid
flowchart TB
    Risk[发现延期风险] --> Notify[通知客户]
    Notify --> Reason[说明原因]
    Reason --> Plan[提供新时间表]
    
    Plan --> Accept{客户接受?}
    Accept -->|是| Continue[继续服务]
    Accept -->|否| Refund[退款协商]
    
    Refund --> Solution[解决方案]
    Solution --> End([结束])
    Continue --> End
    
    style Notify fill:#C4A35A,stroke:#8B7355,stroke-width:2px,color:#fff
    style Accept fill:#5B8DB8,stroke:#3D5A73,stroke-width:2px,color:#fff
```

### 7.2 客户投诉处理流程

```mermaid
flowchart TB
    Complaint[接收投诉] --> Record[记录详情]
    Record --> Analyze[分析原因]
    
    Analyze --> Level{投诉级别}
    Level -->|一般| HandleA[24小时内响应]
    Level -->|严重| HandleB[4小时内响应]
    Level -->|重大| HandleC[1小时内响应]
    
    HandleA --> Resolve[解决问题]
    HandleB --> Resolve
    HandleC --> Resolve
    
    Resolve --> Feedback[客户反馈]
    Feedback --> Satisfied{满意?}
    
    Satisfied -->|是| Close[结案]
    Satisfied -->|否| Escalate[升级处理]
    
    Escalate --> Resolve
    Close --> Summary[案例总结]
    
    style Complaint fill:#ffccbc,stroke:#bf360c,stroke-width:2px,color:#000
    style Resolve fill:#5B8DB8,stroke:#3D5A73,stroke-width:2px,color:#fff
    style Close fill:#7BAE7F,stroke:#5A8A5E,stroke-width:2px,color:#fff
```
