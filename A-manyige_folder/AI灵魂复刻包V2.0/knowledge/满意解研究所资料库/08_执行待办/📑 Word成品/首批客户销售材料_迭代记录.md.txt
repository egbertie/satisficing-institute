## 🔄 迭代机制说明

**核心原则**：
> 不要等真人信息齐全，先用AI模拟，持续迭代，待真实信息补充后再修正。

**四阶段循环**：
```
模拟 → 测试 → 反馈 → 修正 → 模拟...
```

| 阶段 | 时间 | 负责人 | 输出 |
|------|------|--------|------|
| **模拟** | 持续 | AI+员工体系 | V0.x版本 |
| **测试** | 获客后 | 指挥官 | 客户反馈记录 |
| **反馈** | 每周 | PEO+EEO | 反馈分析报告 |
| **修正** | 持续 | 全体员工 | V(x+1).0版本 |

## 🔄 修正触发条件

**自动触发修正**：
1. 指挥官提供新的真实信息
2. 首批客户接触后收集到新反馈
3. 每周复盘会议识别优化点
4. 五路图腾会诊提出修订建议

**修正流程**：
```
触发条件 → GAME⭐记录 → 相关员工执行 → PEO审核 → 指挥官确认 → 版本更新
```
