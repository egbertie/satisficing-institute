# 管理罗盘实现方案

**目标**：实时掌握33角色成果、进度、任务完成度  
**核心**：数据自动化+可视化仪表盘+预警机制  
**查看方式**：Notion实时仪表盘 + 飞书定时报告

---

## 一、管理罗盘架构

### 1.1 数据看板（4个维度）

```
管理罗盘
├── 📊 角色状态看板（33角色全景）
├── 📋 任务作战室（任务池+进度）
├── ⏰ 时间线视图（甘特图+里程碑）
└── 🚨 风险预警台（延迟+阻塞）
```

### 1.2 数据源（谁提供数据）

| 层级 | 数据提供者 | 更新频率 | 方式 |
|------|-----------|---------|------|
| 决策层 | 满意妞（主控） | 实时 | 自动汇总 |
| 文化层 | 五路图腾 | 每周 | 周报 |
| 执行层 | PEO/EEO/游戏官 | 每日 | 日站会 |
| 员工层 | 各角色负责人 | 每日 | 表单填写 |
| 支撑层 | 系统管理员 | 实时 | 自动抓取 |

| 角色名称 | 标题 | 如：PEO-项目进化官 |

| 下周计划 | 文本 | 3个计划 |

## 四、数据更新机制（谁来填）

### 4.1 自动更新（无需人工）
- **系统监控**：服务器状态、定时任务状态 → 系统管理员自动上报
- **Git提交**：代码/文档提交 → 自动关联任务进度
- **飞书消息**：关键频道消息 → AI自动解析进度

### 4.2 半自动（AI辅助）
- **日站会**：AI询问各角色"今日进展/明日计划/阻塞问题" → 自动填充表格
- **周报生成**：基于每日数据 → AI自动生成周报 → 角色确认

### 4.3 人工确认（关键节点）
- **里程碑完成**：负责人标记完成 → PEO审核 → 更新状态
- **风险升级**：黄色风险 → 红色阻塞 → 指挥官决策
