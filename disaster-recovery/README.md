## 🚀 快速开始

### 1. 阅读策略文档
了解RTO/RPO目标、风险矩阵、分层恢复策略。

```bash
# 阅读主策略文档
cat 01-灾备策略文档.md
```

### 2. 配置备份脚本

```bash
cd 02-自动化备份脚本/

# 1. 编辑环境变量模板
vim backup-configs.sh  # 设置你的GITHUB_TOKEN等

# 2. 测试单个备份
./backup-github.sh --dry-run

# 3. 执行完整备份
./backup-master.sh --daily
```

### 3. 设置定时任务

```bash
# 编辑crontab
crontab -e

# 添加以下条目 (每日凌晨2点执行)
0 2 * * * cd /disaster-recovery/02-自动化备份脚本 && ./backup-master.sh --daily >> logs/cron.log 2>&1

# 每周深度备份 (周日凌晨3点)
0 3 * * 0 cd /disaster-recovery/02-自动化备份脚本 && ./backup-master.sh --weekly >> logs/weekly.log 2>&1
```

### 4. 定期演练

按 `05-定期演练计划.md` 执行月度检查和季度演练。
