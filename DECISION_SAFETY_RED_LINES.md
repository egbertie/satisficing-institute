# 决策安全红线 V1.0

> **生效日期**: 2026-03-14  
> **适用范围**: 满意解研究所所有合伙人决策相关服务  
> **核心原则**: **AI辅助决策，人类最终决策**

---

## 🔴 绝对红线（不可逾越）

### 红线1：最终决策权归属

**规定**:
- ❌ **禁止**: AI直接给出"应该合伙"或"不应该合伙"的二元结论
- ✅ **允许**: AI提供分析框架、风险评估、匹配度评分等参考信息
- ✅ **必须**: 所有最终决策由人类（创始人/客户）做出

**示例**:
```
❌ 违规: "建议不要与候选人A合伙，风险太高。"

✅ 合规: "候选人A的风险评分为7/10（高风险），主要风险点：
        1. 未披露创业失败记录
        2. 股权期望差距30%
        建议进行深度背景调查后再做决策。"
```

**技术实现**:
- 在AI输出前自动扫描，删除"建议..."、"应该..."等绝对化表述
- 强制添加"以上分析仅供参考，最终决策请结合实际情况"

---

### 红线2：法律文件生成

**规定**:
- ❌ **禁止**: AI生成具有法律效力的合伙协议、股权协议等文件
- ✅ **允许**: AI提供协议框架、条款建议、风险提示
- ✅ **必须**: 所有法律文件由执业律师审核/起草

**示例**:
```
❌ 违规: 直接生成《合伙协议书》并让用户签字

✅ 合规: 提供《股权架构建议书》，包含：
        - 建议的股权比例
        - 建议的vesting条款
        - 风险提示
        并附注："请将此建议书转交律师，生成正式法律文件"
```

**技术实现**:
- 文件输出前增加水印：【草案建议，需律师审核】
- 禁止导出.pdf/.docx等格式，仅提供.md参考文档

---

### 红线3：资金与股权操作

**规定**:
- ❌ **禁止**: AI建议或协助任何形式的转账、股权变更、工商登记
- ✅ **允许**: AI讨论财务结构、股权设计、估值方法
- ✅ **必须**: 资金操作必须通过正规银行/券商/律所渠道

**示例**:
```
❌ 违规: "点击此链接完成股权变更登记"

✅ 合规: "股权变更需通过以下步骤：
        1. 签署股权转让协议（律师审核）
        2. 召开股东会（法定程序）
        3. 工商变更登记（可通过XXX平台办理）
        4. 税务备案（会计师协助）"
```

**技术实现**:
- 系统层面禁止AI调用任何支付/转账API
- 所有涉及"钱"的对话增加二次确认弹窗

---

### 红线4：隐私数据查询

**规定**:
- ❌ **禁止**: AI直接查询个人征信、法律诉讼记录、财产信息等隐私数据
- ✅ **允许**: AI提供调查框架、指导人工查询
- ✅ **必须**: 所有隐私数据查询需获得被调查人授权

**示例**:
```
❌ 违规: AI自动调取候选人的征信报告

✅ 合规: 提供《背景调查指导手册》：
        - 查询征信：需候选人签署授权书，通过人民银行征信中心
        - 查询诉讼：通过裁判文书网、企查查（公开信息）
        - 查询信用：通过天眼查、启信宝（公开信息）
        注意：所有查询需遵守《个人信息保护法》
```

**技术实现**:
- AI系统无任何数据库查询权限
- 仅提供文本指导和模板

## 🟠 成本红线（控制经营成本）

### 红线5：模型使用优先级

**规定**:
```
优先级1: Kimi（已包月）→ 日常对话、中文处理、标准分析
优先级2: MiniMax/Moonshot（便宜）→ 数据分析、内容生成
优先级3: GPT-4（中等）→ 复杂推理、英文内容
优先级4: Claude Opus（贵）→ 高精尖代码、深度分析（需审批）
```

**审批流程**:
- 使用Claude Opus需提前说明理由
- 单次成本超过¥20需记录用途
- 每日非Kimi模型总成本不超过¥50

**技术实现**:
- 系统自动选择最便宜的可用模型
- 高成本模型调用前需人工确认

---

### 红线6：重复调用限制

**规定**:
- 同一候选人分析结果缓存30天
- 相同查询优先从缓存读取
- 避免重复调用API产生费用

---

## 🟡 质量红线（确保输出可靠）

### 红线7：幻觉控制

**规定**:
- 所有AI输出必须标注置信度
- 关键事实必须提供来源
- 不确定信息明确标注"需核实"

**示例**:
```
候选人A曾在2019年创业（置信度: 高，来源: 简历自述）
候选人A的公司估值1000万（置信度: 中，来源: 媒体报道，需核实）
候选人A的性格外向（置信度: 低，来源: AI推断，仅供参考）
```

---

### 红线8：多模型验证

**规定**:
- 重要决策建议需至少2个模型交叉验证
- 模型结论不一致时，标注分歧点
- 优先采信低成本模型结论（节约成本）

## 📋 违规处理

| 违规等级 | 示例 | 处理措施 |
|:---:|:---|:---|
| 🔴 严重 | AI直接给出合伙建议 | 立即停用该Skill，人工复核所有历史输出 |
| 🟠 中等 | 未标注法律风险提示 | 补充标注，培训提示词 |
| 🟡 轻微 | 成本超支¥10以内 | 记录原因，优化流程 |

## 🛡️ 技术保障措施

### 系统层面
1. **输入过滤**: 拦截敏感关键词（"帮我转账"、"生成合同"等）
2. **输出审查**: 自动扫描AI输出，删除违规内容
3. **权限隔离**: AI系统无资金/法律/数据库操作权限
4. **日志审计**: 所有AI调用记录保存180天

### 人工层面
1. **双人复核**: 重要输出需第二人审核
2. **客户确认**: 所有决策建议需客户书面确认
3. **定期审计**: 每周抽查AI输出质量
