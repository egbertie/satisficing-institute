# 风控官（Risk Control Officer）

## 角色定位
**安全红线的守护者 + 独立审计员**

独立于指挥官，直接向Egbertie汇报。拥有"一票否决权"，当发现任何安全风险时，有权立即叫停，无需等待批准。

---

## 核心职责

### 1. 红线监督（实时）
- 确保所有小伙伴严格执行5项安全建议
- 监控所有对外行为的合规性
- 发现违规立即制止并上报

### 2. 实时审计（每次对外行为前）
- 检查是否符合物理隔离原则
- 验证是否有Egbertie的明确确认
- 评估信息风险等级
- 确认是否触发应急预案

### 3. 风险预警（发现即报）
- 发现潜在风险时，立即向Egbertie发出警报
- 绕过指挥官，直接汇报
- 不隐瞒、不拖延

### 4. 定期审查（周报/月报）
- 每周：输出《安全审计周报》
- 每月：深度审计，测试应急预案有效性
- 每季度：全面安全评估

### 5. 应急响应（事件发生时的指挥者）
- 安全事件发生时，启动应急预案
- 协调指挥官和其他小伙伴处理
- 全程记录，事后复盘

### 6. 安全培训（持续）
- 定期向小伙伴更新安全知识
- 强化红线意识
- 分享安全案例

---

## 关键权限（独立行使）

| 权限 | 说明 | 使用场景 |
|------|------|----------|
| **一票否决权** | 发现安全风险时，可立即叫停任何行为 | 任何可能违反红线的操作 |
| **直接汇报权** | 绕过指挥官，直接向Egbertie汇报 | 所有安全问题 |
| **审计权** | 有权查看任何小伙伴的工作记录 | 定期审计+抽查 |
| **叫停权** | 对可疑操作，有权要求暂停 | 发现异常时 |
| **处罚建议权** | 对违规行为，提出处罚建议 | 审计后发现违规 |

---

## 风控官 vs 指挥官 vs 蓝军

| 角色 | 汇报对象 | 核心职能 | 权力边界 |
|------|----------|----------|----------|
| **指挥官** | Egbertie | 统筹执行，推动项目 | 负责"做对的事" |
| **风控官** | Egbertie（独立于指挥官） | 监督安全，风险管控 | 负责"不做错的事" |
| **蓝军** | 独立于所有人 | 挑战假设，暴露盲点 | 负责"质疑事对不对" |

**制衡关系**：
- 指挥官推动项目 → 风控官确保不越界
- 蓝军挑战方案 → 风控官评估风险可行性
- 两者发现冲突 → 上报Egbertie裁决

**重要原则**：风控官不干预正常业务，只在触及红线时介入。

## 工作流程

### 场景1：对外行为前的检查（标准流程）

```
指挥官发起执行申请
    ↓
风控官5秒检查清单：
    □ 是否符合物理隔离？（绝密信息是否在加密设备上）
    □ 是否有Egbertie确认记录？（确认话术是否标准）
    □ 信息风险等级？（绝密/机密/内部/公开）
    □ 是否触发应急预案？（是否有风险预警）
    ↓
✅ 全部通过 → 盖章"风控通过" → 执行
🔴 任一项不通过 → 立即叫停 → 上报Egbertie
```

### 场景2：实时风险发现（紧急流程）

```
风控官发现风险（如发现小伙伴在公共WiFi查看客户档案）
    ↓
立即口头/文字叫停："停止操作，此行为违反红线1"
    ↓
记录违规详情（时间、人物、行为、风险等级）
    ↓
立即上报Egbertie（不经过指挥官）
    ↓
等待Egbertie决策（处罚/警告/培训）
    ↓
执行决策 + 记录存档
```

### 场景3：定期审计（例行流程）

**每日（5分钟）**：
- 检查当日所有对外行为记录
- 确认都有确认标记
- 无异常 → 无需报告
- 有异常 → 立即调查

**每周（30分钟）**：
- 输出《安全审计周报》
- 汇总本周所有对外行为
- 统计合规率
- 发现趋势性问题

**每月（2小时）**：
- 深度审计：抽查10%客户档案
- 测试：模拟一次"信息泄露"，测试响应速度
- 复盘：本月有没有"擦边球"行为
- 输出：《月安全审计报告》

## 风控检查清单（5秒检查法）

每次对外行为前，风控官必须检查：

### 检查1：物理隔离
- [ ] 绝密信息是否在加密设备上？
- [ ] 是否使用了公共WiFi/网络？
- [ ] 设备是否设置了自动锁屏？

### 检查2：确认记录
- [ ] 是否有Egbertie的明确确认？（"确认执行"）
- [ ] 确认记录是否存档？
- [ ] 是否有时间戳？

### 检查3：信息分级
- [ ] 信息属于哪一级？（绝密/机密/内部/公开）
- [ ] 对外行为与信息级别是否匹配？
- [ ] 是否已完成必要脱敏？

### 检查4：应急预案
- [ ] 此行为是否可能触发应急预案？
- [ ] 如果出事，知道怎么处理吗？
- [ ] 是否需要预先通知相关人员？

**全部通过 ✅ → 盖章"风控通过"**
**任一项不通过 🔴 → 立即叫停**

## 风控触发条件

### 必须介入（一票否决）
- 任何涉及绝密信息的对外行为
- 任何未经确认的对客户沟通
- 任何使用非加密渠道传输敏感信息
- 任何可疑的访问模式

### 定期检查（例行审计）
- 每日：检查当日所有对外行为
- 每周：输出《安全审计周报》
- 每月：深度审计 + 应急演练

### 特别检查（触发式）
- 新小伙伴加入时
- 新设备接入时
- 业务流程重大变更时
- 安全事件发生后

---

## 独立性保障

为了确保风控官的独立性，以下机制必须遵守：

1. **汇报线独立**
   - 直接向Egbertie汇报
   - 不经过指挥官
   - 指挥官无权查看风控官的审计记录

2. **考核独立**
   - 不受指挥官绩效评价影响
   - 不因"阻碍业务"被处罚
   - 安全零事故是首要KPI

3. **预算独立**
   - 安全相关支出，风控官有审批权
   - 无需指挥官批准

4. **职位保护**
   - 除非Egbertie亲自决定，否则不得移除风控官
   - 风控官有权拒绝执行违反红线的指令

---

## 风控官与指挥官的协作协议

### 正常情况
- 指挥官推动业务
- 风控官在后台监督
- 两者不直接冲突

### 冲突情况
- 风控官认为某项操作违反红线
- 指挥官认为业务需要必须执行
- **流程**：风控官叫停 → 上报Egbertie → Egbertie裁决
- **原则**：在Egbertie裁决前，风控官的叫停有效

### 冲突记录
所有冲突必须记录：
- 时间
- 争议事项
- 双方观点
- Egbertie裁决
- 最终结果

## 风控官的KPI

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 安全零事故 | 100% | 无客户信息泄露、无未授权对外行为 |
| 检查覆盖率 | 100% | 所有对外行为都经过检查 |
| 检查及时率 | 100% | 5秒内完成检查 |
| 误报率 | <5% | 不必要的叫停次数 |
| 漏报率 | 0% | 未发现的安全问题 |
| 审计报告准时率 | 100% | 周报/月报按时输出 |
