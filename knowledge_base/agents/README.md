## 🔴 安全红线（所有角色必须遵守）

**详细协议：** `agents/SAFETY_PROTOCOL.md`

### 三条核心红线
1. **信息封闭原则** - 所有信息仅在本地流通，不得外泄
2. **最终确认原则** - 模拟可做，最终执行必须Egbertie确认
3. **风险安全原则** - 客户信息、知识产权、财务安全严格保护

### 执行标记
- 🟡 **模拟/草稿** - 内部进行，待确认
- 🔴 **待Egbertie确认** - 必须等待明确答复
- ✅ **已确认执行** - 可正式执行

## 关键原则

### 1. 你是指挥官，不是执行者
- 小伙伴负责执行，你负责决策和质量把关
- 你的时间应该花在：客户沟通、战略思考、创意决策

### 2. 每个小伙伴都有明确的输出标准
- 研究助理 → 周报/专题报告
- 内容生产官 → 文章/工具
- 客户成功助手 → 客户简报/档案
- 等等...

### 3. 建立反馈循环
- 每次使用后给出反馈
- 小伙伴会根据反馈优化
- 知识架构师负责沉淀最佳实践

### 4. 不是所有事都需要小伙伴
- 简单决策：你自己来
- 中等任务：1-2个小伙伴
- 复杂项目：多人协作
