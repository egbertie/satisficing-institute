### 2.3 特殊格子说明

#### IPO评审（终点格）

进入IPO评审需要满足以下条件：
- ✅ 拥有完整的3人团队（含创始人）
- ✅ 团队能力覆盖：技术+商业+产品/运营
- ✅ 团队信任度 ≥ 6点
- ✅ 剩余资源 ≥ 3个任意币

**评审流程**：
1. 投骰子决定评审结果（1-6点）
2. 根据团队完整性获得加成：
   - 能力互补：+1
   - 信任度≥8：+1
   - 有COO角色：+1
3. 结果≥4点：成功IPO，游戏胜利
4. 结果<4点：回退到第25格，重新准备

### 8.3 核心规则详解

#### 招募规则
1. **招募条件**：
   - 玩家必须在"面试"类格子或抽到候选人
   - 团队上限：1名创始人 + 2名合伙人（共3人）
   - 支付对应资源成本

2. **招募流程**：
   ```
   抽取/获得候选人卡 → 选择评估方式 → 做出决策 → 支付成本 → 加入团队
   ```

3. **团队组合限制**：
   - 不能招募与创始人同类型的合伙人（避免能力冗余）
   - IPO要求：团队必须覆盖技术+商业+产品/运营能力

#### 评估规则
1. **评估选项**：
   - **直觉判断**：免费，准确率60-75%
   - **满意解评估**：消耗时间币，揭示特定信息
   - **深度调查**：消耗金钱+时间，高准确率

2. **决策时间限制**：
   - 普通招募：无时间限制
   - "限时窗口"格子：60秒倒计时
   - 超时则视为放弃

#### 事件处理优先级
```
1. 意外事件（强制立即执行）
2. 危机事件（必须应对）
3. 机会事件（可选择）
4. 格子基础效果
```

### 8.4 胜利条件

#### 基础胜利
- 首位到达IPO评审（第30格）
- 满足IPO条件后通过评审

#### IPO评审流程
```
┌─────────────────────────────────────────────────────┐
│                 IPO评审流程                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  前置条件检查：                                       │
│  ✓ 团队3人完整（创始人+2合伙人）                      │
│  ✓ 能力覆盖完整（技术+商业+产品/运营）                │
│  ✓ 团队信任度 ≥ 6                                    │
│  ✓ 剩余资源 ≥ 3个任意币                              │
│                                                     │
│  评审投骰：                                          │
│  • 基础点数：1d6                                     │
│  • 加成检查：                                        │
│    - 能力完美互补：+1                                │
│    - 团队信任度≥8：+1                                │
│    - 拥有COO角色：+1                                 │
│    - 使用深度尽职调查过所有合伙人：+1                │
│                                                     │
│  结果判定：                                          │
│  • 总分≥4：成功IPO，游戏胜利                         │
│  • 总分<4：准备不足，回退到第25格                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 8.5 特殊规则

#### 合伙人离开机制
1. **主动离开**：忠诚度不足时，危机事件中可能离开
2. **被动解除**：花费1时间币+1信任币，可无损解除合作
3. **离开后果**：失去该合伙人所有能力，信任度-1

#### 玩家间互动
1. **CTO争夺战**：特定格子触发，竞价争夺候选人
2. **人才推荐**：可向其他玩家推荐不需要的候选人
3. **信息交易**：可协商交换资源或信息（非强制）

#### 游戏结束变体
```
【快速模式】（45分钟）
• 棋盘缩短至20格
• 团队只需2人（创始人+1合伙人）
• IPO信任度要求降至4点

【史诗模式】（2小时）
• 棋盘扩展至40格
• 团队可招募3名合伙人
• 增加"全球化"终点格

【教育模式】（90分钟）
• 每轮增加"反思讨论"环节
• 必须解释决策理由
• 复盘满意解应用
```

## 九、教育目标实现

### 9.1 体验"满意解"决策过程

#### 游戏机制映射
| 满意解概念 | 游戏机制 | 体验设计 |
|------------|----------|----------|
| 有限理性 | 资源限制（时间/金钱） | 无法完美评估所有候选人 |
| 信息成本 | 工具卡消耗资源 | 权衡调查深度与成本 |
| 足够好即可 | 非最优解也可成功 | 平衡型团队胜过全能型 |
| 动态调整 | 合伙人可更换 | 接受初始选择不完美 |

#### 关键教学点
1. **第10-15格**：资源紧张期，必须做出取舍
2. **团队磨合事件**：初期选择不完美，需要磨合
3. **IPO评审设计**：不要求完美团队，只要"足够好"

### 9.2 理解合伙人匹配的多维度

#### 能力维度可视化
- 候选人卡上的星级系统
- 能力雷达图对比工具
- 团队组合效果实时显示

#### 隐藏维度揭示
- 忠诚度（长期合作基础）
- 价值观（冲突预防）
- 风险点（意外准备）

### 9.3 感受决策中的压力和不确定性

#### 压力设计
```
【资源压力】
• 时间币每回合重置 → 催促快速决策
• 机会窗口限时 → 制造紧迫感
• 竞争对手进度 → 比较焦虑

【信息压力】
• 直觉 vs 理性选择 → 认知冲突
• 不完整信息决策 → 接受不确定性
• 危机事件突袭 → 应对突发状况
```

### 9.4 学习直觉和理性的平衡

#### 直觉系统
- 低成本但可能出错
- 某些角色直觉加成
- 紧急情况下依赖直觉

#### 理性系统
- 高成本但准确
- 资源投入可累积信息
- 最终决策的依据

#### 平衡教学
游戏鼓励混合策略：直觉筛选 → 理性验证 → 满意决策
