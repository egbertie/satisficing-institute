## 二、辩论流程标准化

### 2.1 单回合流程（30分钟）

```
时间节点    角色          动作                      时长
─────────────────────────────────────────────────────────────
T+0         满意妞        提出初始方案               2分钟
            └── 包含: 背景、问题、建议方案、预期结果

T+2         黎红雷        伦理角度回应               3分钟
            └── 要点: 价值观对齐、伦理风险、长期影响

T+5         罗汉          数学角度回应               3分钟
            └── 要点: 概率评估、期望效用、风险量化

T+8         客户替身      需求角度回应               3分钟
            └── 要点: 可接受度、执行可行性、情感反应

T+11        蓝军          弱点攻击                   3分钟
            └── 要点: 系统性挑刺、极端场景、假设质疑

T+14        全员          自由辩论                   10分钟
            └── 规则: 
                - 每人发言不超过2分钟
                - 必须回应他人观点
                - 满意妞控制节奏

T+24        满意妞        总结分歧点和共识           4分钟
            └── 输出: 共识清单、分歧清单、待澄清点

T+28        满意妞        生成变体方案               2分钟
            └── 方法: 针对分歧点生成3-5个改进方案
─────────────────────────────────────────────────────────────
```

### 2.2 完整辩论流程（3-5回合）

```
回合1: 初始方案辩论
  └── 输入: 初始决策方案
  └── 输出: 变体方案A、B、C

回合2: 变体方案A辩论
  └── 输入: 变体方案A
  └── 输出: 改进方案A1、A2

回合3: 变体方案B辩论
  └── 输入: 变体方案B
  └── 输出: 改进方案B1、B2

回合4（可选）: 综合方案辩论
  └── 输入: 整合最优元素的综合方案
  └── 输出: 最终候选方案

回合5（可选）: 最终方案确认
  └── 输入: 最终候选方案
  └── 输出: 推荐方案 + 备选方案
```

### 2.3 辩论触发条件

| 场景 | 触发辩论 | 参与智能体 | 回合数 |
|------|----------|-----------|--------|
| 新决策方案 | 自动 | 全部 | 3-5 |
| 高风险决策 | 强制 | 全部 | 5 |
| 常规决策 | 简化 | 2-3个 | 2-3 |
| 紧急决策 | 快速 | 2个 | 1-2 |

## 三、PUCT变体：辩论中的探索-利用平衡

### 3.1 公式定义

借鉴AlphaZero的PUCT公式，设计辩论中的节点选择策略：

```
DPUCT(d) = Q(d) + c_debate * P(d) * sqrt(N_parent) / (N_d + 1)

其中：
- d: 决策方案/变体
- Q(d): 方案质量评分（历史平均）
- P(d): 先验概率（模式网络输出）
- N_parent: 父节点辩论次数
- N_d: 该方案辩论次数
- c_debate: 探索常数（默认2）
```

### 3.2 应用示例

```
场景: 评估3个候选方案

方案A: Q=0.8, P=0.5, N=5
方案B: Q=0.7, P=0.3, N=2
方案C: Q=0.6, P=0.2, N=0 (新方案)

计算DPUCT（假设c_debate=2, N_parent=7）:

方案A: 0.8 + 2*0.5*sqrt(7)/6 = 0.8 + 0.44 = 1.24
方案B: 0.7 + 2*0.3*sqrt(7)/3 = 0.7 + 0.53 = 1.23
方案C: 0 + 2*0.2*sqrt(7)/1 = 0 + 1.06 = 1.06

结论: 选择方案A进行下一轮辩论
```

### 3.3 动态调整

| 阶段 | c_debate值 | 策略倾向 |
|------|-----------|----------|
| 早期探索 | 3-4 | 优先尝试新方案 |
| 中期平衡 | 2 | 探索与利用平衡 |
| 后期收敛 | 1 | 优先优化最优方案 |

## 五、运行手册

### 5.1 启动辩论

```bash
# 命令格式
debate start --topic "决策主题" \
             --client 客户替身ID \
             --complexity high/medium/low \
             --rounds 3

# 示例
debate start --topic "张建国股权分配" \
             --client zhangjianguo \
             --complexity high \
             --rounds 4
```

### 5.2 监控辩论

```bash
# 查看辩论状态
debate status --id DEB-2026-001

# 强制进入下一轮
debate next --id DEB-2026-001

# 终止辩论
debate stop --id DEB-2026-001 --reason "已达成共识"
```

### 5.3 输出处理

```bash
# 导出辩论记录
debate export --id DEB-2026-001 --format yaml

# 生成决策建议报告
debate report --id DEB-2026-001 --include-all

# 标注决策模式
debate label --id DEB-2026-001 --case-id CASE-001
```

## 六、质量控制

### 6.1 辩论质量检查表

| 检查项 | 标准 | 检查方法 |
|--------|------|----------|
| 参与度 | 每个智能体至少发言3次 | 自动统计 |
| 观点多样性 | 存在实质性分歧 | 语义分析 |
| 深度 | 触及至少2层假设 | 逻辑分析 |
| 实用性 | 有可执行建议 | 专家评估 |
| 记录完整性 | 所有输出物齐全 | 自动检查 |

### 6.2 常见问题处理

| 问题 | 症状 | 解决方案 |
|------|------|----------|
| 观点趋同 | 缺乏实质性分歧 | 增强蓝军攻击性 |
| 脱离实际 | 建议过于理想化 | 强化客户替身权重 |
| 过度批判 | 所有方案被否定 | 调整蓝军参数 |
| 效率低下 | 单回合超时 | 设置发言时间限制 |
