# 案例：硬科技创业团队股权纠纷

## 基本信息
| 项目 | 内容 |
|------|------|
| **案例ID** | CASE-001 |
| **行业** | 智能硬件 |
| **融资阶段** | 天使轮 |
| **团队规模** | 3位创始人 |
| **案例来源** | 公开报道整理 |

## 背景
某智能硬件创业公司，三位创始人：
- CTO（技术主导，持股40%）
- CEO（运营主导，持股35%）  
- CMO（市场主导，持股25%）

## 问题
1. **技术话语权过大**: CTO认为产品是自己做的，要求控股
2. **市场验证分歧**: CMO认为应该快速推广，CTO坚持继续打磨
3. **股权结构隐患**: 40/35/25结构没有预留期权池，后续融资困难

## 决策失误点
1. **股权分配依据**: 按出资额分配，未考虑长期贡献
2. **决策机制缺失**: 没有约定重大事项表决规则
3. **退出机制空白**: 未约定合伙人退出处理方式

## 后果
- 团队内斗18个月
- 产品上市延期
- 天使轮资金耗尽
- 公司清算，账上剩7万元

## 满意解分析
**LIU（价值纯度）**: 三人初心都是做产品，但后期目标分化
**SIMON（理性框架）**: 股权结构缺乏动态调整机制
**CONFUCIUS（合伙伦理）**: 只谈利益，未建立信任基础

## 教训
1. 股权分配应考虑长期贡献，而非初期出资
2. 必须约定决策机制和退出条款
3. 合伙人选择阶段就要评估价值观一致性
