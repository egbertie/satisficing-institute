# 案例：政府补贴合伙人选择

## 基本信息
| 项目 | 内容 |
|------|------|
| **案例ID** | CASE-002 |
| **行业** | 新能源 |
| **补贴类型** | 工信部创新基金 |
| **补贴金额** | 500万 |
| **案例来源** | 用户访谈整理 |

## 背景
创始人A获得政府补贴资格，需要找合伙人B共同申报：
- A有技术，缺市场和资金
- B承诺带来客户资源和200万跟投
- 双方约定50/50股权

## 问题
1. **资源承诺未兑现**: B承诺的客户资源未能落地
2. **资金迟迟不到位**: B的200万跟投只到位50万
3. **补贴使用分歧**: A想用补贴研发，B想用补贴还前期债务

## 决策失误点
1. **未验证资源真实性**: 仅凭口头承诺就确定股权
2. **未设置兑现条件**: 股权一次性给出，未与资源兑现挂钩
3. **补贴用途未约定**: 未在合伙协议中明确补贴资金用途

## 后果
- 补贴申报延迟6个月
- B占用公司资金偿还个人债务
- 最终对簿公堂，项目停滞
- 补贴资格险些被取消

## 满意解分析
**SIMON（理性框架）**: 股权分配缺乏兑现条件设计
**GUANYIN（极限测试）**: 未对B的资源承诺做压力测试
**HUINENG（知行合一）**: 未将约定写入合伙协议

## 教训
1. 资源承诺必须有兑现时间表和验证机制
2. 股权应分期兑现，与资源落地挂钩
3. 补贴资金用途必须在合伙协议中明确
