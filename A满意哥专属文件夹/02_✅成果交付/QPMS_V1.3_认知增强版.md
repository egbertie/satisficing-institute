## 一、版本概述

### 1.1 升级背景

QPMS V1.2 实现了基于感知力的合伙人匹配核心功能，但在实际应用中发现：
- 客户决策质量受认知偏差影响显著
- 传统匹配算法未考虑心理偏差因素
- 缺乏系统性的决策支持工具

### 1.2 V1.3核心升级

```
V1.2 基础功能
    ├─ 候选人评估模型
    ├─ 匹配算法
    └─ 决策支持报告
        ↓
V1.3 认知增强
    ├─ 【新增】认知偏差检测模块
    ├─ 【新增】框架重构建议引擎
    ├─ 【新增】损失厌恶度量表集成
    ├─ 【增强】决策质量评估算法
    └─ 【新增】决策偏差诊断器（首单工具）
```

### 1.3 价值主张

> "不只是找到好合伙人，更是帮助你在有限理性和认知偏差的双重约束下，做出真正满意的决策"

### 2.2 核心模块详解

#### 模块一：认知偏差检测模块 (Cognitive Bias Detection Module, CBDM)

**功能目标**：自动识别客户决策过程中的认知偏差

**技术规格**：

| 组件 | 功能 | 技术栈 |
|------|------|--------|
| 输入处理器 | 解析客户文本输入 | NLP (BERT-based) |
| 情感分析器 | 识别情绪倾向和强度 | 情感分析模型 |
| 偏差分类器 | 识别5种主要偏差 | 规则引擎 + ML |
| 严重度评估器 | 计算偏差指数 | 加权评分算法 |
| 报告生成器 | 输出可视化报告 | 数据可视化 |

**检测的偏差类型**：

```python
BIAS_TYPES = {
    "loss_aversion": {
        "name": "损失厌恶",
        "keywords": ["失去", "稀释", "被抢走", "降低", "削弱"],
        "emotional_markers": ["恐惧", "焦虑", "担心"],
        "weight": 0.30
    },
    "endowment_effect": {
        "name": "禀赋效应",
        "keywords": ["我的", "我创建的", "独一无二", "无可替代"],
        "emotional_markers": ["骄傲", "占有欲"],
        "weight": 0.25
    },
    "status_quo_bias": {
        "name": "现状偏见",
        "keywords": ["维持现状", "不想改变", "再看看", "等等"],
        "emotional_markers": ["惰性", "舒适"],
        "weight": 0.20
    },
    "anchoring_effect": {
        "name": "锚定效应",
        "keywords": ["上轮融资", "去年估值", "一开始说", "原来"],
        "emotional_markers": ["固执"],
        "weight": 0.15
    },
    "overconfidence": {
        "name": "过度自信",
        "keywords": ["我很清楚", "我肯定", "绝对", "不可能错"],
        "emotional_markers": ["傲慢", "轻视"],
        "weight": 0.10
    }
}
```

**输入输出示例**：

```
【输入】
客户描述："我担心引入合伙人后，我的股权会被稀释，
          控制权也会减弱。现在的团队虽然不够强，
          但至少听话。而且我的技术在市场上是独一无二的。"

【处理过程】
1. 关键词提取：
   - "稀释" → loss_aversion +15
   - "控制权减弱" → loss_aversion +10
   - "虽然...但是" → status_quo_bias +12
   - "独一无二" → endowment_effect +18

2. 情感分析：
   - 恐惧：0.72
   - 焦虑：0.65
   - 防御：0.58

3. 偏差评分：
   - 损失厌恶：78/100（高）
   - 禀赋效应：72/100（高）
   - 现状偏见：55/100（中等）
   - 锚定效应：25/100（低）
   - 过度自信：45/100（中等）

【输出】
- 综合偏差指数：67/100（需要干预）
- 主要偏差：损失厌恶 + 禀赋效应
- 风险等级：中等偏高
```

#### 模块二：框架重构建议引擎 (Frame Refactoring Engine, FRE)

**功能目标**：提供认知框架重构建议，改善决策质量

**重构策略库**：

```python
REFRAME_STRATEGIES = {
    "loss_to_gain": {
        "name": "损失转收益",
        "trigger": "loss_aversion_high",
        "rules": [
            {
                "pattern": "失去{X}股权",
                "reframe": "保留{100-X}%的股权，获得一个{Y}倍大的公司"
            },
            {
                "pattern": "控制权削弱",
                "reframe": "影响力放大，通过合伙人的专业能力"
            }
        ]
    },
    "reference_shift": {
        "name": "参考点转换",
        "trigger": "anchoring_effect_high",
        "rules": [
            {
                "pattern": "当前估值{X}",
                "reframe": "2年后预期估值{X*3}，现在的比例是..."
            }
        ]
    },
    "endowment_break": {
        "name": "禀赋效应打破",
        "trigger": "endowment_effect_high",
        "rules": [
            {
                "action": "引入第三方估值",
                "script": "让我们看看市场上类似技术的估值..."
            },
            {
                "action": "对比案例",
                "script": "类似公司的创始人是如何评估的..."
            }
        ]
    }
}
```

**建议生成流程**：

```
偏差检测结果
      ↓
策略匹配（基于偏差类型和严重度）
      ↓
个性化建议生成
      ↓
输出结构：
├─ 偏差说明（是什么）
├─ 影响分析（为什么重要）
├─ 重构建议（怎么做）- 3-5条
└─ 支持工具（用什么）
```

#### 模块三：损失厌恶度量表集成 (Loss Aversion Measurement Scale, LAMS)

**量表设计**：

```
维度1：股权稀释敏感度（权重30%）
├─ Q1: 如果引入合伙人让您的股权从100%→70%，但公司估值增长5倍，您的感受？
│   A. 非常不舒服（+20） B. 有点不舒服（+10） C. 可以接受（+5） D. 非常乐意（+0）
├─ Q2: 您认为多少股权比例是"不可接受"的底线？
│   A. 低于60%（+15） B. 低于50%（+10） C. 低于40%（+5） D. 都可以接受（+0）
└─ Q3: 相比股权比例，您更看重？
    A. 绝对控制权（+20） B. 相对控制权（+10） C. 实际影响力（+5） D. 经济回报（+0）

维度2：机会成本认知（权重25%）
├─ Q4: 每拖延一个月找到合适合伙人，您认为的机会成本是？
│   A. 超过100万（+0，认知清晰） B. 50-100万（+5） C. 10-50万（+10） D. 说不清楚（+20）
├─ Q5: 您认为"不决策"本身是一种决策吗？
    A. 是，而且往往是最差的（+0） B. 是，但有时是正确的（+10） C. 不是（+20）

维度3：风险偏好（权重25%）
├─ Q6: 确定获得50万 vs 50%概率获得120万，您选择？
│   A. 确定50万（+15，风险厌恶） B. 犹豫后选50万（+10） C. 选120万（+5）
├─ Q7: 确定损失30万 vs 50%概率损失70万/0损失，您选择？
    A. 确定损失（+5，反射效应弱） B. 赌一把（+20，反射效应强）

维度4：后悔厌恶（权重20%）
├─ Q8: 您更担心？
│   A. 选错了合伙人（+10） B. 错过了好合伙人（+10） C. 两者都担心（+15） D. 都不担心（+5）
└─ Q9: 过去的决策中，您更容易后悔？
    A. 做错了的事（+15） B. 没做的事（+5） C. 差不多（+10）
```

**评分解释**：

| 分数区间 | 损失厌恶等级 | 干预策略 |
|----------|-------------|---------|
| 0-25 | 低 | 强调风险，理性分析 |
| 26-50 | 中低 | 标准沟通 |
| 51-75 | 中等 | 轻度框架重构 |
| 76-100 | 高 | 深度认知干预 |

#### 模块四：决策偏差诊断器 (Partner Decision Bias Diagnoser, PDBD)

**产品定位**：首单获客工具，免费/低价版本

**功能流程**：

```
【Step 1: 信息收集】（3分钟）
├─ 基础信息：行业、阶段、团队规模
├─ 决策场景：引入合伙人/股权分配/...
└─ 开放式描述："请描述您对引入合伙人的顾虑..."

【Step 2: 快速测试】（2分钟）
├─ 5道情景选择题
└─ 基于Kahneman经典实验改编

【Step 3: AI分析】（实时）
├─ NLP分析文本输入
├─ 偏差识别
└─ 案例匹配

【Step 4: 生成报告】（即时）
├─ 偏差雷达图
├─ 风险等级
├─ 改进建议
└─ 专家咨询CTA
```

**报告模板**：

```markdown
# 合伙人决策偏差诊断报告

## 基本信息
- 诊断ID：PDBD-2026-03-15-XXXX
- 诊断时间：2026-03-15
- 决策场景：早期合伙人引入

## 综合评分

### 🎯 决策健康指数：62/100
（中等风险 - 需要关注）

## 偏差分析

### 🔴 损失厌恶：82/100（偏高）
**表现**：
- 对"失去控制权"的敏感度极高
- 更关注会失去什么，而非能获得什么

**可能影响**：
- 过于保守，错过优质合伙人
- 谈判中让步空间不足

**建议**：
1. 尝试"收益框架"重新思考：不是"失去30%"，而是"获得70%的更大公司"
2. 计算不同决策路径的期望值

### 🟡 禀赋效应：65/100（中等偏高）
**表现**：
- 对现有团队和技术估值偏高
- 难以接受外部视角的评估

**建议**：
1. 获取第三方市场估值
2. 参考同类公司的合伙人案例

### 🟢 现状偏见：45/100（正常范围）
**表现**：改变意愿尚可

## 风险预测

基于您的偏差模式，如果不进行干预：
- 65%概率：延迟决策，错过窗口期
- 25%概率：过度保守，合伙人不欢而散
- 10%概率：做出满意决策

## 个性化建议

1. **立即行动**：预约30分钟专家咨询（免费）
2. **自我学习**：阅读《合伙人决策中的认知陷阱》
3. **工具试用**：使用框架转换器优化您的决策描述

## 下一步

[预约专家解读] [下载详细报告] [分享给合伙人]

---
*本报告由QPMS认知增强版生成*
```

### 4.2 关键算法

#### 偏差检测算法

```python
class BiasDetector:
    def __init__(self):
        self.nlp_model = load_bert_model()
        self.bias_rules = load_bias_rules()
    
    def detect(self, text: str) -> BiasReport:
        # 1. 情感分析
        sentiment = self.nlp_model.analyze_sentiment(text)
        
        # 2. 关键词匹配
        keywords = self.extract_keywords(text)
        
        # 3. 偏差评分
        bias_scores = {}
        for bias_type, rules in self.bias_rules.items():
            score = self.calculate_bias_score(keywords, sentiment, rules)
            bias_scores[bias_type] = score
        
        # 4. 生成报告
        return BiasReport(
            scores=bias_scores,
            overall_risk=self.calculate_overall_risk(bias_scores),
            recommendations=self.generate_recommendations(bias_scores)
        )
```

#### 框架重构算法

```python
def reframe_strategy(bias_type: str, original_text: str) -> str:
    strategies = {

            ]
        }
    }
    
    strategy = strategies.get(bias_type)
    # 匹配模式并应用模板
    return apply_template(original_text, strategy)
```
