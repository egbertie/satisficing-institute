## 一、流程概述

### 核心理念

**赫伯特·西蒙**的满意解理论指出：在信息不完备、时间有限、认知有限的现实中，决策者追求的是"足够好"而非"理论上最优"。

**六祖慧能**的顿悟智慧告诉我们：真正的洞见往往在极限情境中迸发——"若能内外兼修，淤泥定生红莲"。

**72小时极限决策流程**正是将两者结合：
- 用**时间压力**模拟真实决策情境
- 用**结构化流程**确保评估系统性
- 用**直觉触发机制**捕捉理性无法触及的真知

---

## 二、适用场景

### 推荐使用

- ✅ 已有2-3个合伙人候选人，需要快速筛选
- ✅ 面临融资窗口、市场机会等时间压力
- ✅ 创始人决策风格偏直觉，需要结构化验证
- ✅ 合伙人选择陷入纠结，需要外部推动

### 不适用场景

- ❌ 只有一个候选人（缺乏比较基准）
- ❌ 创始人完全没有准备，对需求不清晰
- ❌ 存在重大法律风险或诚信问题需优先处理

## 三、72小时流程详解

### 预备阶段（决策前1-3天）

**目标**：准备充分，确保72小时高效运转

**关键动作**：

| 动作 | 负责人 | 交付物 | 时间 |
|:----:|:------:|:------:|:----:|
| 需求澄清访谈 | 教练 | 需求画像 | 2h |
| 候选人资料收集 | 客户 | 候选人档案 | 1天 |
| 评估工具准备 | 系统 | 评估问卷 | 自动 |
| 五图腾数字团队就位 | PEO | 会诊排期 | 4h |

**客户准备清单**：
- [ ] 明确自身优势和短板
- [ ] 整理2-3个候选人的详细资料
- [ ] 清理日程，确保72小时内可响应
- [ ] 心理准备：接受极限压力

---

### Day 1：信息风暴与初步筛选（0-24h）

#### 上午（0-4h）：信息输入

**目标**：在最短时间内获取最大信息量

**流程**：
```
09:00-10:00  候选人A深度访谈（1h）
10:30-11:30  候选人B深度访谈（1h）
14:00-15:00  候选人C深度访谈（1h）
15:30-17:00  背景调查与资料核实（1.5h）
```

**工具**：合伙人深度访谈问题清单（4轮问题）

**五图腾参与**：
- **LIU（土）**：监督访谈是否触及价值本质
- **SIMON（金）**：确保问题覆盖7维度框架

---

#### 下午（4-12h）：7维度初步评分

**目标**：基于第一天信息完成初步量化评估

**评分表**：

| 维度 | 权重 | 候选人A | 候选人B | 候选人C |
|:----:|:----:|:-------:|:-------:|:-------:|
| 价值观契合度 | 20% | /10 | /10 | /10 |
| 能力互补性 | 20% | /10 | /10 | /10 |
| 承诺可信度 | 15% | /10 | /10 | /10 |
| 沟通效率 | 15% | /10 | /10 | /10 |
| 利益一致性 | 15% | /10 | /10 | /10 |
| 退出可接受性 | 10% | /10 | /10 | /10 |
| 成长匹配度 | 5% | /10 | /10 | /10 |
| **加权总分** | 100% | | | |

**当日输出**：
- 3位候选人的7维度初评分数
- 淘汰1位明显不匹配者（如有）
- 进入Day 2的2位候选人名单

**五图腾参与**：
- **GUANYIN（水）**：关注客户评分时的焦虑程度
- **CONFUCIUS（木）**：审查评分背后的伦理考量

---

#### 晚上（12-24h）：第一轮顿悟触发

**目标**：在疲惫状态下捕捉直觉反应

**触发练习**：

**练习1：快速联想**（10分钟）
```
闭上眼睛，深呼吸3次。
现在，第一个浮现在你脑海中的合伙人形象是谁？
不要思考，直接写下名字。
```

**练习2：逆向想象**（15分钟）
```
想象3年后，你和一个合伙人成功了。
回头看，你现在希望当时选了谁？
写下名字和三个理由。
```

**练习3：恐惧探索**（15分钟）
```
想象你选择了候选人A，3年后合作失败了。
最可能的原因是什么？
想象你选择了候选人B，3年后合作失败了。
最可能的原因是什么？
哪个失败你更不能接受？
```

**记录**：直觉反应与理性评分的差异

**五图腾参与**：
- **HUINENG（火）**：设计并引导顿悟触发练习

---

### Day 2：压力测试与冲突模拟（24-48h）

#### 上午（24-32h）：冲突模拟测试

**目标**：在可控压力下观察候选人的反应模式

**场景设计**：

**场景A：股权分配冲突**
```
背景：公司需要再融资，新投资人要求稀释创始团队股权

候选人角色：技术合伙人
你的角色：CEO

冲突点：候选人认为应该等比例稀释，你认为应该按贡献度差异化稀释

任务：与候选人进行15分钟的立场对话
观察：对方的沟通风格、妥协意愿、冲突处理能力
```

**场景B：产品方向分歧**
```
背景：技术团队主张做高端产品线，市场团队主张先做低端走量

候选人角色：技术/市场负责人（视候选人背景而定）
你的角色：CEO

冲突点：资源有限，只能选择一条路线

任务：与候选人进行15分钟的立场对话
观察：对方的论证方式、对不同意见的包容度、输赢心态
```

**评分维度**：
- 沟通风格（攻击性1-10建设性）
- 情绪控制（失控1-10从容）
- 妥协意愿（固执1-10灵活）
- 解决方案质量（差1-10优秀）

**五图腾参与**：
- **HUINENG（火）**：设计冲突场景
- **GUANYIN（水）**：评估冲突后的状态恢复

---

#### 下午（32-40h）：极限情境决策

**目标**：在时间压力下做出选择

**情境设定**：
```
现在是周五下午5点。
投资人周一上午要最终答复，否则窗口关闭。
你必须在今天晚上12点前做出选择。
你还有2小时与候选人沟通的时间。
```

**决策任务**：
1. 与两位候选人各进行30分钟的关键问题澄清
2. 在18:00前完成最终7维度评分
3. 在20:00前写出初步决策建议
4. 在22:00前与教练进行决策对谈
5. 在24:00前做出Go/No-Go决策

**限制条件**：
- 不允许联系新的推荐人
- 不允许查阅新的资料
- 不允许推迟决策

**五图腾参与**：
- **SIMON（金）**：确保评分逻辑严谨
- **LIU（土）**：防止在压力下偏离价值本质

---

#### 晚上（40-48h）：第二轮顿悟触发

**目标**：在高度疲惫状态下捕捉深层直觉

**触发练习**：

**练习4：身体扫描**（20分钟）
```
分别回忆与两位候选人的相处时刻。
注意你的身体反应：
- 想起谁时肩膀更放松？
- 想起谁时胃部更舒服？
- 想起谁时呼吸更顺畅？

身体比大脑更诚实。
```

**练习5：梦境记录**（次日早晨）
```
明天醒来后，立即记录：
- 是否做了与合伙人相关的梦？
- 梦中的感受是什么？
- 梦境给你什么启示？
```

**五图腾参与**：
- **HUINENG（火）**：解读身体信号与梦境

---

### Day 3：决策验证与承诺固化（48-72h）

#### 上午（48-56h）：数字替身会诊

**目标**：五图腾数字团队联合评估

**会诊流程**：

```
09:00-09:30  案情简报（PEO主持）
09:30-10:00  LIU（土）观点：价值纯度评估
10:00-10:30  SIMON（金）观点：方法论严谨性审查
10:30-11:00  GUANYIN（水）观点：决策状态评估
11:00-11:30  CONFUCIUS（木）观点：伦理契合度审查
11:30-12:00  HUINENG（火）观点：直觉洞见解读
12:00-12:30  综合讨论与建议形成
```

**输出**：《五图腾会诊意见书》

---

#### 下午（56-64h）：决策确认

**目标**：在多重验证下确认最终决策

**决策确认清单**：

| 确认项 | 自检 | 教练确认 | 五图腾确认 |
|:------:|:----:|:--------:|:----------:|
| 价值观契合度 ≥7分 | ☐ | ☐ | ☐ |
| 能力互补性 ≥8分 | ☐ | ☐ | ☐ |
| 承诺可信度 ≥7分 | ☐ | ☐ | ☐ |
| 直觉与理性一致 | ☐ | ☐ | ☐ |
| 可接受最坏结果 | ☐ | ☐ | ☐ |

**决策仪式**：

**书面确认**（30分钟）
```
亲爱的未来的我：

今天是2026年X月X日，我做出了选择合伙人的决定。

我选择：_______________

核心理由（不超过3条）：
1. ___________________
2. ___________________
3. ___________________

我知道的最坏可能是：______________
如果发生，我将：__________________

我对此决策的信心指数（1-10）：_____

签名：___________ 日期：___________
```

**五图腾参与**：
- 全体：联合签署《决策确认书》

## 四、风险控制机制

### 流程风险

| 风险点 | 应对措施 |
|:------:|:--------:|
| 客户无法承受压力 | 设置"安全词"，可随时暂停 |
| 信息收集不充分 | Day 1延长4h（备用） |
| 候选人无法配合 | 改用情景模拟替代真人测试 |
| 客户决策瘫痪 | 强制决策机制，超时视为放弃 |

### 决策风险

| 风险点 | 应对措施 |
|:------:|:--------:|
| 直觉与理性严重冲突 | 延长观察期1周 |
| 两位候选人分数接近 | 增加一轮共创测试 |
| 关键信息缺失 | 延期决策，补充尽调 |

## 六、成功案例

### CASE-A：AI芯片创业团队

**背景**：
- 创始人：技术背景，连续创业者
- 候选人：2位，均为前同事
- 时间压力：融资窗口仅剩1周

**72小时流程亮点**：
- Day 1：发现候选人A存在隐瞒兼职情况
- Day 2：压力测试揭示候选人B的妥协意愿更强
- Day 3：顿悟练习显示身体倾向于候选人B

**结果**：
- 选择候选人B
- 6个月后成功完成A轮融资
- 目前合作良好
