> 满意解研究所 · 案例库建设规范
> 
> "从真实中提炼智慧，从经验中萃取方法"

---

## 一、研究目标

### 核心目标

1. **积累实证数据**：收集硬科技创业合伙人匹配的真实案例
2. **提炼成功模式**：识别成功合伙的共性因素
3. **警示失败陷阱**：总结导致合伙破裂的关键风险
4. **迭代方法论**：基于案例反馈持续优化评估框架

### 研究原则

| 原则 | 说明 |
|:----:|:----:|
| **真实性** | 所有案例必须基于真实事件，不得虚构 |
| **保密性** | 敏感信息脱敏处理，保护当事人隐私 |
| **系统性** | 统一数据收集框架，确保可比性 |
| **持续性** | 长期跟踪，捕捉合伙关系演变 |

---

## 二、案例采集标准

### 入选条件

**必须满足**：
- [ ] 硬科技领域（芯片、新材料、新能源、生物医药、航空航天等）
- [ ] 有明确的合伙人选择/匹配过程
- [ ] 可获取足够的信息进行7维度评估

**优先采集**：
- 有明确结果（成功/失败）的案例
- 有戏剧性转折的案例
- 能体现方法论有效性的案例

### 案例分级

| 等级 | 标准 | 研究价值 |
|:----:|:----:|:--------:|
| **P0** | 完整7维度数据 + 长期跟踪 + 详细访谈 | 核心研究样本 |
| **P1** | 完整7维度数据 + 结果确认 | 重要参考样本 |
| **P2** | 部分数据 + 二手信息 | 辅助参考 |

## 四、访谈指南

### 创始人访谈（90分钟）

**第一部分：背景与动机（20分钟）**
```
1. 请介绍一下您的创业背景和这个项目。
2. 当初为什么决定要找一个合伙人？
3. 您对理想合伙人的画像是什么？
```

**第二部分：匹配过程（30分钟）**
```
4. 您和合伙人是怎么认识的？
5. 从认识到达成合作意向，花了多长时间？
6. 这个过程中您做了哪些评估？
7. 有没有什么关键事件影响了您的决策？
```

**第三部分：合作实况（25分钟）**
```
8. 实际合作中和预期有什么差异？
9. 遇到过什么冲突？怎么解决的？
10. 如果重来一次，您会在评估时注意什么？
```

**第四部分：反思与建议（15分钟）**
```
11. 用一个词形容您的合伙关系。
12. 对其他正在找合伙人的创始人，您有什么建议？
```

---

### 合伙人访谈（60分钟）

**核心问题**：
```
1. 当初为什么选择加入这个团队？
2. 合作前您做了哪些评估？
3. 实际合作中和预期有什么差异？
4. 您如何看待双方的互补性？
5. 对未来的合作有什么期待？
```

### 案例更新机制

**定期更新**：
- 每季度回访P0案例，更新状态
- 每年度全面复盘，重新评估7维度

**触发更新**：
- 案例出现重大转折（融资/破裂/上市等）
- 发现之前评估存在重大偏差
- 方法论迭代需要补充数据

## 八、伦理规范

### 隐私保护

- 所有公开案例必须获得当事人书面授权
- 敏感信息（姓名、公司名、具体金额）脱敏处理
- 案例编号唯一对应，访问权限分级管理

### 使用规范

- 案例仅用于研究和服务优化
- 不得用于商业推广（除非获得授权）
- 引用案例需注明来源
