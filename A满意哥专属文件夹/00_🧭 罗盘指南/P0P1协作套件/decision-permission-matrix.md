## 风险分级标准

### 低风险（AI自主）
**特征**: 只读操作、内部数据、可逆操作
**权限**: AI自主执行，无需确认
**通知**: 仅记录日志
**典型场景**:
- 搜索查询（网页、学术、专利）
- 数据分析（内部数据库）
- 报告草稿生成（未发送）
- 内部文件读取

### 中风险（执行后告知）
**特征**: 影响内部状态、可撤销、不涉及外部
**权限**: AI执行，事后告知
**通知**: 每日摘要
**典型场景**:
- 生成报告（保存到本地/Notion）
- 定时任务调整
- 内部文件修改/删除
- 仪表盘数据更新

### 高风险（事前确认）
**特征**: 对外影响、不可逆、涉及敏感信息
**权限**: AI建议，必须确认
**通知**: 即时推送，等待确认
**典型场景**:
- 发送消息（飞书/邮件/短信）
- 对外发布内容
- 删除关键文件
- API密钥操作

## 风险评估算法

```javascript
function assessRisk(action, context) {
  let riskScore = 0;
  
  // 基础风险
  const baseRisk = {
    'search': 0,
    'analyze': 5,
    'draft': 5,
    'generate': 20,
    'modify': 30,
    'delete': 50,
    'send': 60,
    'config': 40
  };
  riskScore += baseRisk[action.type] || 30;
  
  // 影响范围
  if (context.scope === 'external') riskScore += 30;
  if (context.scope === 'critical') riskScore += 40;
  
  // 可逆性
  if (!context.reversible) riskScore += 20;
  
  // 敏感信息
  if (context.containsSensitive) riskScore += 25;
  
  // 历史成功率
  if (context.historicalSuccessRate < 0.8) riskScore += 15;
  
  return {
    score: riskScore,
    level: riskScore < 30 ? 'low' : riskScore < 70 ? 'medium' : 'high',
    requiresConfirmation: riskScore >= 70
  };
}
```

## 执行流程

### 低风险管理流程
```
请求 → 风险评估 → 自主执行 → 记录日志 → 完成
```

### 中风险管理流程
```
请求 → 风险评估 → 自主执行 → 加入每日摘要 → 完成
```

### 高风险管理流程
```
请求 → 风险评估 → 生成建议 → 即时通知 → 等待确认 
  ↓确认
执行 → 记录结果 → 完成
  ↓拒绝/超时
取消 → 记录原因 → 结束
```

## 异常处理

### 误判情况
如果AI评估错误，用户可：
1. 立即中断操作
2. 反馈"此操作应为X风险"
3. AI学习并调整评估参数

### 紧急情况
对于真正紧急的高风险操作：
1. AI标注"紧急"标签
2. 缩短确认等待时间（5分钟）
3. 多渠道通知（飞书+短信）
4. 超时自动执行（如配置允许）
