## S2: 系统考虑（输入→处理→输出→反馈闭环）

### 2.1 输入规范

**必需输入**：
| 输入项 | 格式 | 说明 |
|--------|------|------|
| 项目基本情况 | 结构化文本 | 技术领域、发展阶段、团队构成 |
| 发展需求 | 清单 | 资金、空间、人才、市场等需求 |
| 区位偏好 | 枚举值 | 深圳/香港/广州/东莞等 |

**可选输入**：
- 现有资源禀赋
- 竞争对手分布
- 政策敏感度说明

### 2.2 处理流程

**阶段一：需求解析（水·溯源）**
```
项目信息 → 核心需求提取 → 资源缺口识别 → 优先级排序
```

**阶段二：政策匹配（水·流动）**
```
需求清单 → 政策数据库检索 → 匹配度评估 → 最优方案推荐
```

**阶段三：战略建议（水·汇聚）**
```
政策匹配结果 → 区位优劣势分析 → 实施路径设计 → 风险预警
```

### 2.3 输出规范

**标准输出物**：
| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 区位评估报告 | Markdown | SWOT分析+政策匹配度评分 |
| 政策机遇清单 | 表格 | 政策名称+适用条件+申请路径+截止日期 |
| 实施路径图 | 流程图 | 分阶段行动计划+里程碑 |
| 风险预警 | 清单 | 政策变化风险+竞争风险+合规风险 |

### 2.4 反馈机制

- 政策建议的申请成功率跟踪
- 政府对接的实际效果反馈
- 政策环境变化的及时更新

### 2.5 故障处理

| 故障类型 | 识别信号 | 处理机制 |
|----------|----------|----------|
| 政策已过期 | 客户反馈申请被拒 | 立即更新数据库，提供替代方案 |
| 信息不准确 | 与官方文件不符 | 核实来源，修正并注明时效 |
| 需求理解偏差 | 建议与实际情况不符 | 重新沟通，细化需求分析 |

```yaml
skill:
  name: 地理战略与政策咨询
  description: 基于粤港澳大湾区政策研究，为硬科技创业提供区位战略评估与政府资源对接建议

  input:
    required:
      - 项目基本情况
      - 发展需求清单

      - 区位评估报告

  examples:
    - input: "AI芯片初创团队，需要研发空间与政府补贴，倾向深圳"
      output: "推荐前海或河套合作区，可申请XX补贴，匹配度85%..."
```

### 4.2 使用示例

**示例：深港区位战略评估**
```
用户输入：
"我们是机器人初创团队，已获天使轮融资。CTO常驻香港（港科大背景），
CEO和工程团队在深圳。需要扩大团队（计划招20人），申请政府研发补贴。
请问应该如何布局？"

专家输出：
【深港区位战略评估报告】

一、区位优劣势分析

深圳布局：
✓ 工程人才储备丰富，招聘相对容易
✓ 产业链配套完善（大疆、优必选等生态）
✓ 政府补贴力度大（研发资助最高1000万）
✗ 香港核心人才跨境通勤成本高
✗ 部分高端人才（如海外AI博士）偏好香港

香港布局：
✓ 对接国际资本与人才便利
✓ 港科大等高校产学研协同
✓ 进口设备与零部件通关便利
✗ 办公成本高（深圳3-5倍）
✗ 招聘工程师困难，人力成本高
✗ 政府直接补贴较少

二、推荐方案：双总部架构

【深圳总部】（工程与运营中心）
- 选址建议：南山区（科技园/前海）或宝安区（成本低）
- 功能：产品研发、生产制造、国内销售
- 规模：15-18人（工程师为主）

【香港节点】（国际与研发中心）
- 选址建议：科学园或数码港
- 功能：国际合作、融资对接、前沿研发
- 规模：3-5人（CTO+核心算法）

三、政策机遇清单

┌─────────────────┬────────────┬──────────┬─────────────────────┐
│ 政策名称        │ 适用条件   │ 资助额度 │ 申请截止            │
├─────────────────┼────────────┼──────────┼─────────────────────┤
│ 深圳科创委创业资助 │ 成立<3年   │ 最高100万 │ 每年6月/12月        │
│ 前海深港合作区补贴 │ 港资背景   │ 租金补贴50% │ 常年受理            │
│ 香港科技券计划    │ 香港注册   │ 最高60万港币 │ 常年受理            │
│ 河套合作区专项资金 │ 符合产业方向 │ 最高500万 │ 2026年6月30日       │
└─────────────────┴────────────┴──────────┴─────────────────────┘

四、实施路径

Phase 1（1-2月）：
- 注册深圳公司（若尚未注册）
- 申请前海或河套合作区入驻
- 启动香港科技券申请

Phase 2（3-6月）：
- 深圳办公室装修入驻
- 大规模招聘（利用深圳人才优势）
- 申请深圳科创委资助

Phase 3（6-12月）：
- 香港设立代表处或子公司
- 申请河套合作区专项资金
- 建立深港两地协同机制

五、风险预警

⚠️ 政策变动风险：河套合作区政策细则可能调整，建议密切关注
⚠️ 人才流动风险：深港两地通勤不便，核心人才需明确长期定位
⚠️ 合规风险：跨境数据传输、税务安排需专业咨询
```

## S7: 对抗验证（失效场景测试）

### 7.1 失效场景识别

**场景一：政策突变**
```
失效模式：政策突然调整或取消
识别信号：官方发布政策变更通知
熔断机制：立即标记相关政策为"待核实"，暂停推荐
```

**场景二：信息过时**
```
失效模式：政策已过期或申请窗口关闭
识别信号：客户反馈申请失败
熔断机制：建立政策时效性标注，超过3个月自动提醒复核
```

### 7.2 熔断机制

- 政策信息超过3个月未更新 → 自动标注"需核实"
- 客户投诉政策信息错误 → 暂停相关推荐，人工复核
