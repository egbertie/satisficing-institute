## S2: 系统考虑（输入→处理→输出→反馈闭环）

### 2.1 输入规范

**必需输入**：
| 输入项 | 格式 | 说明 |
|--------|------|------|
| 评估数据集 | CSV/JSON | 历史合伙人评估数据，含结果标签 |
| 决策约束条件 | 结构化文本 | 时间限制、信息完整度、资源约束 |
| 目标变量定义 | 枚举值 | 评估目标（匹配度/冲突风险/满意度） |

**可选输入**：
- 外部基准数据集
- 行业统计参数
- 先验概率分布假设

**输入质量要求**：
- 样本量：n ≥ 30（统计显著性基本要求）
- 数据完整性：缺失率 < 10%
- 数据标准化：已进行归一化或标准化处理

### 2.2 处理流程

**阶段一：数据探索（金·采集）**
```
原始数据 → 描述性统计 → 分布检验 → 相关性分析 → 数据质量报告
```
- 计算均值、中位数、标准差等描述统计量
- 检验正态性、异方差性等分布特征
- 分析变量间相关性，识别共线性问题

**阶段二：模型构建（金·精炼）**
```
特征工程 → 模型选择 → 参数估计 → 模型验证 → 算法设计
```
- 基于贝叶斯理论或频率学派方法构建模型
- 使用交叉验证评估模型性能
- 设计计算高效的算法实现

**阶段三：结果解释（金·锻造）**
```
模型输出 → 置信区间计算 → 显著性检验 → 业务解释 → 决策建议
```
- 提供点估计与区间估计
- 标注统计显著性水平（p值）
- 将数学结果转化为业务语言

### 2.3 输出规范

**标准输出物**：
| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 数学模型文档 | LaTeX/Markdown | 公式推导+参数说明+约束条件 |
| 算法伪代码 | 代码块 | 时间复杂度+空间复杂度分析 |
| 统计验证报告 | 表格+图表 | 信度效度指标+置信区间 |
| 决策阈值建议 | 数值 | 基于统计的满意标准量化值 |

**输出质量标准**：
- 数学严谨性：公式推导完整，边界条件明确
- 计算可行性：算法复杂度可控，可工程化实现
- 结果可解释性：统计术语附带业务含义解释

### 2.4 反馈机制

**模型性能反馈**：
- 预测准确率与真实结果的对比
- 模型在新数据上的泛化能力
- 计算效率与资源消耗

**业务价值反馈**：
- 决策建议的采纳率
- 采纳后的实际效果
- 客户满意度评分

### 2.5 故障处理

**输出偏差类型**：
| 故障类型 | 识别信号 | 处理机制 |
|----------|----------|----------|
| 过拟合 | 训练集表现好，测试集差 | 增加正则化，简化模型 |
| 数据不足 | 样本量<n=30 | 标注置信度限制，建议增加数据 |
| 假设 violated | 分布检验不通过 | 改用非参数方法 |
| 计算 overflow | 数值异常 | 检查数值稳定性，改用稳健算法 |

### 4.2 使用示例

**示例：PFI评分卡算法优化**
```
用户输入：
"现有PFI评分卡使用简单加权平均，但客户反馈区分度不够。
历史数据显示：匹配度评分3.5以上的合伙人，6个月内分手率仍有25%。
请设计更精准的算法。"

专家输出：
【PFI评分卡算法优化方案 V2.0】

一、问题诊断
原模型：PFI = Σ(wi × xi)  (i=1~5, Σwi=1)
问题：线性加权假设各维度独立，未考虑交互效应；阈值固定为3.0，未考虑基率

二、新模型设计
采用逻辑回归+贝叶斯更新的混合模型：

P(成功|X) = 1 / (1 + e^-(β₀ + Σβᵢxᵢ + Σβᵢⱼxᵢxⱼ))

其中：
- X = [技术理解力, 阶段匹配度, 文化适配性, 风险共担, 市场资源]
- 增加交互项：技术理解力×阶段匹配度，文化适配性×风险共担
- β参数通过历史数据MLE估计

三、动态阈值设定
基于贝叶斯决策理论，设定满意阈值：

θ* = argmin_θ [C₁×P(成功)×I(评分<θ) + C₂×P(失败)×I(评分≥θ)]

其中C₁、C₂分别为漏报与误报成本，根据客户风险偏好调整

四、统计验证（基于历史数据回测）
┌──────────────┬─────────┬─────────┬─────────┐
│ 指标         │ 原模型  │ 新模型  │ 提升    │
├──────────────┼─────────┼─────────┼─────────┤
│ AUC-ROC      │ 0.68    │ 0.82    │ +20.6%  │
│ 精确率       │ 0.61    │ 0.79    │ +29.5%  │
│ 召回率       │ 0.73    │ 0.81    │ +11.0%  │
│ F1分数       │ 0.66    │ 0.80    │ +21.2%  │
└──────────────┴─────────┴─────────┴─────────┘

五、实施建议
1. 收集至少100组新数据重新训练
2. 设置A/B测试，对比新旧模型效果
3. 每季度更新模型参数
```

### 4.3 限制与边界

- 相关≠因果，模型仅描述统计关联

## S6: 认知谦逊（局限标注）

### 6.1 知识局限

**统计局限**：
- 所有模型基于历史数据，对未来预测存在固有不确定性
- 小样本情况下统计功效不足
- 模型假设（如独立性、正态性）在现实中常被违反

**方法局限**：
- 量化模型难以捕捉非结构化因素（如气场、直觉）
- 统计显著≠实际显著
- 相关关系≠因果关系

### 6.2 不能替代的领域

- 商业直觉与经验判断
- 伦理与人文价值考量
- 具体行业深度洞察
- 个体心理特质分析

### 6.3 人工复核触发条件

- 样本量不足（n<30）
- 模型预测与专家判断严重偏离
- 涉及重大决策的模型输出

## S7: 对抗验证（失效场景测试）

### 7.1 失效场景识别

**场景一：数据稀疏**
```
失效模式：新行业/新场景缺乏历史数据
识别信号：样本量n<10
熔断机制：降级为专家经验判断，标注高不确定性
```

**场景二：分布漂移**
```
失效模式：市场环境变化导致历史模式失效
识别信号：模型预测准确率持续下降
熔断机制：触发模型重训练，暂停自动决策
```

**场景三：过度拟合**
```
失效模式：模型在训练集表现好，实际应用差
识别信号：训练准确率>>验证准确率
熔断机制：简化模型，增加正则化
```

### 7.2 熔断机制

**L1预警**：置信区间宽度>0.3
**L2警报**：样本量<20
**L3熔断**：预测概率接近0.5（高度不确定）
