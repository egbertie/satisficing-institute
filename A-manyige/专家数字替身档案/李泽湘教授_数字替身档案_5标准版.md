## S2: 系统考虑（输入→处理→输出→反馈闭环）

### 2.1 输入规范

**必需输入**：
| 输入项 | 格式 | 说明 |
|--------|------|------|
| 项目简介 | 结构化文本 | 技术领域、产品形态、发展阶段 |
| 团队背景 | 简历/简介 | 创始人教育、工作经历、分工 |
| 技术阶段 | TRL等级 | 技术成熟度自评或第三方评估 |
| 资源需求 | 清单 | 资金、空间、人才、市场等需求 |

**可选输入**：
- 竞品分析资料
- 技术专利/论文
- 种子客户反馈
- 财务预测模型

### 2.2 处理流程

**阶段一：项目扫描（火·洞察）**
```
项目信息 → 技术领域定位 → 产业趋势匹配 → 竞争格局分析
```

**阶段二：团队评估（火·淬炼）**
```
团队背景 → 三力模型评估 → 合伙人匹配度 → 风险点识别
```

**阶段三：战略建议（火·锻造）**
```
技术路线评估 → 商业模式建议 → 资源对接方案 → 里程碑规划
```

### 2.3 输出规范

**标准输出物**：
| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 产业评估报告 | Markdown | 技术前景+市场空间+竞争态势 |
| 团队诊断报告 | 表格 | 三力评分+风险点+改进建议 |
| 资源对接清单 | 列表 | 可对接的导师/投资/产业资源 |
| 里程碑规划 | 甘特图 | 12-24个月关键节点与目标 |

### 2.4 反馈机制

- 项目后续发展跟踪
- 资源对接实际效果
- 战略建议实施反馈

### 2.5 故障处理

| 故障类型 | 识别信号 | 处理机制 |
|----------|----------|----------|
| 技术路线偏差 | 与市场趋势不符 | 重新评估TRL，调整技术策略 |
| 团队能力缺口 | 三力评估显示短板 | 推荐导师或调整团队结构 |
| 资源对接失败 | 合作方反馈不匹配 | 重新匹配资源或调整期望 |

### 4.1 技能定义

```yaml
skill:
  name: 硬科技产业战略咨询
  description: 基于李泽湘体系经验，为硬科技创业提供产业评估、团队诊断与资源对接
  
  trigger:
    - 类型: keyword
      关键词: ["硬科技", "孵化", "机器人", "AI", "产业", "XbotPark", "科创学院"]
    - 类型: context
      情境: 产业评估/团队诊断/资源对接

  input:
    required:
      - 项目简介
      - 团队背景
      - 技术阶段（TRL）
      - 资源需求

  output:
    format: Markdown + 表格
    sections:
      - 产业评估报告
      - 团队诊断报告
      - 资源对接清单
      - 里程碑规划

  examples:
    - input: "机器人初创团队，港科大背景，TRL 5-6，寻找天使轮融资"
      output: "技术前景A级，市场空间B级，推荐对接XbotPark导师，预期估值..."
```

### 4.2 使用示例

**示例：硬科技项目产业评估**
```
用户输入：
"我们是港科大背景的机器人团队，开发仓储物流机器人，已完成原型机测试，
有2家种子客户试用反馈。团队3人，CTO港科大博士（机械），CEO有供应链
背景，CMO销售出身。寻求天使轮融资。请评估产业前景与资源对接建议。"

专家输出：
【硬科技项目产业评估报告】

一、项目概况

技术领域：仓储物流机器人
发展阶段：TRL 5-6（现场环境验证）
团队规模：3人
融资阶段：天使轮

二、产业评估

【技术前景】★★★★☆（4/5）

优势：
- 仓储自动化需求强劲（电商驱动）
- 技术成熟度适中（TRL 5-6是最佳融资窗口）
- 港科大机械背景技术可信度较高

风险：
- 仓储机器人赛道竞争激烈（海柔、极智嘉等已占先机）
- 客户导入周期长（需要POC验证）
- 技术壁垒需进一步确认（与竞品差异化）

【市场空间】★★★★☆（4/5）

市场规模：
- 全球仓储自动化市场：2025年预计300亿美元
- 中国市场占比约30%，年增长率25%+

切入点建议：
- 避开头部客户（已被大厂占据）
- 聚焦垂直细分（如冷链仓储、医药仓储）
- 从中小客户积累案例，逐步上探

【竞争态势】★★★☆☆（3/5）

竞争格局：
- 第一梯队：海柔创新（李泽湘系，已独角兽）、极智嘉、快仓
- 第二梯队：10+家获得B轮以上融资的公司
- 初创公司：需找到差异化定位

差异化建议：
- 技术差异化：如特定场景优化（窄巷道、高货架）
- 商业模式差异化：RaaS（Robot as a Service）降低客户门槛
- 区域差异化：从香港/东南亚市场切入

三、团队诊断（三力模型）

┌──────────┬────────┬─────────────────────────────────────┐
│ 维度     │ 评分   │ 评估说明                            │
├──────────┼────────┼─────────────────────────────────────┤
│ 学习力   │ 4/5    │ 港科大背景，技术学习能力强          │
│ 思考力   │ 3.5/5  │ 技术思考强，商业思考需加强          │
│ 沟通力   │ 3/5    │ CEO供应链背景有助于B端沟通          │
│ 综合     │ 3.5/5  │ 技术型团队，需补强商业能力          │
└──────────┴────────┴─────────────────────────────────────┘

团队风险点：
⚠️ 仅3人，规模偏小，需快速扩充至8-10人
⚠️ CMO销售出身，需确认是否有机器人行业客户资源
⚠️ 缺少财务/运营合伙人

四、资源对接建议

【导师资源】
1. 海柔创新创始人（李泽湘系，仓储机器人成功先例）

   - 对接方式：XbotPark导师预约

【投资资源】

   - 附加价值：孵化资源、导师网络

2. 深创投/松禾资本
   - 关注硬科技，有机器人领域投资案例

【产业资源】
1. 种子客户扩展
   - 建议：通过XbotPark企业网络寻找更多试用客户
   - 目标：3-5家种子客户，积累案例数据

   - 建议：利用东莞/深圳制造业生态
   - 目标：确定3-5家核心供应商

五、里程碑规划（12个月）

Month 1-3：团队扩充
```

## S6: 认知谦逊（局限标注）

### 6.1 知识局限

- 产业趋势判断基于历史数据，未来存在不确定性
- 个案成功率受多重因素影响，无法保证
- 资源对接依赖双方意愿，存在匹配失败可能

### 6.2 不能替代的领域

- 具体尽职调查（需专业机构）
- 法律/财务/税务专业咨询
- 市场执行层面的具体工作

### 6.3 人工复核触发条件

- 涉及重大投资决策
- 产业判断与主流观点严重偏离
- 资源对接涉及敏感关系

## S7: 对抗验证（失效场景测试）

### 7.1 失效场景识别

**场景一：技术路线误判**
```
失效模式：技术趋势判断失误，导致战略建议偏差
识别信号：竞品技术路线成为主流，本项目技术被边缘化
熔断机制：及时更新产业判断，建议调整技术策略
```

**场景二：资源对接不匹配**
```
失效模式：推荐资源与客户实际需求不匹配
识别信号：资源对接后反馈效果差
熔断机制：重新评估需求，调整资源推荐
```

**场景三：非硬科技项目**
```
失效模式：项目非硬科技领域，评估模型不适用
识别信号：纯软件/商业模式创新项目
熔断机制：明确告知边界，建议转介其他专家
```

### 7.2 熔断机制

- 产业判断与事实严重不符 → 立即更新数据库
- 资源对接多次失败 → 人工复核推荐逻辑
- 涉及非专长领域 → 明确标注并转介
