# 飞书API接入技术问题求助

> 各位大神好，我们是刚接触飞书开放平台的初学者，在接入过程中遇到了几个问题，想请教一下社区里的前辈们。

## 背景介绍

我们正在做一个AI助手项目，希望通过飞书API实现本地Markdown文档自动同步到飞书云文档/多维表格。

- **应用类型**：企业内部应用（已创建）
- **已申请权限**：drive:file:read/write, docx:document:read/write
- **开发环境**：Python + requests
- **目标**：把本地知识库自动同步到飞书，方便团队协作

## 遇到的问题

### 问题1：API创建的文档用户看不到（权限隔离？）

**现象**：
- 通过API成功创建了文档（返回200，有document_id）
- 文档确认存在（可以查询到详情）
- **但是**：在飞书云文档界面里找不到这个文档
- 文档URL访问也看不到

**我们的代码**：
```python
POST /docx/v1/documents
Body: {"title": "测试文档"}
返回：成功，document_id: xxx
```

**猜测**：
- 应用创建的文档默认只有应用自己可见？
- 需要设置某个权限才能让企业内用户看到？
- 还是在飞书管理后台有可见性设置？

**已尝试**：
- 查看云文档"我的文件" → 没有
- 查看"满意解研究所"文件夹 → 没有
- 直接访问文档URL → 看不到

---

### 问题2：多维表格写入权限不足（Forbidden）

**现象**：
- 可以读取多维表格信息（list tables成功）
- 但是写入记录时返回 **403 Forbidden**
- 错误信息：没有权限

**我们的代码**：
```python
POST /bitable/v1/apps/{base_id}/tables/{table_id}/records
返回：403 Forbidden
```

**已尝试**：
- 申请bitable相关权限（已开通）
- 尝试添加应用为协作者 → 无法添加（应用不是用户）

**问题**：
- 企业应用如何获得多维表格的写入权限？
- 是否需要表格所有者在后台设置什么？

---

### 问题3：文档导入不支持HTML格式

**现象**：
- 飞书导入界面支持的格式：Word、Excel、PPT、Markdown、CSV、XMind等
- **没有HTML格式选项**

**我们的场景**：
- 本地是Markdown文件
- 想转为飞书文档
- 尝试用Python markdown库转HTML → 但飞书不支持导入HTML

** workaround尝试**：
- 直接上传HTML到云盘（可行，但不是在线文档）
- 需要保留格式和结构

**问题**：
- Markdown转飞书文档的最佳实践是什么？
- 有没有批量导入Markdown的方法？

---

### 问题4：应用无法被添加为协作者

**现象**：
- 尝试在多维表格/文档中添加协作者
- 搜索应用名称（Kimi决策助手-满意妞）→ 搜不到
- 应用不是"用户"，无法像普通用户一样被添加

**问题**：
- 企业应用应该如何获得对资源的访问权限？
- 只能通过API权限申请，不能通过协作者方式？

本地Markdown文件 → 飞书云文档/多维表格

不需要很复杂的功能，就是能把文档内容同步过去，团队能在飞书里查看和编辑。
