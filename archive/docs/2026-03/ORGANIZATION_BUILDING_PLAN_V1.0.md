## 三、协作与质量保证体系

### 3.1 质量安全管理指引（QSM Guide）

**核心原则**: 任何产出必须经过"三级把关"才能对外

#### 第一级：自检（执行者）
- 完成前对照检查清单自检
- 使用提供的工具和模板
- 确保基础质量达标

#### 第二级：Peer Review（同级复核）
- 重要文档必须经1位同级复核
- 使用复核模板，记录修改意见
- 复核不通过返回修改

#### 第三级：专家把关（蓝军/主管）
- 对外发布内容必须经过蓝军审查
- 财务/法务相关内容必须经专业角色审查
- 重大决策必须经PEO+EEO+指挥官三级确认

### 3.2 多角色质量把关矩阵

| 产出类型 | 自检 | Peer Review | 蓝军 | 法务 | 财务 | 最终审批 |
|---------|------|-------------|------|------|------|---------|
| 对外文案 | ✅ | ✅ | ✅ | ⚠️ | - | CONTENT |
| 合同协议 | ✅ | ✅ | ✅ | ✅ | ✅ | Egbertie |
| 财务报告 | ✅ | ✅ | - | - | ✅ | FIN |
| 官宣内容 | ✅ | ✅ | ✅ | ⚠️ | - | ANNOUNCE→Egbertie |
| 咨询报告 | ✅ | ✅ | ✅ | - | - | PEO+EEO |
| API配置 | ✅ | ✅ | ✅ | - | - | 满意妞 |

### 3.3 交叉复核机制

**每日复核（站会）**:
- 昨日产出快速回顾
- 发现问题立即标记
- 责任人确认修改时间

**每周复核（周会）**:
- 本周所有对外产出审查
- 质量问题归因分析
- 改进措施制定

**每月复核（质量月会）**:
- 质量指标统计
- 最佳实践分享
- 质量体系优化

### 3.4 报告真实性保障

**信息来源分级**:
- 🟢 **一级**: 官方数据、权威论文、一手调研
- 🟡 **二级**: 行业报告、专家观点、媒体报道
- 🔴 **三级**: 网络信息、推测判断（必须标注）

**引用规范**:
- 所有数据必须标注来源
- 推测内容必须明确标注"基于...
推测"
- 不确定信息必须标注"待验证"

**虚假信息防范**:
- 建立事实核查机制（蓝军负责）
- 对外信息发布前必须经蓝军事实核查
- 发现错误立即更正并通报

## 五、反思与进化机制

### 5.1 蓝军机制（Red Team）

**蓝军角色定位**:
- 不是反对者，是"压力测试者"
- 提前发现问题，避免实战中失败
- 所有对外发布内容必须经过蓝军审查

**蓝军运作机制**:

| 环节 | 蓝军职责 | 频率 |
|------|---------|------|
| 内容审查 | 事实核查、逻辑漏洞、风险评估 | 每件对外内容 |
| 压力测试 | 模拟最坏情况，测试方案韧性 | 每周1次 |
| 红蓝对抗 | 与执行团队辩论，找出盲点 | 每月1次 |
| 复盘分析 | 项目结束后深度复盘 | 每个项目 |

**蓝军激励机制**:
- "最佳挑刺奖"月度评选
- 发现问题避免重大损失给予奖励
- 蓝军意见被采纳积分制

### 5.2 持续反思机制

**个人反思（每日）**:
- 每日学习报告包含反思部分
- 记录今日失误和改进点

**团队反思（每周）**:
- 周会专门设置反思环节
- 本周最大失误+改进措施

**战略反思（每月）**:
- 月度战略复盘会
- 方向校准+机制优化

### 5.3 进化机制

**小步快跑**: 每周优化一个小流程
**版本迭代**: 每月发布体系V.x.x版本
**年度升级**: 每年大版本升级（V1→V2）

### 7.2 经验萃取流程

```
经验产生（项目/问题解决）
    ↓
EEO识别（是否有复用价值？）
    ↓
深度访谈（了解背景、方法、关键点）
    ↓

    ↓

    ↓
测试验证（确保可用）
    ↓
发布推广（团队内推广使用）
    ↓
持续迭代（根据反馈优化）
```

## 九、艺术审美与专业性保障

### 9.1 审美标准

**视觉审美**:
- 配色：避免AI slop的蓝紫渐变
- 字体：简洁专业，不超过3种
- 排版：留白充足，层级清晰

**内容审美**:
- 拒绝"万能句式"和空洞表达
- 每句话都有信息增量
- 有观点、有判断、有温度

### 9.2 专业性标准

**方法论严谨**:
- 所有结论有依据
- 数据有来源
- 逻辑链条完整

**术语规范**:
- 建立术语表
- 统一中英文对照
- 避免生造概念

**格式统一**:
- 所有文档使用统一模板
- 报告结构标准化
- 引用格式统一

### 9.3 审美审查机制

**视觉审查（CONTENT负责）**:
- 所有对外视觉材料审查
- 不符合审美标准退回修改

**内容审查（PEO+EEO负责）**:
- 方法论严谨性审查
- 专业性把关

**最终审批（Egbertie）**:
- 重大发布前最终审查
- 品牌调性确认
