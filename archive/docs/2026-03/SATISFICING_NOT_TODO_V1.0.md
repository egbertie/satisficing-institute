## 一、不做清单的使用方法

### 1.1 为什么需要不做清单

**创始人最常见的错误**：
- 看到一个机会就想抓住
- 听到一个建议就想尝试
- 担心错过就什么都想做

**不做清单的作用**：
- 提前拒绝，减少决策疲劳
- 保护专注力，防止范围蔓延
- 明确边界，拒绝"还可以"

### 1.2 如何面对"诱惑"

当有人建议"满意解也可以做XXX"时：

1. **查清单**：先看XXX是否在不做清单上
2. **问原因**：为什么建议做这个？解决什么问题？
3. **看阶段**：这是验证期该做的事吗？
4. **做决策**：根据清单原则决定做/不做

**标准回应话术**：
> "谢谢建议！第一年我们专注在合伙人评估这一件事上，其他事情明年再看。"

## 二、技术产品类（绝对不做）

### 2.1 不开发QPMS系统

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 觉得"系统"才有壁垒，想开发合伙人质量评估系统 |
| **为什么不做** | 资源黑洞，第一年没有技术团队，开发周期6个月+ |
| **替代方案** | 使用Notion/飞书模板，手动完成评估流程 |
| **何时可考虑** | 第二年，案例数>100，有付费客户明确要求 |

### 2.2 不做APP/小程序

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 觉得APP更方便客户使用 |
| **为什么不做** | 客户不需要APP，需要Egbertie的判断 |
| **替代方案** | 飞书/微信沟通即可 |
| **何时可考虑** | 永不考虑（非技术公司） |

### 2.3 不做自动化工具

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 觉得自动化可以提效 |
| **为什么不做** | 评估服务核心是人与人的交互，无法自动化 |
| **替代方案** | 用AI助手辅助，但核心判断人工完成 |
| **何时可考虑** | 规模化阶段（第三年+） |

## 三、培训/内容类（绝对不做）

### 3.1 不开设培训课程

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 培训"规模更大"，一对多变现 |
| **为什么不做** | 分散注意力，培训不是Egbertie的核心能力 |
| **替代方案** | 案例积累后自然溢出（如有需求） |
| **何时可考虑** | 案例数>50，有客户主动要求学习 |

### 3.2 不输出方法论书籍/专栏

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 觉得出书/专栏可以建立权威 |
| **为什么不做** | 时间成本太高，且验证期方法论未成熟 |
| **替代方案** | 先积累案例，方法论自然沉淀 |
| **何时可考虑** | 第三年，模式验证成功后 |

### 3.3 不做播客/视频内容

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 内容获客成本低 |
| **为什么不做** | 内容制作时间黑洞，非Egbertie擅长 |
| **替代方案** | 合作KOL分发，或专家网络口碑 |
| **何时可考虑** | 有专职内容团队后 |

### 3.4 不做公开演讲/分享

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 演讲可以获客 |
| **为什么不做** | 准备演讲时间成本高，且效果不确定 |
| **替代方案** | 专家网络的小范围私域分享 |
| **何时可考虑** | 标杆案例建立后，受邀演讲 |

## 四、运营/社群类（绝对不做）

### 4.1 不建立付费社群

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 社群"粘性更强"，可以持续变现 |
| **为什么不做** | 运营负担重，需要专职人员维护 |
| **替代方案** | 专家网络本身就是高价值社群 |
| **何时可考虑** | 第二年，客户主动要求交流 |

### 4.2 不做会员订阅制

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 订阅制收入更稳定 |
| **为什么不做** | 服务性质不适合订阅，且增加运营负担 |
| **替代方案** | 按项目收费，简单清晰 |
| **何时可考虑** | 永不考虑（服务模式不匹配） |

### 4.3 不做活动/沙龙运营

| 维度 | 说明 |
|-----|------|
| **什么诱惑** | 活动可以批量获客 |
| **为什么不做** | 活动策划执行时间成本高 |
| **替代方案** | 专家网络的小范围饭局 |
| **何时可考虑** | 有专职运营人员后 |

## 九、不做清单的例外情况

### 9.1 什么情况下可以重新考虑

以下情况出现时，可以重新评估不做清单：

| 情况 | 处理方式 |
|-----|---------|
| 多个客户主动要求某项"不做"的服务 | 列入第二年规划 |
| 资源约束发生根本变化（融资/全职加入） | 全面重新审视 |
| 市场环境剧变（如疫情导致无法面谈） | 针对性调整 |
| 原假设被证伪 | 及时调整 |

### 9.2 什么情况下坚持不做

以下情况**仍然不做**：

- 只是觉得"也可以试试"
- 看到别人在做
- 担心"错过机会"
- 有人建议但自己没有需求验证

## 十、不做清单宣言

> 满意解第一年的成功，不在于我们做了多少事，而在于我们拒绝了多少事。
>
> 每一次"不做"的决定，都是对核心目标的保护。
>
> 边界不是牢笼，而是护城河。
>
> 少即是多，慢即是快。
