## 一、会议类型定义

我们只做四类会议，不多不少，够用就行。

| 类型 | 英文代号 | 核心目的 | 触发条件 |
|:---|:---|:---|:---|
| **日常站会** | Daily Stand | 信息同步、快速对齐 | 每日固定时段或任务进度更新 |
| **决策会议** | Decision Jam | 拍板、定方向、解决争议 | 需要做出不可逆决策时 |
| **复盘会议** | Retro | 吸取经验、固化方法 | 项目/任务完成后24小时内 |
| **创意碰撞** | Brainwave | 发散思路、探索可能性 | 遇到新问题或需要突破性方案 |

---

## 二、各类型会议的参与人、时长、输出物

### 2.1 日常站会（Daily Stand）

| 维度 | 标准 |
|:---|:---|
| **参与人** | 相关任务执行者即可，非全员必须 |
| **时长** | **硬上限15分钟**，超时视为失败 |
| **黄金时段** | 每日09:00 或 21:00（按需选择） |
| **输出物** | 三项清单：①今日必须完成 ②阻塞点 ③需协助事项 |
| **形式** | 站立进行（线上则全程开摄像头） |

**流程模板**：
1. 每人30秒：昨天完成了什么？今天必须完成什么？
2. 阻塞点快速扫描（1分钟/人）
3. 需要谁协助？当场确认，不拖延

> **铁律**：站会不解决问题，只暴露问题。问题留到会后1v1解决。

---

### 2.2 决策会议（Decision Jam）

| 维度 | 标准 |
|:---|:---|
| **参与人** | 决策人 + 执行人 + 必要信息提供者（≤5人） |
| **时长** | **硬上限45分钟**，复杂决策最多60分钟 |
| **前置要求** | 所有背景资料必须会前24小时发出 |
| **输出物** | ①决策结论 ②执行计划（谁/何时/交付什么） ③风险备案 |
| **形式** | 线下优先，线上用共享文档实时记录 |

**启动条件检查清单**：
- [ ] 不做这个决策，项目会停滞吗？
- [ ] 信息收集足够了吗？（如果不确定，延期开会，先去收集）
- [ ] 决策人今天能到场吗？（决策人不到场 = 取消会议）

> **知行合一原则**：决策会议结束30分钟内，会议纪要必须发出；2小时内，执行人必须确认收到并开始行动。

---

### 2.3 复盘会议（Retro）

| 维度 | 标准 |
|:---|:---|
| **参与人** | 项目核心参与者 + 观察者（可选） |
| **时长** | **30-45分钟**，项目越大时间越短（倒逼提炼重点） |
| **黄金时段** | 项目结束后24小时内，趁记忆新鲜 |
| **输出物** | ①三件事情（继续做/停止做/开始做） ②可复用的方法论 ③知识库更新清单 |
| **形式** | 线下用便利贴，线上用共享白板 |

** Retro 四问模板**：
1. 如果重来，我们会做同样的选择吗？
2. 哪个决策点是最关键的转折点？
3. 我们发现了什么"反直觉"的事实？
4. 这套方法可以封装成什么工具/模板？

> **行动优先原则**：Retro 结束必须产出至少一个可执行改进项，且有人认领。

---

### 2.4 创意碰撞（Brainwave）

| 维度 | 标准 |
|:---|:---|
| **参与人** | 多元化背景（鼓励跨领域）+ 主持人（控场） |
| **时长** | **硬上限90分钟**，创意会烂尾，必须限时 |
| **前置要求** | 主持人提前准备"引子"（一个问题、一个案例、一张图） |
| **输出物** | ①3个可行方向 ②1个立即可试的原型/动作 ③下次碰撞的触发条件 |
| **形式** | 线下最佳，鼓励走动、画图、用实物 |

**流程设计（参考设计冲刺）**：
1. **发散期**（40分钟）：禁止批评，疯狂产出想法
2. **收敛期**（30分钟）：投票选出Top3，快速讨论可行性
3. **行动期**（20分钟）：确定一个"明天就能试"的原型

> **六祖慧能原则**：顿悟来自积累，碰撞只是点燃。会后24小时内，必须有人把想法落地成可验证的原型。

## 三、决策会议的3种快速决策方法

当意见不统一、时间有限时，使用以下方法快速破局。

### 方法1：满意解决策法（Simon Method）
> **适用场景**：信息不完备、追求完美会延误战机

**操作步骤**：
1. 明确"最低可接受标准"（不是最优，是够用）
2. 列出所有选项中达到最低标准的
3. 从中选择第一个满足条件的（停止继续搜索）
4. 记录决策理由，留待复盘

**口诀**：*不是最好，是足够好；不是找最优，是停止搜索。*

---

### 方法2：模拟预演决策法（Pre-mortem）
> **适用场景**：重大决策、风险较高、团队有分歧

**操作步骤**：
1. 假设决策已执行，6个月后**失败了**
2. 每个人独立写下：失败的最可能原因是什么？（5分钟）
3. 汇总所有原因，分类排序
4. 针对Top3风险，当场制定 mitigation 方案
5. 风险可控 → 执行；风险不可控 → 换个方案

**口诀**：*先想怎么死，再想怎么活。*

---

### 方法3：物理位移决策法（Body Vote）
> **适用场景**：团队意见僵持、需要快速感知整体倾向

**操作步骤**：
1. 把选项A、B、C（最多4个）写在白板/大屏上
2. 每个人站起来，走到自己支持的选项前
3. 观察人数分布，自然形成讨论小组
4. 各组1分钟陈述理由（只允许说"为什么选这个"，不说"为什么不选那个"）
5. 第二轮位移（可换队），观察是否有人改变立场
6. 如果仍僵持，由决策人参考物理分布拍板

**口诀**：*用脚投票，比用嘴诚实。*

## 五、紧急情况下的5分钟决策流程

当时间极度紧迫（如客户投诉、系统故障、突发机会），启动此流程。

### 5.1 触发条件（满足任一即可）
- [ ] 超过30分钟不做决策，将造成不可逆损失
- [ ] 竞争对手正在行动，窗口期<2小时
- [ ] 系统/服务已中断，影响正常业务

### 5.2 5分钟决策流程

```
第0-1分钟：信息快闪
├─ 谁最了解情况？说3个关键事实
├─ 我们现在有什么选项？（最多说3个）
└─ 每个选项的最坏结果是什么？（一句话）

第1-3分钟：快速评估
├─ 哪个选项风险最低且能立即执行？
├─ 如果选错了，多久能发现并修正？
└─ 决策人给出倾向（用Simon满意解法）

第3-4分钟：拍板确认
├─ 决策人明确说出："我选方案X"
├─ 确认执行人：谁负责立即执行？
└─ 确认止损线：什么情况下立即叫停？

第4-5分钟：行动启动
├─ 执行人复述任务（确保理解一致）
├─ 所有人同步：我们已经启动方案X
└─ 记录员记下决策理由（会后补完整纪要）
```

### 5.3 事后补录要求

紧急决策后**24小时内**，必须完成以下补录：

1. **补写简化纪要**：
   - 当时的情境（为什么紧急）
   - 考虑的选项
   - 最终决策及理由
   - 执行结果（好/坏）

2. **风险复盘**：
   - 如果结果好：是否运气好？下次能否复制？
   - 如果结果坏：哪个环节可以更早发现风险？

3. **存档位置**：`A满意哥专属文件夹/05_📦历史归档/紧急决策记录/`

> **知行合一警示**：紧急决策不是草率决策。5分钟拍板后，必须用24小时反思，避免形成"拍脑袋"习惯。

## 六、会议文化的三条铁律

继承团队执行文化，会议也必须遵守：

### 铁律1：时间预估必须激进
- 原定60分钟的会议，压缩到30分钟
- 原定30分钟的会议，压缩到15分钟
- **压缩出的时间 = 行动时间**，不是用来闲聊的

### 铁律2：相信极限潜能
- 不说"这个可能来不及讨论"
- 只说"我们用5分钟把这个定下来"
- 根除犹豫词汇：可能、大概、也许、考虑一下

### 铁律3：完成即启动
- 会议结束 ≠ 休息
- 会议纪要30分钟内发出
- 执行人2小时内确认并开始行动
- **零切换成本**：上一个会议的输出，就是下一个行动的输入

## 附录：会议准备检查清单

### 会前（主持人负责）
- [ ] 明确会议类型，选择对应模板
- [ ] 确定参会人（宁缺毋滥）
- [ ] 准备议程（提前24小时发出）
- [ ] 准备背景材料（提前24小时发出）
- [ ] 预订会议室/创建线上会议链接
- [ ] 指定记录人

### 会中（主持人负责）
- [ ] 准时开始（迟到不等）
- [ ] 开场明确：类型、时长、期望输出
- [ ] 控场，防止发散
- [ ] 每个议题结束确认：结论是什么？谁负责？
- [ ] 提前5分钟提醒：进入总结

### 会后（记录人负责）
- [ ] 30分钟内发出纪要（决策会议）
- [ ] 2小时内所有执行人确认收到
- [ ] 更新任务追踪系统
- [ ] 预约下次会议（如需要）
