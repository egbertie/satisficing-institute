## 工具选型总览

### 推荐工具栈总览

```
满意解知识管理工具栈
├── 捕获层
│   ├── Zotero (文献)
│   ├── Obsidian Web Clipper (网页)
│   └── Voice Memos (语音)
├── 存储层
│   ├── Obsidian (个人知识)
│   ├── Notion/飞书文档 (团队知识)
│   └── NAS/MinIO (文件存储)
├── 处理层
│   ├── Ollama (本地LLM)
│   ├── Weaviate (向量数据库)
│   └── Neo4j (知识图谱)
├── 协作层
│   ├── Git (版本控制)
│   └── n8n (工作流自动化)
└── 展示层
    ├── MkDocs (文档站点)
    └── Streamlit (数据应用)
```

### 选型标准

| 维度 | 权重 | 评估标准 |
|------|------|---------|
| **功能性** | 30% | 是否满足核心需求 |
| **开放性** | 25% | 开源、API、数据可导出 |
| **成本** | 20% | TCO（总拥有成本） |
| **易用性** | 15% | 学习曲线、用户体验 |
| **社区** | 10% | 活跃度、插件生态 |

## 六、自研vs外购决策

### 6.1 决策框架

```
决策树：
核心功能？
├── 是 → 自研（战略投资）
└── 否 → 差异化需求？
    ├── 是 → 定制开发
    └── 否 → 通用需求？
        ├── 是 → 开源方案
        └── 否 → SaaS购买
```

### 6.2 各组件决策建议

| 组件 | 建议 | 理由 |
|------|------|------|
| **文献管理** | 外购(Zotero) | 成熟开源，无需重复造轮子 |
| **文档协作** | 外购(飞书) | 国内生态，维护成本低 |
| **知识图谱** | 混合 | Neo4j外购 + 上层应用自研 |
| **RAG系统** | 自研 | 核心差异化能力 |
| **AI助手** | 混合 | Ollama外购 + 提示词/工作流自研 |
| **文件存储** | 外购(TrueNAS) | 基础设施，稳定优先 |
| **笔记系统** | 外购(Obsidian) | 成熟产品，专注内容 |
| **工作流** | 混合 | n8n外购 + 自定义工作流 |
