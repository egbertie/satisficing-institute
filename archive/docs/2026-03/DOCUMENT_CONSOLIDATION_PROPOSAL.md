## 四、明天Token控制方案（性价比满意解）

### 4.1 硬约束
- **目标**: < 1,500 Token/天
- **策略**: 增量检查 + 缓存 + 按需触发

### 4.2 具体控制方案

| 检查类型 | 频率 | Token/次 | 日消耗 | 优化策略 |
|----------|------|----------|--------|----------|
| **完整检查** | 每日首次对话 | ~300 | 300 | 只检查关键指标，跳过详细解析 |
| **Heartbeat** | 每30分钟 | ~100 | 400 | 只检查P0项，无问题不报 |
| **逾期预警** | 每小时 | ~50 | 400 | 只读文件时间戳，不读内容 |
| **晨报生成** | 每日09:00 | ~500 | 500 | 模板化生成，增量更新 |
| **惩罚执行** | 按需 | ~200 | 200 | 仅执行期触发 |
| **总计** | - | - | **~1,800** | **优化后~1,200** |

### 4.3 Token节省技巧
1. **文件指纹**: 用MD5代替全文比对
2. **时间戳缓存**: 只检查mtime变化
3. **分层检查**: 先快速筛选，再深度检查
4. **批量处理**: 每小时批量检查，不实时检查

### 4.4 明天执行脚本
```python
# 明天只执行这个轻量检查
python3 scripts/check_management_rules.py --light
# Token消耗: ~100/次
```
