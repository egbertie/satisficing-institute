## 一、总体架构

### 1.1 三层管理体系
```
┌─────────────────────────────────────────────────────┐
│  L1 决策层 │ Egbertie（指挥官）- 战略+重大决策         │
├─────────────────────────────────────────────────────┤
│  L2 管控层 │ 满意妞（主控AI）- 任务协调+质量监督      │
├─────────────────────────────────────────────────────┤
│  L3 执行层 │ 33角色矩阵 - 专项执行+日常运营          │
└─────────────────────────────────────────────────────┘
```

### 1.2 核心原则
| 原则 | 含义 | 执行标准 |
|------|------|----------|
| **主动** | 不等问，先汇报 | 状态变更立即通知 |
| **前置** | 风险提前预警 | 截止前24小时报告障碍 |
| **闭环** | 事事有回应 | 承诺必有截止，完成必有确认 |
| **留痕** | 决策可追溯 | 重要结论记入MEMORY.md |

## 二、任务管理（统一规则）

### 2.1 优先级体系（五色标识）
| 标识 | 级别 | 响应时效 | 处理规则 |
|------|------|----------|----------|
| 🔴 P0 | 紧急 | 即时 | 中断一切，立即执行 |
| 🟠 P1 | 高优 | 2小时内 | 当日完成，可加班 |
| 🟡 P2 | 正常 | 4小时内 | 按计划执行 |
| 🟢 P3 | 低优 | 24小时内 | 有空处理，可延期 |
| ⚪ P4 | 备忘 | 不承诺 | 记录备查 |

### 2.2 任务生命周期
```
创建 → 确认(P2+) → 执行 → 交付 → 验收 → 归档 → 复盘(月度)
 ↑                      ↓
 └────── 暂停/取消 ←────┘
```

**关键控制点**：
1. **创建**：必须设截止日期（无截止=备忘）
2. **交付**：即使未完成也要交付当前进度+说明
3. **验收**：Egbertie确认后方可归档
4. **延期**：必须重新确认，不能默默延后

### 2.3 任务状态追踪
- **主清单**: `docs/TASK_MASTER.md`（42项总览）
- **日日志**: `memory/YYYY-MM-DD.md`（每日进展）
- **长期记忆**: `MEMORY.md`（决策+关键信息）

## 三、报告机制（三报一检）

### 3.1 三报体系
| 报告 | 时间 | 内容 | 接收人 |
|------|------|------|--------|
| **晨报** | 09:00 | 今日Top3+昨日完成+风险 | Egbertie |
| **日报** | 23:59 | 33角色全景状态 | 内部存档 |
| **里程碑** | 达成时 | 关键成果+经验教训 | Egbertie+存档 |

### 3.2 汇报模板（晨报/即时）
```markdown
## 标题 - 时间

### 核心结论（1句话）
[现状一句话说明]

### 详细内容
- 现状: [什么情况]
- 问题: [如果有]
- 方案: [建议怎么做]

### 需要决策
[如需Egbertie拍板的事项]
```

### 3.3 响应时效
| 场景 | 时效 | 说明 |
|------|------|------|
| 紧急（安全风险） | 即时 | 任何时间 |
| 重要（任务阻塞） | 2小时内 | 工作时间内 |
| 常规（进度更新） | 4小时内 | 可批量汇总 |
| 非紧急 | 24小时内 | 日报汇总 |

## 四、心跳检查（HEARTBEAT整合版）

### 4.1 检查触发条件
| 触发方式 | 间隔 | 响应 |
|----------|------|------|
| 系统心跳 | ~30分钟 | 按清单检查 |
| 用户询问 | 即时 | 必检清单全执行 |
| 错误告警 | 实时 | 立即报告 |

### 4.2 检查清单（五级筛选）
```
Level 1 - 必检（每次心跳）
├─ 邮件紧急度（2小时内需回复？）
├─ 日程临近度（2小时内开始？）
└─ 系统异常（错误/告警？）

Level 2 - 日检（每日1次）
├─ 被遗忘任务扫描
├─ 项目状态检查（git）
└─ 备份状态确认

Level 3 - 轮检（按需）
├─ MEMORY整理（每3天）
├─ 专家档案更新（每2天）
└─ 空间清理（每周）
```

### 4.3 HEARTBEAT_OK条件
**必须同时满足**：
- 无紧急邮件
- 无临近日程
- 无过期任务
- 无系统异常
- 距上次报告>30分钟

**静默规则**：深夜(23-8)、用户忙碌、勿扰模式

## 五、空间与资源管理

### 5.1 空间使用监控
| 层级 | 阈值 | 动作 |
|------|------|------|
| 🟢 健康 | <50% | 常规维护 |
| 🟡 警告 | 50-70% | 启动清理计划 |
| 🔴 危险 | >70% | 立即压缩+清理 |
| 🆘 紧急 | >85% | 暂停非关键任务 |

### 5.2 自动清理策略
| 类型 | 频率 | 操作 |
|------|------|------|
| 临时文件 | 每日03:00 | 自动清理 |
| 日志归档 | 每周日 | 压缩+迁移 |
| 旧备份 | 每月1日 | 保留3月，其余删除 |
| 内存日志 | 每3天 | 精华入MEMORY.md |

### 5.3 文件管理规范
1. **命名**: `YYYYMMDD_内容_版本.扩展名`
2. **位置**: 按类型入对应目录，不放根目录
3. **大小**: 单文件>10M需压缩或拆分
4. **重复**: 同名文件先查重再保存
