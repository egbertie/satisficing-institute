# 理论引入标准作业程序 (SOP)
## Theory Introduction Standard Operating Procedure

**文档版本**: V1.0  
**创建日期**: 2026-03-15  
**适用范围**: 满意解研究所新理论引入评估与实施  
**责任人**: 研究主管 + 品牌主管  

---

## 一、目的与范围

### 1.1 目的
建立标准化的新理论引入流程，确保：
- 新理论与现有体系有机整合
- 品牌一致性得到保护
- 引入过程可追溯、可评估

### 1.2 适用范围
适用于以下理论的引入决策：
- 心理学理论（如双系统理论、心理账户理论）
- 经济学理论（如博弈论、信息经济学）
- 管理学理论（如复杂系统理论、组织行为学）
- 哲学/伦理学思想（如儒商哲学扩展）
- 跨学科融合理论

### 1.3 责任分工

| 角色 | 职责 |
|------|------|
| **研究主管** | 第一性原理拆解、学术价值评估 |
| **品牌主管** | 品牌一致性检查、对外话术设计 |
| **产品主管** | 产品化可行性评估 |
| **创始人** | 最终审批（V级及以上理论引入） |

## 三、阶段1：第一性原理拆解

### 3.1 拆解清单

**必填问题**:
```
□ 这个理论的核心假设是什么？（3条以内）
□ 这个理论的适用范围是什么？（场景、人群、限制条件）
□ 这个理论的边界条件是什么？（什么情况下失效？）
□ 这个理论的学术渊源是什么？（创始人、关键文献）
□ 这个理论是否获得过重要学术认可？（诺贝尔奖等）
```

### 3.2 冲突与互补分析

**与现有理论的冲突分析**:

| 冲突类型 | 示例 | 处理策略 |
|----------|------|----------|
| **假设冲突** | 新理论假设人完全理性 | 明确边界，限定适用范围 |
| **结论冲突** | 新理论建议vs满意解建议相反 | 识别条件差异，条件化整合 |
| **范式冲突** | 新理论属于不同学科范式 | 建立翻译机制，概念映射 |

**与现有理论的互补分析**:

| 互补类型 | 示例 | 整合策略 |
|----------|------|----------|
| **层次互补** | 新理论解释"为什么"，满意解解释"怎么做" | 四层次模型整合 |
| **场景互补** | 新理论适用于A场景，满意解适用于B场景 | 场景化分工 |
| **机制互补** | 新理论解释认知机制，满意解决策框架 | 机制嵌入 |

### 3.3 输出物
- 《第一性原理拆解报告》（1-2页）
- 冲突/互补分析矩阵

## 四、阶段2：四层次定位

### 4.1 层次定位决策树

```
新理论四层次定位
│
├─ 是否涉及伦理、价值观、人生意义？
│   ├─ 是 → 价值层（第四层）
│   └─ 否 → 继续
│
├─ 是否涉及心理机制、认知过程、神经科学？
│   ├─ 是 → 认知层（第三层）
│   └─ 否 → 继续
│
├─ 是否涉及决策方法、流程、策略？
│   ├─ 是 → 方法论层（第二层）
│   └─ 否 → 继续
│
└─ 是否涉及工具、实操、具体行动？
    ├─ 是 → 执行层（第一层）
    └─ 否 → 需要重新审视理论适用性
```

### 4.2 层次定位表

| 层次 | 定位标准 | 示例理论 | 整合方式 |
|------|----------|----------|----------|
| **价值层** | 伦理基础、人生意义 | 儒商哲学、存在主义 | 五路图腾吸纳 |
| **认知层** | 心理机制、偏差识别 | 前景理论、双系统理论 | 偏差检测模块 |
| **方法论层** | 决策流程、策略规则 | 满意解、博弈论 | 方法论嵌入 |
| **执行层** | 工具、实操 | QPMS、检查清单 | 工具功能扩展 |

### 4.3 与现有理论关系图谱

**输出格式**:
```
【关系图谱示例：前景理论】

前景理论（认知层）
    │
    ├─ 增强 → 西蒙满意解（方法论层）
    │          "解释为什么满意解是必要的"
    │
    ├─ 互补 → 双系统理论（认知层）
    │          "共同解释认知机制"
    │
    ├─ 支持 → 五路图腾（价值层）
    │          "为价值判断提供认知基础"
    │
    └─ 工具化 → QPMS（执行层）
               "嵌入偏差检测功能"
```

```
   答：_______________________________________________

5. 【品牌稀释防范】是否可能造成品牌稀释？如何防范？
   答：_______________________________________________

6. 【主角地位】对外话术如何确保"满意解"仍是主角？
   示例：_______________________________________________
```

### 5.2 品牌稀释风险评估

**评估维度**:

| 维度 | 低风险（1分） | 中风险（2分） | 高风险（3分） |
|------|--------------|--------------|--------------|
| **知名度** | 学术圈小众 | 学术界知名 | 大众熟知 |
| **独立性** | 依赖满意解 | 可独立应用 | 自成体系 |
| **混淆度** | 概念差异大 | 概念有重叠 | 概念易混淆 |
| **话语权重** | 作为支撑引用 | 并列地位 | 可能喧宾夺主 |

**风险等级**:
- 4-6分：🟢 低风险，可直接引入
- 7-9分：🟡 中风险，需调整引入方式
- 10-12分：🔴 高风险，需重新评估或放弃

## 六、阶段4：整合方案设计

### 6.1 概念映射表

**模板**:

| 新理论概念 | 满意解对应概念 | 整合概念 | 对外表述 |
|------------|----------------|----------|----------|
| | | | |

**示例（前景理论整合）**:

| 前景理论概念 | 西蒙概念 | 整合概念 | 对外表述 |
|--------------|----------|----------|----------|
| 参考点 | 抱负水平 | 决策基准 | "找到你的决策锚点" |
| 损失厌恶 | 搜索成本 | 决策成本 | "保守倾向管理" |
| 概率权重 | 简化模型 | 概率校准 | "不确定下的判断" |

### 6.2 冲突解决机制

**常见冲突及解决方案**:

| 冲突场景 | 解决方案 | 示例 |
|----------|----------|------|
| 新理论建议与满意解建议相反 | 条件化整合："在X条件下用A，在Y条件下用B" | 新理论建议"最大化"，满意解建议"满意即可" → "信息完备时追求最优，信息有限时满意即可" |
| 新理论概念与满意解概念重叠 | 概念分层：底层用新理论，上层用满意解 | 两者都涉及"标准" → 前景理论解释"参考点如何形成"，满意解解释"如何设定抱负水平" |
| 新理论体系过于庞大 | 选择性引入：只引入与满意解相关的部分 | 博弈论庞大体系 → 只引入"不完全信息博弈"相关部分 |

### 6.3 对外统一话术

**话术设计原则**:
1. 满意解始终是主角
2. 新理论作为支撑或补充
3. 使用通俗化表述
4. 强调整合价值而非理论本身

**话术模板**:
```
"满意解研究所在西蒙'满意解'理论的基础上，
融合[新理论名称]的[核心洞察]，
形成了更[形容词]的决策框架。

[新理论]帮助我们[具体作用]，
让满意解在[具体场景]中更加[具体价值]。"

示例（双系统理论）:
"满意解研究所在西蒙'满意解'理论的基础上，
融合双系统理论对直觉与理性的研究，
形成了更完整的认知决策框架。

双系统理论帮助我们理解什么时候该相信直觉、
什么时候该启动理性分析，
让满意解在快速决策场景中更加可靠。"
```

## 八、理论分级与审批权限

### 8.1 理论分级标准

| 级别 | 定义 | 审批权限 | 示例 |
|------|------|----------|------|
| **I级** | 微调补充，不改变框架 | 研究主管 | 概念细化、案例补充 |
| **II级** | 增量引入，同层次扩展 | 研究主管+品牌主管 | 心理账户理论 |
| **III级** | 跨层次引入，框架调整 | 创始人 | 双系统理论 |
| **IV级** | 新层次增加，体系扩展 | 创始人 | 博弈论 |
| **V级** | 核心理论补充，品牌影响大 | 创始人+外部顾问 | 第二基础理论 |

### 8.2 审批流程

```
I级: 研究主管审批 → 备案
II级: 研究主管 + 品牌主管联签 → 备案
III级: 研究主管提案 → 创始人审批 → 执行
IV级: 研究主管提案 → 创始人审批 → 内部测试 → 执行
V级: 研究主管提案 → 创始人初审 → 外部顾问评估 → 创始人终审 → 执行
```
