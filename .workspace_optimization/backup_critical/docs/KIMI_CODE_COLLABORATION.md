# Kimi Code 协同开发工作流

## 概述
**目标**: 最大化利用Kimi Code的代码生成能力，同时控制token消耗

**核心原则**:
- 我（Kimi）负责：架构、设计、审查、复杂逻辑
- Kimi Code负责：代码生成、模板实现、批量任务

---

## Token优化策略

### 1. 任务分解
```
大任务 → 拆分为小模块 → 分批提交

示例:
❌ "写一个完整的RSS监控器"
✅ "先写RSS监控器的配置文件加载模块"
✅ "再写RSS解析核心函数"
✅ "最后写主控制循环"
```

### 2. 模板复用
```
标准模板 → 替换关键部分 → 快速生成

所有Skill使用统一模板:
- templates/skill_template.py (已创建)
- Kimi Code只需替换占位符
- 大幅减少生成token
```

### 3. 增量开发
```
已有代码 → 描述修改点 → Kimi Code实现

示例:
"在现有RSS监控器基础上，添加邮件通知功能。
需要在__init__中添加smtp配置，在检测到新文章时调用send_email方法。
"
```

### 4. 批处理
```
同类任务 → 批量描述 → 一次生成

示例:
"请为以下3个监控器生成配置文件模板:
1. RSS监控器 - 需要feed_url, check_interval字段
2. 网页监控器 - 需要url, selector字段  
3. 社交媒体监控器 - 需要platform, keywords字段
"
```

## 协作流程

### 阶段1: 架构设计（我负责）
```
我:
1. 设计Skill整体架构
2. 定义接口规范
3. 划分功能模块
4. 输出: 设计文档 + 模块清单
```

### 阶段2: 代码生成（Kimi Code负责）
```
Kimi Code:
1. 基于模板生成代码框架
2. 实现具体功能函数
3. 添加基础注释
4. 输出: 代码文件
```

### 阶段3: 代码审查（我负责）
```
我:
1. 审查代码质量
2. 检查规范符合度
3. 提出修改建议
4. 输出: 审查报告
```

### 阶段4: 优化完善（协同）
```
Kimi Code: 根据审查意见修改
我: 复杂逻辑优化、性能调优
输出: 最终代码
```

## 代码生成提示词模板

### 模板1: 基于模板生成
```
请基于以下模板，生成{Skill名称}的代码:

模板文件: templates/skill_template.py

需要替换的占位符:
- {SkillClassName}: {实际类名}
- {Skill名称}: {实际名称}
- {功能描述}: {详细描述}
- {作者}: {AI小伙伴姓名}

核心功能要求:
1. {功能1}
2. {功能2}
3. {功能3}

请生成完整可运行的代码。
```

### 模板2: 增量修改
```
请修改以下代码，添加{新功能}:

现有代码:
```python
{代码片段}
```

需要添加的功能:
- {功能描述}
- 输入: {输入格式}
- 输出: {输出格式}

请在保持原有代码结构的基础上，添加新功能。
```

### 模板3: 批量生成
```
请为以下{数量}个Skill生成配置文件:

1. {Skill1名称}
   - 配置项: {配置项1}, {配置项2}
   - 默认值: {值1}, {值2}

2. {Skill2名称}
   - ...

请生成标准的JSON配置文件。
```
