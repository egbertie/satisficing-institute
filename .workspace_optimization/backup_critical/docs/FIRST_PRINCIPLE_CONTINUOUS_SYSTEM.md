> **机制版本**: v1.0  
> **生效日期**: 2026-03-15  
> **检视频率**: 每周六 10:00

---

## 机制概述

基于第一性原则，建立工作空间的持续自我检视和优化机制，确保系统始终保持精简、高效、可进化。

### 核心原则

1. **自动化发现**: 系统自动识别低效项
2. **用户确认**: 所有优化需用户确认后执行
3. **渐进优化**: 小步快跑，持续改进
4. **可回滚**: 所有变更可回滚到之前状态

## 检视内容

### 1. Skill体系检视

#### 检视项
- [ ] Skill总数是否超过20个
- [ ] 是否存在重复功能
- [ ] 是否有30天未使用的Skill
- [ ] 命名是否规范

#### 自动化检测
```python
def check_skill_health():
    issues = []
    
    # 检测重复
    duplicates = find_duplicate_skills()
    if duplicates:
        issues.append({"type": "duplication", "items": duplicates})
    
    # 检测未使用
    unused = find_unused_skills(days=30)
    if unused:
        issues.append({"type": "unused", "items": unused})
    
    return issues
```

### 2. 文档体系检视

#### 检视项
- [ ] 文档总数是否合理
- [ ] 是否存在版本冲突
- [ ] 是否有过时文档
- [ ] 索引是否最新

### 3. Cron体系检视

#### 检视项
- [ ] Cron执行成功率
- [ ] 空转率是否过高
- [ ] Token消耗是否合理

### 4. 数据记忆检视

#### 检视项
- [ ] 日志文件是否过多
- [ ] 知识图谱是否更新
- [ ] 备份是否完整

## 优化决策矩阵

| 优先级 | 条件 | 用户确认 | 执行方式 |
|-------|-----|---------|---------|
| **P0** | 安全风险/高Token消耗 | 必须 | 等待用户确认后执行 |
| **P1** | 功能重复/文档冗余 | 建议 | 生成详细报告供决策 |
| **P2** | 命名规范/格式优化 | 可选 | 可自动执行 |
| **P3** | 索引更新/日志清理 | 无需 | 自动执行并记录 |

## Cron配置

### 每周检视Cron

```yaml
name: "weekly-first-principle-audit"
schedule: "0 10 * * 6"  # 每周六10:00
payload:
  kind: "agentTurn"
  task: |
    ## 每周第一性原则检视
    
    1. 执行全面审计
       - 使用 first-principle-auditor Skill
       - 生成审计报告
    
    2. 分析结果
       - 识别P0/P1/P2/P3级别问题
       - 计算预期收益
    
    3. 发送给用户
       - 总结关键发现
       - 列出优化建议
       - 等待用户确认
    
    4. 如用户确认，执行优化
       - 使用 workspace-optimizer Skill
       - 生成执行报告
```

### 条件触发Cron

```yaml
name: "conditional-health-check"
schedule: "0 */6 * * *"  # 每6小时检查一次
condition: |
  skill_count > 20 OR
  daily_token > 50000 OR
  doc_conflicts > 3
action: |
  触发第一性原则检视
```

## CLI集成

### 检视命令

```bash
# 手动触发检视
claw audit --all

# 查看检视历史
claw audit --history

# 查看特定维度
claw audit --skill
claw audit --doc
claw audit --cron

# 生成并查看报告
claw audit --report --format markdown
```

```bash
# 执行优化（需用户确认）
claw optimize --all

# 模拟运行
claw optimize --dry-run

# 回滚
claw optimize --rollback [backup_id]
```
