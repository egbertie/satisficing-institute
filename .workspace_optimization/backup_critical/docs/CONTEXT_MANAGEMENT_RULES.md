# 上下文管理规则

> **版本**: V1.0  
> **创建时间**: 2026-03-15  
> **目的**: 优化上下文加载效率，建立分层管理机制

---

## 上下文分层架构

### 第一层：核心上下文 (Always Load)

**加载时机**: 每次会话启动自动加载  
**大小限制**: < 1MB  
**文件清单**:

| 文件 | 用途 | 更新频率 |
|------|------|----------|
| `SOUL.md` | AI身份定义 | 极少 |
| `USER.md` | 用户档案 | 偶尔 |
| `AGENTS.md` | 工作协议 | 极少 |
| `MEMORY.md` | 精简长期记忆 | 定期 |
| `HEARTBEAT.md` | 心跳协议 | 偶尔 |

### 第二层：任务上下文 (On-Demand Load)

**加载时机**: 任务开始时按需加载  
**大小限制**: < 5MB  
**文件清单**:

| 文件/目录 | 用途 | 加载触发 |
|-----------|------|----------|
| `TASK_MASTER.md` | 当前任务追踪 | 任务查询 |
| `skills/{current}/SKILL.md` | 相关Skill文档 | Skill调用 |
| `memory/YYYY-MM-DD.md` | 当日日志 | 当日回溯 |
| `docs/MEETING_PROTOCOL.md` | 会议机制 | 会议场景 |

### 第三层：归档上下文 (Lazy Load)

**加载时机**: 显式请求时加载  
**存储位置**: `archive/`  
**文件清单**:

| 目录 | 内容 | 保留期限 |
|------|------|----------|
| `archive/2026-03/` | 3月历史日志 | 永久 |
| `knowledge_base/` | 参考资料 | 永久 |
| `deliverables/` | 交付物归档 | 永久 |

---

## 加载规则

### 自动加载 (第一层)

```
会话启动
    ↓
自动加载第一层
    ↓
等待用户输入
```

### 按需加载 (第二层)

```
用户请求
    ↓
解析意图
    ↓
识别相关上下文
    ↓
动态加载第二层
    ↓
执行处理
```

### 懒加载 (第三层)

```
用户显式请求
    ↓
确认加载范围
    ↓
加载归档上下文
    ↓
处理并释放
```

## 维护规则

### 日常维护

| 操作 | 频率 | 负责人 |
|------|------|--------|
| 归档昨日日志 | 每日 | 自动 |
| 更新MEMORY.md | 每3天 | AI |
| 清理临时文件 | 每日 | 自动 |

### 归档流程

1. **每日23:00** 自动归档前日memory文件
2. **归档路径**: `archive/YYYY-MM/`
3. **保留原文件**: 1天后删除原位置文件
4. **索引更新**: 更新归档索引

### MEMORY.md精简规则

**保留内容**:
- 用户基本信息
- 当前活跃项目(最多3个)
- 关键系统配置
- 自建核心Skill清单

**移至完整版**:
- 历史项目详情
- 已归档任务记录
- 详细配置说明

| 指标 | 目标 | 监控方式 |
|------|------|----------|
| 核心上下文大小 | < 1MB | 定期检查 |
| 会话启动时间 | < 3秒 | 用户感知 |
| 上下文命中率 | > 80% | 日志分析 |
| 归档完整性 | 100% | 校验和 |

## 应急处理

### 上下文过载

**症状**: 响应变慢、截断频繁  
**处理**:
1. 立即切换到精简模式
2. 只加载第一层上下文
3. 用户确认后按需加载

### 上下文丢失

**症状**: 无法找到历史信息  
**处理**:
1. 检查archive目录
2. 通过索引恢复
3. 必要时从备份还原
