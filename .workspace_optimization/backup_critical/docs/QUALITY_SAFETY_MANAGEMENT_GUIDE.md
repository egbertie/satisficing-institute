> **生效日期**: 2026-03-10  
> **制定者**: 满意妞 + PEO + EEO  
> **审批人**: Egbertie

## 一、三级把关体系

### 1.1 体系概览

```
┌─────────────────────────────────────────────────────┐
│  第一级：自检（执行者）                              │
│  ├── 完成前对照检查清单自检                          │
│  ├── 使用提供的工具和模板                            │
│  └── 确保基础质量达标                                │
│                          ↓ 通过后                    │
│  第二级：Peer Review（同级复核）                     │
│  ├── 重要文档必须经1位同级复核                       │
│  ├── 使用复核模板，记录修改意见                      │
│  └── 复核不通过返回修改                              │
│                          ↓ 通过后                    │
│  第三级：专家把关（蓝军/主管）                       │
│  ├── 对外发布内容必须经过蓝军审查                    │
│  ├── 财务/法务相关内容必须经专业角色审查             │
│  └── 重大决策必须经PEO+EEO+指挥官三级确认            │
└─────────────────────────────────────────────────────┘
```

### 1.2 各级详细要求

#### 第一级：自检清单（Checklist）

**所有文档自检项**:
- [ ] 标题明确，与内容匹配
- [ ] 无错别字，标点符号正确
- [ ] 格式统一，符合模板要求
- [ ] 引用来源已标注
- [ ] 敏感信息已脱敏
- [ ] 无AI slop（蓝紫渐变/万能句式）

**对外文案额外检查**:
- [ ] 符合品牌调性（专业/有格调）
- [ ] 无争议性表述
- [ ] 联系方式正确
- [ ] 法律风险已排查

**技术文档额外检查**:
- [ ] 代码可运行
- [ ] 配置参数正确
- [ ] 版本号已更新
- [ ] 依赖项已列出

#### 第二级：Peer Review流程

**审查人选择**:
- 同领域角色优先
- 不能是执行者本人
- 审查人需有L2以上级别

**审查内容**:
1. 事实准确性（数据/引用/结论）
2. 逻辑完整性（推理是否严密）
3. 表达清晰度（是否易懂）
4. 格式规范性（是否符合标准）

**审查记录模板**:
```markdown
## Peer Review 记录

**文档**: [文档名]
**执行者**: [姓名]
**审查人**: [姓名]
**审查时间**: [时间]

### 审查意见
1. [意见1] - [修改建议]
2. [意见2] - [修改建议]

### 修改确认
- [ ] 意见1已修改
- [ ] 意见2已修改
- [ ] 其他修改

**审查结论**: ☐ 通过 ☐ 需修改后复审
**审查人签名**: [签名]
```

#### 第三级：专家把关

**蓝军审查重点**:
- 事实核查（数据来源验证）
- 逻辑漏洞（推理断裂点）
- 风险评估（潜在负面影响）
- 竞品对比（是否有竞争优势）

**法务审查重点**:
- 合规性（是否符合法规）
- 知识产权（是否有侵权风险）
- 合同条款（是否有法律漏洞）
- 隐私保护（是否有泄露风险）

**财务审查重点**:
- 数据准确性（计算是否正确）
- 预算合理性（是否符合实际）
- 成本效益（ROI是否合理）
- 风险敞口（财务风险评估）

## 二、多角色质量把关矩阵

### 2.1 矩阵总表

| 产出类型 | 自检 | Peer Review | 蓝军 | 法务 | 财务 | 最终审批 | 归档位置 |
|---------|:----:|:-----------:|:----:|:----:|:----:|:--------:|:--------:|
| 对外文案（公众号/知乎） | ✅ | ✅ | ✅ | ⚠️ | - | CONTENT | `docs/content/published/` |
| 官宣内容（重大发布） | ✅ | ✅ | ✅ | ✅ | - | Egbertie | `docs/announcements/` |
| 合同协议 | ✅ | ✅ | ✅ | ✅ | ✅ | Egbertie | `docs/legal/contracts/` |
| 财务报告 | ✅ | ✅ | - | - | ✅ | FIN | `docs/finance/reports/` |
| 咨询报告（给客户） | ✅ | ✅ | ✅ | - | - | PEO+EEO | `docs/deliverables/` |
| API配置/代码 | ✅ | ✅ | ✅ | - | - | 满意妞 | `configs/approved/` |
| 研究论文/白皮书 | ✅ | ✅ | ✅ | ✅ | - | EEO | `docs/research/` |
| 培训材料 | ✅ | ✅ | ⚠️ | - | - | EEO | `docs/training/` |
| 内部备忘录 | ✅ | - | - | - | - | 直接发布 | `docs/internal/` |

**图例**:
- ✅ 必须审查
- ⚠️ 建议审查（高风险内容需审查）
- - 不需要审查

### 2.2 特殊情况处理

**加急发布（需在2小时内完成）**:
- 可合并Peer Review和自检（由执行者自行复核）
- 蓝军审查可改为事后审查（发布后24小时内）
- 但法务审查**不可省略**（涉及法律风险）

**紧急修复（漏洞/错误）**:
- 先修复后补流程（30分钟内完成）
- 修复后24小时内补全审查记录
- 重大错误需发布更正声明

## 三、报告真实性保障

### 3.1 信息来源分级

| 级别 | 标识 | 说明 | 使用限制 |
|------|:----:|------|---------|
| **一级** | 🟢 | 官方数据、权威论文、一手调研 | 可直接引用，优先使用 |
| **二级** | 🟡 | 行业报告、专家观点、媒体报道 | 需标注来源，谨慎使用 |
| **三级** | 🔴 | 网络信息、推测判断、非官方数据 | 必须标注"基于...推测" |

### 3.2 引用规范

**标准格式**:
```markdown
根据[来源名称]（[发布日期]），[具体内容]。
链接：[URL]

示例：
根据麦肯锡《中国硬科技报告》（2025-12），
硬科技创业公司平均需要18个月完成首轮融资。
链接：https://www.mckinsey.com/...
```

**推测标注**:
```markdown
基于[数据/事实]，我们推测[结论]。
这一推测尚需进一步验证。

示例：
基于当前5个案例的数据，我们推测合伙人匹配
成功率约为80%。这一推测尚需更多案例验证。
```

### 3.3 虚假信息防范机制

**事实核查流程**:
1. 所有数据必须通过至少2个独立来源验证
2. 专家观点必须标注专家姓名和资质
3. 网络信息必须通过权威渠道二次确认
4. 不确定信息必须标注"待验证"

**蓝军事实核查清单**:
- [ ] 所有数据有来源
- [ ] 所有引用可查证
- [ ] 所有专家有资质
- [ ] 无夸大或误导性表述
- [ ] 无未经证实的推测

**虚假信息处理**:
- 发现虚假信息立即撤回
- 发布更正声明（48小时内）
- 责任人进行复盘（为何未检出）
- 更新流程防止再次发生

## 四、交叉复核机制

### 4.1 每日复核（站会中）

**时间**: 每日09:30站会最后5分钟
**主持人**: 满意妞
**流程**:
1. 昨日产出快速回顾（1分钟）
2. 发现问题立即标记（2分钟）
3. 责任人确认修改时间（2分钟）

**输出**: 问题清单（记录在`memory/YYYY-MM-DD.md`）

### 4.2 每周复核（周会中）

**时间**: 每周日20:00周会
**时长**: 15分钟
**参与者**: 全员
**流程**:
1. 本周所有对外产出审查（10分钟）
   - 抽查3-5份文档
   - 检查是否经过三级把关
   - 检查质量是否达标
2. 质量问题归因分析（3分钟）
   - 为什么会出现质量问题
   - 是流程问题还是执行问题
3. 改进措施制定（2分钟）
   - 下周如何避免

**输出**: 质量周报（记录在`docs/quality/weekly/YYYY-MM-DD.md`）

### 4.3 每月复核（质量月会）

**时间**: 每月最后一个周五
**时长**: 60分钟
**参与者**: PEO + EEO + CONTENT + 蓝军 + Egbertie
**议程**:
1. **质量指标统计**（15分钟）
   - 三级把关通过率
   - 返工率
   - 错误类型分布
2. **最佳实践分享**（20分钟）
   - 本月质量最高的3个产出
   - 为什么做得好
   - 可复制的方法
3. **质量体系优化**（20分钟）
   - 现有流程的问题
   - 改进建议
   - 下月优化计划
4. **奖惩通报**（5分钟）
   - 质量优秀者表彰
   - 质量问题警告

**输出**: 质量月报（记录在`docs/quality/monthly/YYYY-MM.md`）

## 五、执行保障

### 5.1 工具支持

**质量检查工具**:
- `scripts/quality_check.py` - 自动检查文档格式、拼写、引用
- `scripts/peer_review_template.py` - 生成Peer Review记录模板
- `scripts/fact_check.py` - 辅助事实核查（标注待验证信息）

**审查追踪工具**:
- 所有审查记录保存在`docs/quality/reviews/`
- 使用统一命名格式：`[文档名]_[审查类型]_[日期].md`

### 5.2 培训机制

**新成员培训**:
- 入职必读：本指引全文
- 实操演练：模拟审查一份文档
- 考核通过：独立完成一次三级把关

**定期培训**:
- 每月质量月会后，针对本月问题进行专项培训
- 每季度邀请外部专家进行质量意识培训

### 5.3 激励机制

**质量奖励**:
- "零缺陷奖"：连续1个月无质量问题
- "最佳审查员"：Peer Review质量最高
- "事实核查先锋"：发现重大虚假信息避免损失

**质量惩罚**:
- 未经审查对外发布：警告+复盘
- 虚假信息造成损失：严重警告+流程改进
- 连续3次质量问题：降级处理（L3→L2）

### 附录B：常见问题FAQ

**Q1: 时间紧急，可以跳过Peer Review吗？**
A: 可以，但需满足：1) 事后24小时内补审查；2) 法务审查不可跳过；3) 记录原因。

**Q2: 找不到合适的Peer Reviewer怎么办？**
A: 由满意妞指定，或升级为专家直接审查（跳过Peer Review）。

**Q3: 蓝军和法务意见冲突怎么办？**
A: 以法务意见为准（法律风险优先），蓝军意见记录备案。

**Q4: 如何界定"对外发布"？**
A: 任何离开团队内部、可能被外部人员看到的内容，都算对外发布。
