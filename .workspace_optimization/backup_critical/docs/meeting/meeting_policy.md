# 满意解研究所会议制度

> 适用对象：满意解研究所全体成员（33人）
> 生效日期：2026年3月12日
> 版本：v1.0

---

## 一、会议体系概览

为提升团队协作效率、确保信息透明流通，研究所建立三层会议体系：

| 会议类型 | 频率 | 时长 | 参与者 | 核心目的 |
|---------|------|------|--------|---------|
| **每日站会** | 工作日每天 | 15分钟 | 项目组成员（≤12人） | 同步进度、暴露障碍 |
| **周例会** | 每周一/五 | 60分钟 | 全所成员（33人） | 阶段复盘、资源协调 |
| **月度回顾** | 每月最后周五 | 90分钟 | 核心成员+邀请制 | 深度复盘、战略对齐 |

---

## 二、每日站会（Daily Standup）

### 2.1 基本信息
- **时间**：每日 09:30-09:45（或各项目组自定）
- **时长**：严格控制在 **15分钟** 内
- **形式**：站立进行（物理或虚拟）
- **参与者**：项目组成员（建议8-12人/组）

### 2.2 议程结构
每人发言限时 **1-2分钟**，回答三个问题：

1. **昨天完成** — 昨天做了什么？达成什么成果？
2. **今天计划** — 今天计划做什么？优先级是什么？
3. **遇到的障碍** — 需要什么帮助？有什么阻塞？

### 2.3 运行规则

| 规则 | 说明 |
|------|------|
| ❌ 不解决问题 | 障碍记录后，会后单独讨论 |
| ❌ 不打断他人 | 全程专注倾听，不插话 |
| ✅ 准时开始 | 到点开始，不等迟到者 |
| ✅ 关注整体 | 发言围绕团队目标，非个人流水账 |
| ✅ 更新看板 | 会后立即同步任务状态到项目管理工具 |

### 2.4 分组建议（33人配置）

| 项目组 | 人数 | 站会时间 | 负责人 |
|--------|------|----------|--------|
| 研究一组 | 10人 | 09:30 | 组长A |
| 研究二组 | 10人 | 09:45 | 组长B |
| 研究三组 | 8人 | 10:00 | 组长C |
| 行政支持组 | 5人 | 10:15 | 行政主管 |

> 注：跨组协作问题在周例会统一协调

## 三、周例会（Weekly Sync）

### 3.1 基本信息
- **时间**：每周五 16:00-17:00（或周一 09:00-10:00）
- **时长**：**60分钟**，严格时间盒
- **形式**：会议室/线上会议
- **参与者**：全所33人，必须全员参与

### 3.2 议程结构（建议分配）

| 阶段 | 内容 | 时长 | 负责人 |
|------|------|------|--------|
| 1. 数据回顾 | 关键指标回顾（OKR进度、项目里程碑） | 10分钟 | 运营/PM |
| 2. 项目组汇报 | 各组5分钟：本周成果、下周计划、需协调事项 | 20分钟 | 各组长 |
| 3. 跨组协调 | 跨组依赖、资源冲突讨论 | 15分钟 | 主持人 |
| 4. 决策事项 | 需要集体决策的问题快速表决 | 10分钟 | 所长/主管 |
| 5. 行动总结 | 确认下周行动项、责任人、截止时间 | 5分钟 | 会议记录 |

### 3.3 会前准备

**汇报组需提前准备：**
- [ ] 本周完成的任务清单（附数据/成果）
- [ ] 未完成任务的原因说明
- [ ] 下周计划任务清单（含优先级）
- [ ] 需要其他组支持的事项

**会议记录需提前准备：**
- [ ] 上周行动项完成状态检查表
- [ ] 本周决策事项清单

### 3.4 运行规则

- ✅ **数据说话**：汇报必须带数据，避免主观描述
- ✅ **聚焦变化**：只说与上周相比的变化，不说重复背景
- ✅ **提前共享**：汇报材料会前1小时发到群里
- ✅ **当场决策**：复杂问题会前单独沟通，会上只决策
- ❌ **不展开技术细节**：技术问题会后单独讨论

## 四、月度回顾（Monthly Retrospective）

### 4.1 基本信息
- **时间**：每月最后一个周五 14:00-15:30
- **时长**：**90分钟**
- **形式**：工作坊风格，鼓励互动
- **参与者**：
  - **必到**：核心管理层（5-7人）
  - **邀请**：当月表现突出者、新员工代表、项目关键贡献者
  - **规模**：建议15-20人

### 4.2 核心目标
1. **深度复盘**：回顾本月成果与教训
2. **流程优化**：识别并改进工作方法
3. **战略对齐**：确保团队目标与研究所方向一致
4. **文化建设**：增强团队凝聚力

### 4.3 议程结构

| 阶段 | 内容 | 时长 | 方法 |
|------|------|------|------|
| 1. 数据回顾 | 月度OKR达成情况、关键项目复盘 | 15分钟 | 数据可视化展示 |
| 2. 亮点分享 | 本月3个最佳实践/成功故事 | 15分钟 | 当事人分享 |
| 3. 痛点诊断 | 本月遇到的系统性问题 | 20分钟 | 分组讨论+汇总 |
| 4. 改进实验 | 确定下月要尝试的改进措施 | 20分钟 | 优先级投票 |
| 5. 战略更新 | 所长分享战略思考与方向调整 | 15分钟 | 演讲+Q&A |
| 6. 庆祝时刻 | 认可贡献者、团队庆祝 | 5分钟 | 颁奖/致谢 |

### 4.4 改进实验追踪

每月确定 **1-3个改进实验**，在下月执行并跟踪：

```
实验名称：____________________
假设：_________________________
负责人：_____________________
成功指标：____________________
截止日期：____________________
```

下月回顾时检查实验效果，有效则固化成制度，无效则废弃。

## 五、特殊会议类型

### 5.1 临时协调会
- **触发条件**：跨组紧急问题需要快速决策
- **时长**：30分钟内
- **参与者**：相关方代表（不超过6人）
- **要求**：必须有明确的议题和预期产出

### 5.2 决策评审会
- **触发条件**：重大项目节点、资源分配决策
- **时长**：60-90分钟
- **参与者**：决策者+执行者+顾问
- **要求**：会前提供决策材料，会上只表决

### 5.3 1:1面谈
- **频率**：主管与下属每两周一次
- **时长**：30分钟
- **内容**：个人发展、工作状态、反馈收集

## 六、会议纪律

### 6.1 时间纪律
- ⏰ 准时开始，不等迟到者
- ⏰ 准时结束，不拖延
- ⏰ 提前5分钟到场测试设备（线上会议）

### 6.2 参与纪律
- 📵 手机静音，不发消息
- 💻 非汇报时间电脑合盖（或仅用于记录）
- 🙋 发言简洁，不跑题

### 6.3 记录纪律
- 📝 每次会议必须产生会议纪要
- 📝 行动项必须有责任人+截止日期
- 📝 纪要24小时内分发

---

## 七、会议效果评估

每季度进行一次会议效果评估：

| 评估维度 | 问题 | 评分（1-5） |
|---------|------|------------|
| 时间效率 | 会议是否按时开始/结束？ | |
| 目标达成 | 会议目的是否实现？ | |
| 参与度 | 成员是否积极参与？ | |
| 行动落地 | 行动项是否被执行？ | |
| 整体满意度 | 对会议体系的整体评价？ | |

**评分<3的维度需在下月回顾中讨论改进方案。**
