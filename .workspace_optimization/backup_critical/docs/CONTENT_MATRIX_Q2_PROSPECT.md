## 三、内容创作规范

### 3.1 标题公式

#### 损失框架标题（高点击率）
- "那个[动作]的创始人，[时间]后[损失]了[数字]"
- "[数字]%的[人群]都犯了这个错误"
- "为什么聪明人总在[场景]上犯错"
- "[动作]的代价：[具体损失]"

#### 好奇框架标题
- "Kahneman没告诉你的：[主题]"
- "从[X]到[Y]：一个[人群]的觉醒"
- "我们阻止了第[数字]号disaster"

#### 实用框架标题
- "[数字]个信号表明你有[问题]"
- "[主题]：完整指南/[年份]版"
- "如何[动作]：[方法]"

### 3.2 内容结构模板

#### 公众号长文模板（3000-5000字）

```
【开头】钩子（损失数据/反直觉观点）- 200字
【痛点】描述问题严重性 - 500字
【原理】认知科学解释（简单易懂）- 800字
【案例】真实故事/案例 - 800字
【方法】具体解决方案 - 1000字
【工具】提供实用工具/模板 - 300字
【CTA】引导下一步行动 - 200字
```

#### 小红书图文模板

```
封面：痛点数字 + 解决方案预告

正文：
- 问题场景（1段）
- 认知偏差解释（2-3点，配emoji）
- 真实影响（1个案例）
- 解决方法（3-5步）
- 行动号召

配图：信息图/步骤图/对比图
```

#### 视频脚本模板（3分钟）

```
0:00-0:15 钩子（震撼数据或问题）
0:15-0:45 问题展开（真实案例引入）
0:45-1:30 原理解释（认知偏差）
1:30-2:15 解决方案（具体方法）
2:15-2:45 结果展示（前后对比）
2:45-3:00 CTA（关注+行动）
```
