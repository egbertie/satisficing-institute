# 空间利用管理策略 V1.0
> 预防性管理机制 - 避免资源成为瓶颈

---

## 一、监控指标

### 1.1 关键阈值
| 指标 | 健康 | 警告 | 危险 | 紧急 |
|------|------|------|------|------|
| 系统盘(/) | <50% | 50-70% | 70-85% | >85% |
| 数据盘(/data) | <60% | 60-75% | 75-90% | >90% |
| 内存使用 | <60% | 60-75% | 75-85% | >85% |
| 单文件大小 | <10M | 10-50M | 50-100M | >100M |

### 1.2 监控频率
- **实时监控**: 自动告警（危险阈值）
- **日报**: 空间使用摘要
- **周报**: 趋势分析+预测

## 二、自动清理任务

### 2.1 定时清理（已配置）
| 任务 | 时间 | 操作 | 预期释放 |
|------|------|------|----------|
| 临时文件清理 | 每日03:00 | 删除/tmp, /var/tmp过期文件 | 5-20G |
| 日志轮转 | 每日03:00 | 压缩7天前日志 | 1-5G |
| 缓存清理 | 每周日 | 清理pip/npm/cache | 2-10G |

### 2.2 智能归档
```
工作区文件生命周期:

活跃期(0-7天) → 工作区
    ↓
观察期(7-30天) → 保留，标记
    ↓
归档期(30-90天) → 压缩，移入archive/
    ↓
冷存期(90天+) → 冷存储或删除
```

## 三、文件管理规范

### 3.1 命名规范
```
YYYYMMDD_内容描述_版本.扩展名

示例:
✓ 20260311_管理制度整合_V2.md
✓ 20260310_蓝军意见整理_V1.1.xlsx
✗ 管理制度.md (无日期)
✗ 新建文件夹 (无意义)
```

### 3.2 目录结构
```
workspace/
├── 📁 active/           # 当前活跃项目
├── 📁 archive/          # 归档文件（压缩）
├── 📁 docs/             # 正式文档
├── 📁 memory/           # 日志+记忆
├── 📁 knowledge_base/   # 知识库
├── 📁 temp/             # 临时文件（自动清理）
└── 📄 README.md         # 索引
```

### 3.3 文件整合原则
1. **同类合并**: 同主题文件合并为一份
2. **版本控制**: 保留最新版，旧版入archive
3. **压缩策略**: 大文件(>10M)考虑压缩
4. **去重检查**: 保存前检查是否已存在

## 四、大文件处理

### 4.1 识别标准
- 单文件 > 10M → 标记关注
- 单文件 > 50M → 必须压缩或拆分
- 单文件 > 100M → 需Egbertie确认存储方式

### 4.2 处理策略
| 类型 | 处理方式 |
|------|----------|
| 文档 | 转PDF压缩/分层存储 |
| 数据 | CSV压缩/数据库存储 |
| 图片 | 转WebP/降低分辨率 |
| 视频 | 外链存储/转低码率 |
| 备份 | 差异备份/增量存储 |

## 六、应急预案

### 6.1 空间不足时
1. **立即**: 清理临时文件（可释放5-20G）
2. **短期**: 压缩旧日志和归档文件
3. **中期**: 迁移大文件至冷存储
4. **长期**: 扩容或优化存储架构

### 6.2 联系升级
- 警告级别: AI自动处理
- 危险级别: 报告Egbertie+自动处理
- 紧急级别: 立即报告+暂停非关键任务
