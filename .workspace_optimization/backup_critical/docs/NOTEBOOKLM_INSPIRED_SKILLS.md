## 二、借鉴开发的自有Skill清单

### Skill 1: 知识库问答引擎（基于来源）
**借鉴**: NotebookLM 的基于来源问答

**功能**:
- 上传文档建立向量索引
- 提问时返回基于文档的回答
- 每个回答附带原文引用（文件+段落）
- 支持多文档交叉引用

**实现方案**:
- 使用 Jina AI Embedding API 做文档向量
- 使用 DuckDB 存储向量索引
- 使用 Kimi 生成带引用的回答
- 文件位置: `skills/knowledge-qa-engine/`

**应用场景**:
- 客户咨询时快速查询方法论
- 合伙人评估时引用标准
- 内部培训时查找案例

---

### Skill 2: 智能文档摘要生成器
**借鉴**: NotebookLM 的多格式摘要

**功能**:
- 输入长文档，自动生成多种格式摘要
- 支持格式: 一句话摘要/段落摘要/要点列表/思维导图
- 自动生成 FAQ
- 自动生成执行清单

**实现方案**:
- 使用 Kimi 长文本能力
- 使用 Jina AI 提取关键句子
- 输出多种格式到指定目录

**应用场景**:
- 长篇报告快速提取要点
- 会议纪要自动生成
- 客户资料快速了解

---

### Skill 3: 主题关联分析器
**借鉴**: NotebookLM 的主题提取和关联

**功能**:
- 分析多个文档，提取共同主题
- 生成主题-文档关联图谱
- 发现文档间的隐含联系
- 生成知识图谱

**实现方案**:
- 使用 NLP 提取关键词和主题
- 使用 NetworkX 生成关联图
- 输出可视化图表

**应用场景**:
- 案例库主题归类
- 专家知识体系梳理
- 方法论体系完善

---

### Skill 4: 时间线自动生成器
**借鉴**: NotebookLM 的时间线功能

**功能**:
- 从文档中提取时间相关信息
- 自动生成项目时间线
- 识别里程碑和关键节点
- 生成可视化时间轴

**实现方案**:
- 使用正则表达式提取日期
- 使用 Kimi 理解上下文
- 输出时间线图表（HTML/Markdown）

**应用场景**:
- 客户发展历程梳理
- 项目进度可视化
- 历史案例整理

---

### Skill 5: 音频学习材料生成器
**借鉴**: NotebookLM 的音频概述

**功能**:
- 将文档转换为播客脚本
- 生成对话式内容
- 使用 TTS 生成音频文件
- 支持多角色对话（专家问答形式）

**实现方案**:
- 使用 Kimi 生成播客脚本
- 使用 TTS 生成音频
- 输出 MP3 + 脚本

**应用场景**:
- 方法论音频课程
- 案例故事音频
- 通勤学习内容

---

### Skill 6: 多格式文档统一处理器
**借鉴**: NotebookLM 的多格式支持

**功能**:
- 统一处理 PDF/Word/Excel/图片
- 提取文本内容
- 转换为标准 Markdown
- 建立文档索引

**实现方案**:
- 使用 Jina AI 网页/文档提取
- 使用 Pandoc 转换格式
- 建立统一文档库

**应用场景**:
- 客户资料统一整理
- 行业报告归档
- 竞品资料分析
