## 一、会议制度概述

### 1.1 目的
建立规范、高效的内部会议机制，确保团队沟通顺畅、信息透明、决策高效。

### 1.2 适用范围
本制度适用于团队所有内部会议。

### 1.3 基本原则

|------|------|

| **核心内容** | 昨日完成、今日计划、阻塞问题 |

**规则：**
- 超时问题会后单独讨论
- 不展开技术细节讨论
- 聚焦三件事，一人不超过2分钟

## 四、会议管理规范

### 4.1 会议发起流程
1. **明确需求**：确定会议目的、必要性
2. **选择类型**：根据内容选择合适的会议类型
3. **提前通知**：至少提前24小时发送议程
4. **准备材料**：会前共享相关资料

### 4.2 会议进行规范
- 主持人把控节奏，确保按议程进行
- 记录人实时记录要点和决议
- 参会人专注参与，减少干扰
- 偏离主题时及时提醒拉回

### 4.3 会议后续跟进
- **24小时内**：发送会议纪要
- **48小时内**：确认行动项负责人
- **每周**：更新行动项跟踪表
- **下次会议**：回顾上次行动项完成情况

### 4.4 会议纪要管理
- 所有会议纪要统一存档
- 命名规范：`MTG-YYYYMMDD-主题-纪要.md`
- 存储位置：`docs/meetings/YYYY-MM/`
