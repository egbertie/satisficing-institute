# 方案C+：平衡式事件驱动设计（优化版）

## 问题识别

**方案C的核心矛盾**：
- 极端可控性（100%人工确认）→ 可能导致长时间空置
- 用户未及时看到通知 → 任务阻塞 → Token效率下降

**需要平衡的原则**：
> **可控性 ≠ 完全手动，而是「分层授权 + 渐进确认」**

---

## 优化框架：三层响应机制

### 第一层：静默自治层（零打扰）
**触发条件**：
- 任务明确、风险可控、历史成功率高
- Token余量 > 30%
- 用户离线时间 < 2小时

**响应方式**：
```
检测到任务可执行 → 自动执行 → 完成后静默归档
仅在摘要中提及："已完成XX任务，节省您XX分钟"
```

**授权清单**（预定义）：
- ✅ 经济环境监测（纯信息采集）
- ✅ 备份任务
- ✅ 知识库整理
- ✅ 日报/周报生成
- ❌ 任何涉及外部发送的任务
- ❌ 任何可能产生承诺的任务
- ❌ 任何Skill安装/更新

### 第二层：通知确认层（轻打扰）
**触发条件**：
- 任务需要确认，但风险中等
- Token余量 15-30%
- 用户离线时间 2-6小时

**响应方式**：
```
检测到任务可执行 → 发送通知 → 等待15分钟 → 无响应则执行
通知内容：
"检测到XX条件已满足，建议执行XX任务。
15分钟内无响应将自动执行（低风险）。
如需阻止请回复：暂停"
```

**授权清单**：
- ⚠️ 客户价值复盘（内部报告）
- ⚠️ 行为效果分析（内部报告）
- ⚠️ 学习进修任务

### 第三层：强制阻断层（高打扰）
**触发条件**：
- 高风险任务
- Token余量 < 15%
- 用户离线时间 > 6小时
- 涉及外部交互

**响应方式**：
```
检测到任务 → 立即通知 → 等待人工确认
无响应 → 任务挂起 → 下次用户在线时提醒
绝不自动执行
```

**阻断清单**：
- 🚫 任何发送给客户的报告
- 🚫 任何对外发布的内容
- 🚫 任何Skill安装/更新/删除
- 🚫 任何涉及费用的操作
- 🚫 任何可能产生法律/商业后果的操作

## 时间优化设计：渐进式触发

### 时间窗口策略

| 用户状态 | 检测频率 | 响应模式 | 示例 |
|----------|----------|----------|------|
| 在线 | 实时 | 立即询问 | "现在有空处理XX吗？" |
| 刚离线（<30min） | 不触发 | 等待回归 | 静默记录 |
| 离线30min-2h | 每小时检查 | 第一层自治 | 自动执行安全任务 |
| 离线2h-6h | 每2小时检查 | 第二层确认 | 15分钟等待窗口 |
| 离线>6h | 每4小时检查 | 第三层阻断 | 仅记录，不执行 |
| 深夜（23:00-07:00）| 每6小时检查 | 第一层自治 | 夜间模式自动执行 |

### 智能提醒升级

**问题**：用户可能没看到通知

**优化**：渐进式提醒
```
T+0min:  发送通知（主通道Kimi）
T+5min:  未读 → 飞书静默推送（第二通道）
T+10min: 仍未读 → 标记"待处理高亮"
T+15min: 第一层任务自动执行 / 第二层开始倒计时
T+30min: 生成"待办摘要"置顶显示
```

## 防空置机制（平衡版）

### 核心原则
> **不是"完全不自启"，而是"分级授权自启"**

### 防空置策略

**轻度空置（30分钟-2小时）**：
- 自动执行第一层任务
- 不产生通知打扰
- 完成后归档记录

**中度空置（2-6小时）**：
- 触发第二层确认流程
- 15分钟确认窗口
- 无响应则执行

**重度空置（>6小时）**：
- 仅记录，不执行
- 用户回归时提供"快速处理清单"
- 一键批量处理积压任务

## 用户自定义配置

### 可配置参数

```yaml
# 用户偏好配置（可调整）
autonomy_level: "balanced"  # conservative / balanced / aggressive

tier1_auto_execute: true    # 第一层是否自动执行
tier2_grace_period: 15      # 第二层等待分钟数
tier3_block_all: true       # 第三层是否完全阻断

night_mode: true            # 夜间是否自动执行第一层
night_hours: "23:00-07:00"  # 夜间时段

max_idle_notify: 2h         # 超过此时长未交互则提醒
urgent_bypass: true         # 紧急任务是否可绕过确认
```

### 三种预设模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **保守模式** | 重要决策期 | 几乎全手动，仅备份自动 |
| **平衡模式** | 日常运营 | 三层渐进，15分钟确认窗口 |
| **积极模式** | 高效冲刺期 | 第一层全自动，第二层5分钟窗口 |

## 实施建议

### 推荐配置（默认）
```yaml
模式: 平衡模式
第一层: 自动执行（备份/监测/整理）
第二层: 15分钟确认窗口（报告/分析）
第三层: 完全阻断（对外/承诺/费用）
夜间: 自动执行第一层
```

### 逐步验证
1. **第一周**：保守模式，观察通知到达率
2. **第二周**：切换到平衡模式，测试15分钟窗口
3. **第三周**：根据反馈微调参数

---

## 总结

**方案C+的核心**：
> 不是「完全控制」或「完全自动」的二选一，
> 而是「智能分层」的动态平衡。

**防空置的关键**：
> 低风险任务自动填充，
> 中风险任务限时确认，
> 高风险任务严格阻断。

**用户可控性**：
> 随时可调整模式、修改授权清单、查看执行日志。
