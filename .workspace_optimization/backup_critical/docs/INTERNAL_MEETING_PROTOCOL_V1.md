## 一、会议体系概览

### 1.1 会议类型矩阵

| 会议类型 | 频率 | 触发条件 | 时长 | 参与方 | 核心目的 |
|----------|------|----------|------|--------|----------|
| **日站会** | 每日 | 固定时间09:00 | 10-15分钟 | Egbertie + 满意妞 | 同步状态、识别阻塞 |
| **周回顾会** | 每周 | 每周日20:00 | 30-45分钟 | Egbertie + 满意妞 | 复盘进展、调整计划 |
| **紧急会议** | 按需 | 触发条件满足 | 15-30分钟 | Egbertie + 满意妞 + 相关角色 | 快速决策、解决危机 |
| **里程碑评审** | 按需 | 里程碑达成 | 45-60分钟 | 全员 | 成果验收、经验沉淀 |

### 1.2 会议优先级规则

```
P0-紧急会议 > 日站会 > 周回顾会 > 里程碑评审
```

- **紧急会议可中断任何其他会议**：如系统故障、严重安全事件
- **日站会准时开始**：不因等待而延迟，迟到者事后补阅纪要
- **周回顾会可合并**：如当周无重要事项，可与下周合并

## 二、日站会（Daily Stand-up）

### 2.1 基本信息
- **时间**: 每日 09:00（北京时间）
- **时长**: 10-15分钟（硬性限制）
- **形式**: 异步文字报告 + 必要时语音补充
- **主持**: 满意妞（自动生成报告）

### 2.2 议程模板

```markdown
## 日站会 - YYYY-MM-DD

### 昨日完成（3项以内）
1. [任务ID] [任务名] - [状态变更]
2. ...

### 今日计划（Top 3）
1. [任务ID] [任务名] - [预期成果]
2. ...
3. ...

### 阻塞/风险（如有）
- ⚠️ [阻塞描述] - [需要支持]

### 需要决策（如有）
- ❓ [问题] - [建议方案A/B]

### 备忘（可选）
- 💡 [想法/发现/提醒]
```

### 2.3 执行规则

| 规则 | 说明 |
|------|------|
| **准时生成** | 09:00 前自动生成晨报，无论Egbertie是否在线 |
| **异步优先** | Egbertie可在方便时阅读，不必实时响应 |
| **阻塞必响应** | 如有阻塞项，Egbertie需在4小时内回复 |
| **决策不隔夜** | 需要决策的问题当天必须闭环 |

## 四、紧急会议（Emergency Meeting）

### 4.1 触发条件

满足以下任一条件即触发：

| 级别 | 触发条件 | 响应时间 | 会议形式 |
|------|----------|----------|----------|
| 🔴 **P0-严重** | 系统故障/数据丢失/安全风险 | 即时（任何时间） | 即时通讯/电话 |
| 🟠 **P1-重要** | 关键任务严重逾期/重大阻塞 | 2小时内 | 即时通讯/语音 |
| 🟡 **P2-一般** | 多任务依赖冲突/资源瓶颈 | 4小时内 | 文字异步 |

### 4.2 紧急会议流程

```
触发条件满足
    ↓
满意妞立即通知Egbertie（标注紧急级别）
    ↓
Egbertie确认参会时间（或授权满意妞代决策）
    ↓
召开紧急会议（最快方式）
    ↓
形成决策 → 立即执行 → 事后补纪要
```

### 4.3 紧急会议议程（极简版）

```markdown
## 紧急会议 - YYYY-MM-DD HH:MM
**触发原因**: [一句话描述]
**紧急级别**: P0/P1/P2

### 现状（2分钟）
[发生了什么]

### 影响评估（3分钟）
- 影响范围: [范围]
- 潜在损失: [损失]
- 时间窗口: [剩余时间]

### 可选方案（5分钟）
| 方案 | 优点 | 缺点 | 耗时 | 推荐度 |
|------|------|------|------|--------|
| A | ... | ... | X小时 | ⭐⭐⭐ |
| B | ... | ... | Y小时 | ⭐⭐ |

### 决策（2分钟）
**决定**: 执行方案[X]
**负责人**: [谁]
**截止时间**: [何时]

### 即时行动（会后立即）
- [ ] [行动1]
- [ ] [行动2]
```

## 七、行动项跟踪方式

### 7.1 行动项生命周期

```
会议产生 → 录入任务系统 → 分配负责人 → 执行 → 验收 → 归档
```

### 7.2 行动项记录格式

```markdown
### 行动项清单 - [会议名称] - YYYY-MM-DD

| ID | 行动项 | 负责人 | 优先级 | 截止 | 状态 | 关联任务 |
|----|--------|--------|--------|------|------|----------|
| ACT-001 | [具体行动] | [负责人] | P1 | YYYY-MM-DD | ⏳ 待启动 | [TASK-XXX] |
| ACT-002 | [具体行动] | [负责人] | P2 | YYYY-MM-DD | 🔄 进行中 | [TASK-XXX] |
```

### 7.3 跟踪规则

| 规则 | 说明 |
|------|------|
| **24小时内入库** | 会议结束后24小时内，行动项必须录入TASK_MASTER.md |
| **关联任务ID** | 每个行动项必须关联到TASK_MASTER.md中的任务ID |
| **验收闭环** | 行动项完成后必须验收，验收人即原会议主持人 |
| **逾期预警** | 行动项截止前24小时自动提醒 |

## 八、与现有任务系统的整合

### 8.1 整合架构

```
内部会议机制
    ├── 日站会 ──────┬──→ 更新 TASK_MASTER.md（状态同步）
    │               └──→ 生成 memory/YYYY-MM-DD.md（日志）
    │
    ├── 周回顾会 ────┬──→ 更新 TASK_MASTER.md（新增/调整任务）
    │               ├──→ 更新 MEMORY.md（重要决策）
    │               └──→ 生成 docs/WEEKLY_REVIEW_YYYY-MM-DD.md
    │
    ├── 紧急会议 ────┬──→ 更新 TASK_MASTER.md（紧急任务）
    │               ├──→ 生成 memory/YYYY-MM-DD-紧急.md
    │               └──→ 即时通知（飞书/其他渠道）
    │
    └── 里程碑评审 ──┬──→ 更新 TASK_MASTER.md（里程碑状态）
                    ├──→ 更新 MEMORY.md（里程碑记录）
                    └──→ 生成 docs/MILESTONE_名称_YYYY-MM-DD.md
```

### 8.2 任务状态同步规则

| 会议类型 | 同步内容 | 同步时机 |
|----------|----------|----------|
| 日站会 | 任务状态变更、进度更新 | 会议后即时 |
| 周回顾会 | 新增任务、优先级调整、截止日期变更 | 会议后24小时内 |
| 紧急会议 | 紧急任务创建、依赖关系调整 | 会议后即时 |
| 里程碑评审 | 里程碑状态更新、阶段任务归档 | 会议后24小时内 |

### 8.3 自动化支持

| 自动化项 | 触发条件 | 执行动作 |
|----------|----------|----------|
| 晨报生成 | 每日09:00 | 自动生成并发送日站会报告 |
| 逾期提醒 | 任务截止前24小时 | 在日站会中标注阻塞 |
| 周会通知 | 每周日08:00 | 发送周回顾会提醒和预习材料 |
| 纪要归档 | 会议结束后 | 自动整理并存储到指定位置 |

## 九、会议纪要与存档

### 9.1 纪要标准格式

所有会议纪要必须包含：
1. **会议基本信息**（时间、参与人、会议类型）
2. **议程执行情况**（原定议程vs实际执行）
3. **关键决策**（含决策依据）
4. **行动项清单**（含负责人和截止）
5. **待解决问题**（需要后续跟进的问题）

### 9.2 存档路径

| 会议类型 | 存档路径 | 保留期限 |
|----------|----------|----------|
| 日站会 | `memory/YYYY-MM-DD.md` | 永久 |
| 周回顾会 | `docs/WEEKLY_REVIEW_YYYY-MM-DD.md` | 永久 |
| 紧急会议 | `memory/YYYY-MM-DD-紧急会议.md` | 永久 |
| 里程碑评审 | `docs/MILESTONE_名称_日期.md` | 永久 |

### 9.3 检索规则

- 所有纪要标题必须包含日期
- 关键决策必须打标签 `[DECISION]`
- 行动项必须打标签 `[ACTION]`
- 风险必须打标签 `[RISK]`
