# Skill外来处理SOP（标准作业程序）

**版本**: V1.0  
**生效日期**: 2026-03-15  
**适用范围**: 所有外部skill上传/安装请求  
**制定原则**: 安全第一性原理 + 效益最大化

---

## 一、流程概览

```
用户上传skill文件
        ↓
【阶段1】完整性检查（只做完整性，不做匹配性）
        ↓
【阶段2】综合评估（安全/功能/成本/可维护）
        ↓
【阶段3】自建迭代（不安装，自建替代）
        ↓
交付自建skill + 更新依赖图谱
```

---

## 二、阶段1：完整性检查（精简版）

### 2.1 检查清单

| 检查项 | 方法 | 通过标准 | Token控制 |
|--------|------|----------|-----------|
| 文件数量 | 计数 | 核心文件齐全（meta/skill/run）| <100 tokens |
| 文件可读 | 尝试读取 | 无权限/损坏错误 | <100 tokens |
| 基础结构 | 验证JSON/YAML | 解析成功 | <200 tokens |

### 2.2 不做的事项（节省Token）

❌ **不做匹配性检查**：
- 不与现有skill对比功能重叠
- 不检查版本兼容性
- 不评估安装必要性

❌ **不做深度分析**：
- 不读取完整skill文档
- 不分析代码实现细节
- 不验证功能可用性

### 2.3 输出

```markdown
**完整性检查报告**
- Skill名称: [name]
- 版本: [version]
- 文件数量: [n]个
- 核心文件: ✅/❌
- 结论: [通过/不通过]
```

---

## 三、阶段2：综合评估（第一性原则）

### 3.1 评估维度

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| **安全审计** | 40% | 代码来源、依赖风险、数据流向、权限要求 |
| **功能价值** | 30% | 填补的能力缺口、独特价值、不可替代性 |
| **成本效益** | 20% | 运行成本、维护成本、资源消耗 |
| **可维护性** | 10% | 文档完整度、社区活跃度、自建难度 |

### 3.2 评估方法

**批量评估**：多个同类skill同时评估，提取各自优点

**对比矩阵**：
```
| 功能 | Skill A | Skill B | Skill C | 自建取舍 |
|------|---------|---------|---------|----------|
| 功能1 | ✅ | ❌ | ✅ | 保留 |
| 功能2 | ❌ | ✅ | ✅ | 保留 |
| 风险X | ⚠️ | ❌ | ⚠️ | 剔除 |
```

### 3.3 评估输出

1. **综合评估报告**：功能对比 + 风险识别
2. **自建决策建议**：功能取舍 + 架构建议
3. **风险清单**：需剔除的安全隐患

## 四、阶段3：自建迭代

### 4.1 自建原则

| 原则 | 说明 |
|------|------|
| **功能精华** | 取多个skill优点，形成最优方案 |
| **风险清零** | 剔除所有识别的安全风险 |
| **成本最优** | 零额外API费用，纯本地运行 |
| **可控可审计** | 全部代码可审计，有完整日志 |

### 4.2 自建流程

```
评估报告
    ↓
架构设计（目录结构 + 接口定义）
    ↓
核心实现（MVP版本）
    ↓
安全加固（沙箱 + 审计 + 限制）
    ↓
测试验证（示例场景）
    ↓
文档交付（使用说明 + 安全策略）
```

### 4.3 输出物

| 文件 | 说明 |
|------|------|
| `skill.json` | Skill元数据 |
| `[module].py` | 核心模块 |
| `sandbox/` | 沙箱隔离代码 |
| `security/` | 安全控制代码 |
| `examples/` | 使用示例 |
| `SECURITY.md` | 安全策略文档 |
| `CHANGELOG.md` | 更新日志 |

## 五、文件清理机制

### 5.1 清理触发条件

| 条件 | 动作 |
|------|------|
| skill评估完成7天后 | 删除原始上传文件 |
| 自建skill交付后 | 保留评估报告，删除中间文件 |
| 磁盘使用率>80% | 紧急清理历史下载文件 |

### 5.2 清理范围

```
/root/openclaw/kimi/downloads/
├── 保留：最近7天的上传文件
├── 删除：7天前的skill原始文件
└── 保留：评估报告、自建skill代码
```

### 5.3 清理脚本

```bash
#!/bin/bash
# /root/.openclaw/workspace/scripts/cleanup_skill_downloads.sh

DOWNLOAD_DIR="/root/openclaw/kimi/downloads"
RETENTION_DAYS=7

# 删除7天前的文件
find "$DOWNLOAD_DIR" -type f -mtime +$RETENTION_DAYS -delete

# 记录清理日志
echo "[$(date)] Cleanup completed" >> /var/log/skill_cleanup.log
```

### 5.4 定时任务

```cron
# 每周日凌晨3点执行清理
0 3 * * 0 /root/.openclaw/workspace/scripts/cleanup_skill_downloads.sh
```
