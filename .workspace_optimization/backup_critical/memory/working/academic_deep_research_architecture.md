## 第一性原则拆解

### 核心目标
构建一个具备**认知脚手架（Cognitive Scaffolding）**的研究代理系统，将AI从"信息检索器"提升为"认知协作伙伴"。

### 本质问题
如何避免AI生成内容的幻觉、溯源不清、证据链断裂等问题？

### 解决方案
通过**六层嵌套模型**强制方法论纪律（Methodological Discipline）：

```
范式锚定 → 情报网格 → 证据链锻造 → 多维分析 → 逻辑架构 → 迭代优化
```

---

## 六层嵌套模型详解

### 第一层：范式锚定（Paradigm Anchoring）

**核心**: 明确研究的认识论、方法论、伦理基础

**三要素**:
- 认识论: 实证主义/解释主义/批判理论/设计科学
- 方法论: 定量/质性/混合/计算实验
- 学术伦理: COPE/TCPS 2/Helsinki Declaration

**认知谦逊原则**:
- 标注知识截止日期
- 置信度编码: High/Medium/Low/Speculative
- 证据链断裂时启动红队思维

---

### 第二层：情报网格构建（Intelligence Grid Construction）

**核心**: 多源三角验证，禁止单一信源

**三阶段**:
1. **横向扫描**（Breadth-first）
   - ≥3个独立搜索查询
   - 覆盖: 学术数据库 + 灰色文献 + 行业数据 + 多语种

2. **纵向钻取**（Depth-first）
   - 5层溯源: 原始数据→一手分析→二手综述→媒体解读→AI合成
   - 禁止直接使用AI合成层

3. **冲突检测**（Conflict Detection）
   - 主动检索反方证据
   - 标注争议点和共识度

---

### 第三层：证据链锻造（Evidence Chain Forging）

**核心**: 所有事实性声明必须可追溯

**溯源编码系统**:
```
^ID^ 作者. (年份). 标题. 期刊/出版社. DOI/URL. [检索日期: YYYY-MM-DD]
```

**证据等级标签**:
- [P1] 原始实验/官方统计/法律条文
- [P2] 系统综述/Meta分析/顶刊研究
- [P3] 工作论文/行业报告/专家访谈
- [P4] 新闻/百科/未经验证（仅作背景）

**反幻觉机制**:
- 禁止"合理化虚构"（Confabulation）
- 引入"不确定性云": [Data Gap: 需要补充X]
- 统计显著性标注: 样本量、置信区间、p值

---

### 第四层：多维分析框架（Multi-dimensional Analytical Framework）

**核心**: 强制多维度视角，避免单一视角偏差

**五维度矩阵**:
1. **时间维度**: 历史演进→当下截面→未来推演（三情景）
2. **空间维度**: 全球比较→区域差异→微观-中观-宏观
3. **学科维度**: 主学科 + 交叉视角1 + 交叉视角2 → 涌现特性
4. **利益相关者**: 政府/市场/公民社会/个体/非人类实体
5. **方法论三角**: 定量 + 定性 + 计算

---

### 第五层：逻辑架构工程（Logical Architecture Engineering）

**核心**: 钻石模型论证结构

```
引言（Hook→Gap→Question→Significance）
    ↓
理论框架 + 概念操作化
    ↓
方法论 + 局限性声明
    ↓
多维度发现（链接证据链）
    ↓
讨论（Agree/Extend/Challenge/Synthesize）
    ↓
原创性声明（理论贡献+实践价值）
    ↓
批判性反思（偏见/盲区/伦理）
```

**逻辑防错检查点**:
- 因果推断: 区分Correlation/Causation/Mechanism
- 概念滑移: 术语定义一致性检查
- 代表性偏差: 样本选择偏差主动披露

---

### 第六层：迭代优化协议（Iterative Refinement Protocol）

**核心**: 三轮自我修正循环

**第一轮**: 信息饱和
- 检查核心数据库是否遗漏
- 检查近3个月预印本

**第二轮**: 逻辑压力测试
- 魔鬼代言人: 假设审稿人是领域 skeptics
- 补充反驳论证与稳健性检验

**第三轮**: 元认知审计
- 引用密度≥3个独立信源/千字
- 区分"本研究发现" vs "前人复述" vs "推测"
- 格式合规检查

## 实操执行指令（标准化流程）

### 阶段1: 研究大纲与证据地图

1. **领域扫描**: web_search识别5+核心权威机构/期刊/数据库
2. **原始验证**: web_open_url验证关键数据（不依赖摘要）
3. **证据地图**: 表格列出核心主张-证据-置信度-反方观点

### 阶段2: 全文撰写

4. **引用锚定**: 每完成一段，立即检查^ID^标注
5. **红队审查**: 生成3个专家批评意见并撰写回应

### 阶段3: 质量审计

6. **元认知审计**: 引用密度、原创性边界、格式合规
7. **人工介入**: 标记需人工验证的关键环节

## 输出规范

### 格式要求
- 总长度: 8000-12000字
- 结构化摘要: Background/Methods/Results/Conclusion
- 关键词: 5-7个
- 参考文献: ≥30条，近5年≥40%
- 格式: Markdown，标题层级清晰

### 禁忌行为
- ❌ "多项研究表明..."（不指明具体研究）
- ❌ "众所周知..."（必须提供溯源）
- ❌ 将推测包装为结论
