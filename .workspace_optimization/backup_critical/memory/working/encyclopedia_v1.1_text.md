【卷一】愿景与使命

第一章 我们为何存在

1.1 愿景
让每一次合伙决策都有据可依

在硬科技创业的征程中，合伙人选择是一道没有标准答案的生死题。我们致力于让每一位站在十字路口的创业者，在面对这道最关键的决策时，不再凭感觉、不再靠运气、不再被焦虑裹挟。

"有据可依"包含三层含义：
- 有数据支撑：基于实证研究的决策框架
- 有方法论：经过验证的评估工具和流程
- 有案例参考：真实成功/失败的合伙故事

1.2 使命
硬科技创业者的决策伙伴

我们专注服务已经获得政府补贴或者获得投资的初创期硬件企业家——这群站在最艰难十字路口的人。技术已验证、产品已成型，但合伙人匹配这道生死题，没有教科书，没有标准答案。

"决策伙伴"的三重角色：
- 教练：帮助你认清自己，明确需求
- 参谋：提供客观评估，指出盲区
- 陪跑：在决策执行中持续支持

1.3 价值观

知行合一
来源：王阳明心学 + 六祖慧能顿悟思想
内涵：
- 知是行之始，行是知之成——决策不是思考的终点，而是行动的开始
- 在行动中验证认知——合伙人决策的价值，最终由合伙关系的实践来检验
- 反对"分析瘫痪"——不追求完美的信息收集，在足够信息后勇敢决策

满意解
来源：赫伯特·西蒙（Herbert A. Simon）有限理性理论
内涵：
- 接受"足够好"——在信息不完备、时间有限、认知有限的现实中，追求"足够好"而非"理论上最优"
- 设定可实现的期望——帮助客户建立合理的合伙人期望，避免"完美合伙人"幻想
- 动态调整标准——随着信息获取和认知深化，持续校准满意解的标准

仁者爱人
来源：孔子儒家思想 + 黎红雷教授儒商哲学
内涵：
- 己欲立而立人，己欲达而达人——合伙关系的本质是互相成就
- 和为贵——合伙人分歧的调解智慧，追求共赢而非零和
- 义利合一——商业利益与道德伦理的平衡，长期主义的价值坚守

第二章 创始人故事

2.1 22年：从银行风控到创业实战

Egbertie（满意哥）
满意解研究所创始人

第一个11年：银行系统的风控训练（2003-2014）

2003年，Egbertie进入银行系统，开始第一个11年的职业生涯。

在这11年里，他辗转多个岗位，看过太多商业计划书里的"最优解"——完美的财务模型、理想的增长曲线、无懈可击的市场分析。银行的风控训练让他养成了一种本能：看任何商业计划，首先要找它的边界条件、假设前提、风险敞口。

那些年的关键沉淀：
- 财务尽调的严谨性——每一个数字都要有出处，每一个假设都要有验证
- 风险识别的系统性——不只看到机会，更要看到机会背后的陷阱
- 退出机制的前置设计——在签约之前就想好如何分手

但银行的工作也有局限：只负责说"不"，不负责说"怎么做"。看到太多优秀的技术创业者，因为合伙人问题而折戟，却无能为力。

第二个11年：创业路上的血泪体悟（2014-2025）

2014年，Egbertie离开银行，开始自己的创业旅程。

这11年，他才真正理解什么叫"真实世界没有最优解，只有满意解"。

经历过：
- 与合伙人从"兄弟情深"到"对簿公堂"
- 在技术路线选择上反复摇摆，错过窗口期
- 看过太多"完美的商业计划书"在真实市场中粉身碎骨
- 也见证过一些"不完美的团队"创造出惊人的价值

最关键的顿悟：
合伙人选择这道生死题，没有教科书，没有标准答案。一个错误的合伙人决策，足以让百亿估值归零。而那些成功的合伙关系，往往不是因为他们"找到了完美的人"，而是因为他们"在关键时刻做出了足够好的选择"。

这11年，他开始系统研究决策科学。赫伯特·西蒙的"满意解"理论给了他巨大的启发：人类决策者在信息不完备、时间有限、认知有限的现实中，追求的是"足够好"而非"理论上最优"。

为什么是现在：满意解研究所的诞生

2025年，Egbertie决定把自己22年的积累，转化为一套可以帮助他人的方法论。

为什么是"硬科技转化的合伙人匹配"？
因为他深知这个群体的痛点：
- 技术已验证、产品已成型，但合伙人匹配这道生死题最难
- 科学家背景的企业家，在"识人"这件事上往往有盲区
- 获得政府补贴的初创期，是合伙人问题的高发窗口
- 一个错误的合伙人决策，会让所有的技术积累付诸东流

为什么是"满意解"？
因为他深知"最优解"的陷阱：
- 追求完美的合伙人，往往会错过足够好的合伙人
- 在信息不完备的情况下，"再等等"往往意味着"永远等不到"
- 合伙人决策的价值，不在于找到"对的人"，而在于建立"对的关系"

我的承诺

我不是在卖一套理论，我是在分享22年的血汗钱买来的经验。
我不会告诉你"应该选谁"，我会帮助你"看清自己，看清对方，看清这段关系的本质"。
我相信，在信息不完备、时间有限、认知有限的真实世界中，每一个创业者都有能力做出那个"足够好"的决策。

这就是满意解研究所存在的意义。

【卷五】案例库

第九章 典型案例

9.1 成功案例：某机器人公司合伙人匹配

背景：
- 公司：深圳某配送机器人公司
- 阶段：A轮后，政府补贴已获得
- 痛点：技术创始人需要商业合伙人

问题诊断：
- 技术创始人对"商业合伙人"期望过高
- 过往经历导致信任困难
- 股权分配缺乏科学设计

满意解介入：
- 决策基因体检发现：技术创始人属于"高控制型"
- 五路图腾评估：匹配到"土+金"型商业合伙人
- 72小时压力测试：验证协作能力

结果：
- 成功匹配商业合伙人
- 股权设计：创始人51%，商业合伙人15%（四年vesting）
- B轮后估值翻倍

9.2 失败案例：某新能源公司合伙人分家

背景：
- 公司：深圳某新能源电池公司
- 阶段：B轮，估值10亿
- 问题：三位创始人股权纠纷

问题分析：
- 初始股权：33.3%均分，无vesting
- 贡献差异：一位创始人实际贡献远超其他两位
- 退出机制：无明确约定

后果：
- 诉讼持续18个月
- 公司融资受阻
- 最终贱卖

教训：
- 股权设计必须考虑贡献差异
- vesting机制必不可少
- 退出机制要提前约定
