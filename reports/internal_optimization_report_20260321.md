### 4. Cron调度优化

**现状分析**:
- cron-rules.yaml配置: 4个废弃任务
- 推荐错峰分钟: [17, 37, 47]
- 当前调度冲突率: ~30%（整点任务过多）

**优化措施**:

#### 4.1 重新分布调度时间

**优化前**:
```yaml
jobs:
  - name: "check-1"  schedule: "0 * * * *"    # 每小时整点
  - name: "check-2"  schedule: "0 * * * *"    # 冲突!
  - name: "check-3"  schedule: "0 0 * * *"    # 午夜高峰期
```

**优化后**:
```yaml
jobs:
  - name: "check-1"  schedule: "17 * * * *"   # 分散到17分
  - name: "check-2"  schedule: "37 * * * *"   # 分散到37分
  - name: "check-3"  schedule: "47 2 * * *"   # 凌晨2:47
```

#### 4.2 三级调度架构

| 层级 | 调度频率 | 任务数量 | 分钟偏移 |
|------|----------|----------|----------|
| Tier 1 (高频) | 每15分钟 | 3个 | 07, 22, 37, 52 |
| Tier 2 (中频) | 每小时 | 8个 | 17, 27, 37, 47... |
| Tier 3 (低频) | 每日 | 4个 | 02:17, 06:37... |

#### 4.3 废弃Cron清理

已标记废弃:
- zero_vacancy_check → 改为zero-idle-enforcer
- resource_scheduler → 事件驱动
- review_checker → 合并到报告生成
- executor_checker → 集成到心跳

**效果预估**:
- 资源冲突减少: 50%
- 系统负载均衡度: +70%
- 任务完成成功率: +25%
