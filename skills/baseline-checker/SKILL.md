# baseline-checker Skill V5标准版本

## S1: 全局考虑

### 输入
- 待检查系统/项目
- 9项基线标准配置
- 检查频率设置

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 系统管理员、项目负责人 |
| **事** | 9项基线自动检查、偏差报告、修复建议 |
| **物** | 配置文件、代码库、文档、日志 |
| **环境** | 开发/测试/生产环境 |
| **外部集成** | Git、CI/CD、监控系统等 |
| **边界情况** | 部分基线不适用、检查失败、误报 |

---

## S2: 系统考虑

### 9项基线标准
1. 配置管理 - 配置版本化、可追溯
2. 文档完整性 - 核心文档存在且更新
3. 代码规范 - 符合团队规范
4. 测试覆盖 - 单元/集成测试达标
5. 安全检查 - 无高危漏洞
6. 性能基线 - 响应时间/资源使用达标
7. 日志规范 - 结构化日志、可查询
8. 备份策略 - 定期备份、可恢复
9. 监控告警 - 关键指标覆盖

### 处理流程
```
配置加载 → 逐项检查 → 结果汇总 → 偏差分析 → 报告生成 → 修复建议
```

## S5: 自我验证

### 质量指标

- 误报率: <5%

### 局限

- 无法覆盖所有业务场景

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 基线不适用 | 标记N/A，不计入总分 |
| 检查失败 | 记录错误，继续其他项 |
| 阈值边界 | 保守判定，提示人工复核 |
