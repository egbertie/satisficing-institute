## 三、监控规则

### 3.1 Token监控（OpenClaw）

#### 每日检查（Heartbeat）
```json
{
  "checkTime": "每日12:00",
  "checkItems": [
    "Token余量百分比",
    "本周已消耗Token",
    "本周剩余Token预算",
    "日均消耗趋势"
  ],
  "alertThreshold": {
    "warning": 30,
    "critical": 15,
    "emergency": 5
  }
}
```

#### 分级响应
| Token余量 | 状态 | 响应措施 |
|-----------|------|----------|
| > 50% | 🟢 健康 | 正常运行，按计划执行 |
| 30%-50% | 🟡 注意 | 优化任务优先级，减少非必要消耗 |
| 15%-30% | 🟠 预警 | 仅执行P0任务，暂停学习/维护类任务 |
| < 15% | 🔴 紧急 | 完全暂停，等待周期重置或用户指令 |

## 四、执行规则

### 4.1 任务优先级与资源匹配

#### P0 - 紧急任务（立即执行）
| 资源 | 使用策略 |
|------|----------|
| OpenClaw Token | 优先使用，不限制 |
| Kimi深度研究 | 如需要，立即使用 |
| WPS | 辅助文档处理 |

#### P1 - 重要任务（当日完成）
| 资源 | 使用策略 |
|------|----------|
| OpenClaw Token | 正常使用，控制在日均预算内 |
| Kimi深度研究 | 每日1次，用于核心研究主题 |
| WPS | 文档生成/润色 |

#### P2 - 一般任务（本周完成）
| 资源 | 使用策略 |
|------|----------|
| OpenClaw Token | 低消耗方式，批量处理 |
| Kimi Agent池 | 文档/表格处理 |
| WPS | 模板/格式转换 |

#### P3 - 低优先级（有空时）
| 资源 | 使用策略 |
|------|----------|
| OpenClaw Token | Token > 50%时才执行 |
| Kimi | 不主动使用，保留额度 |
| WPS | 云同步/备份 |

### 4.2 每日资源分配（标准工作日）

```
09:00 - 12:00 (上午)
├── OpenClaw: ~3K Token（核心任务）
├── Kimi深度研究: 1次（研究主题）
└── WPS: 文档润色/模板

12:00 - 14:00 (午休)
├── OpenClaw: ~1K Token（轻维护）
└── Kimi/WPS: 不使用

14:00 - 18:00 (下午)
├── OpenClaw: ~3K Token（核心任务）
├── Kimi Agent池: 1-2次（文档/表格）
└── WPS: PDF转换/云同步

18:00 - 22:00 (晚间)
├── OpenClaw: ~1K Token（整理归档）
└── Kimi/WPS: 按需使用

22:00 - 06:00 (夜间)
├── OpenClaw: ~2K Token（批量任务）
├── Kimi: 不使用（保留额度）
└── WPS: 云备份
```

### 4.3 周期末策略（下周三前）

#### 最后24小时（下周二12:00后）
| 资源 | 策略 |
|------|------|
| OpenClaw Token | 如剩余较多，执行积压的P2/P3任务 |
| Kimi深度研究 | 如剩余次数多，执行额外研究主题 |
| Kimi PPT | 如剩余次数多，制作备用PPT模板 |
| Kimi Agent池 | 执行文档整理/知识图谱更新 |
