# quality-closure Skill V5标准版本

## S1: 全局考虑

### 输入
- 质量问题报告
- 修复方案
- 验证标准

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 质量管理员、开发者、测试人员 |
| **事** | 问题追踪、修复验证、闭环确认 |
| **物** | 问题单、修复记录、验证报告 |
| **环境** | 项目周期、发布节奏 |
| **外部集成** | 任务管理系统、测试系统 |
| **边界情况** | 修复不彻底、问题复发、资源不足 |

---

## S2: 系统考虑

### 处理流程
```
问题发现 → 记录建档 → 分配修复 → 修复实施 → 验证测试 → 闭环归档
    ↑                                                  ↓
    └──────────── 复发追踪 ← 效果监控 ←───────────────┘
```

### 状态流转
```
待处理 → 处理中 → 待验证 → 已验证 → 已关闭
  ↓         ↓         ↓         ↓
已分配   已修复   验证失败   复发重开
```

---

---

## S4: 自动化集成

### 触发条件
- 问题修复提交
- 自动测试通过
- 人工验收确认

### 追踪机制
- 7天复发监控
- 关联问题标记
- 趋势分析

---

## S5: 自我验证

### 质量指标
- 闭环率: >95%
- 复发率: <5%
- 平均闭环时间: <3天

---

## S6: 认知谦逊

### 局限
- 无法保证100%不复发
- 验证覆盖有限
- 依赖测试质量

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 修复不彻底 | 验证失败，退回修复 |
| 问题复发 | 重开问题，标记复发 |
| 验证遗漏 | 监控期发现，触发告警 |
| 资源不足 | 延期处理，记录原因 |
