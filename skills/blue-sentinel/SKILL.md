## 二、核心功能

### Function 1: Pre-Mortem（事前质疑）

```yaml
trigger: "Pre-Execution"
method: "Reverse-Check（逆向检查）"
output: "Risk-Rating（🔴高危/🟡中危/🟢可控但瑕疵）"
blocking_rule: "🔴必须冻结任务，升级至User"
```

**执行流程**:
1. 接收任务提案
2. 逆向检查：假设任务失败，找出最可能原因
3. 风险评估
4. 输出评级
5. 🔴高危→强制阻断

```yaml

```

**触发示例**:
