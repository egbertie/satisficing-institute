## 清单说明

本清单是蓝军所有审计工作的**强制检查项**。任何审计报告必须通过以下10项检查，方可输出。

**使用方式**:
1. 每项检查必须明确标注通过/失败/不适用
2. 失败项必须说明具体问题和证据
3. 🔴 硬否决项失败→任务冻结
4. 🟡 要求标注项失败→必须标注后放行

---

## 10项认知审计检查

### 1. 信源独立性 (Source Independence) 🔴

**检查内容**: 关键数据是否来自单一信源？

**判定标准**:
- ❌ 单一信源 → 硬否决
- ✅ 两个及以上独立信源 → 通过
- ⚠️ 信源看似不同但实际同源 → 标注风险

**输出要求**:
```
[信源独立性检查]
状态: ❌ 失败
问题: 关键数据仅来自单一信源（某公众号文章）
证据: 未找到其他独立验证
阻断: 🔴 硬否决 - 必须补充独立信源
```

---

### 2. 时效性 (Timeliness) 🟡

**检查内容**: 数据是否过期？

**判定标准**:
- ❌ 数据 > 6个月 → 必须标注过期
- ❌ 数据 > 1年 → 建议不使用
- ⚠️ 快速变化领域（如AI/股票）> 1个月 → 标注

**输出要求**:
```
[时效性检查]
状态: 🟡 要求标注
数据时点: 2025-06-01（9个月前）
建议: 标注[数据截至2025年6月，可能已变化]
```

---

### 3. 因果混淆 (Causal Confusion) 🟡

**检查内容**: 相关性是否被误认为因果性？

**判定标准**:
- ❌ "A和B相关，因此A导致B" → 要求澄清
- ❌ 无控制变量的因果论断 → 要求补充

**输出要求**:
```
[因果混淆检查]
状态: 🟡 要求澄清
问题: "销量增长因为价格上涨" - 可能混淆因果
建议: 改为"销量增长伴随价格上涨，因果方向需进一步验证"
```

---

### 4. 幸存者偏差 (Survivorship Bias) 🟡

**检查内容**: 是否只分析了成功案例而忽略失败案例？

**判定标准**:
- ❌ 仅引用成功案例 → 要求补充失败案例
- ❌ 样本来自"已成功者" → 标注偏差

**输出要求**:
```
[幸存者偏差检查]
状态: 🟡 要求补充
问题: 仅分析了3家成功创业公司
缺失: 未考虑同期失败的97家公司
建议: 补充失败案例分析，或标注[基于幸存者样本]
```

---

### 5. 基底率忽视 (Base Rate Neglect) 🟡

**检查内容**: 是否忽略了基础概率？

**判定标准**:
- ❌ 罕见病检测阳性→直接判定患病 → 错误
- ❌ 未考虑先验概率的条件概率计算 → 要求重算

**输出要求**:
```
[基底率忽视检查]
状态: 🟡 要求计算
问题: "检测阳性，因此患病概率高" - 忽略基底率
实际: 该病发病率0.1%，检测准确率95%
正确: 阳性后实际患病概率约16%
```

---

### 6. 锚定效应 (Anchoring Effect) 🟢

**检查内容**: 是否被初始数字不恰当影响？

**判定标准**:
- ⚠️ 谈判/估价场景的第一个数字 → 提示
- ⚠️ 对比时使用的参照数字 → 检查合理性

**输出要求**:
```
[锚定效应检查]
状态: 🟢 提示
注意: 分析使用了"行业平均"作为锚点
建议: 确认该锚点是否适用于当前场景
```

---

### 7. 确认偏误 (Confirmation Bias) 🟡

**检查内容**: 是否只寻找支持证据而忽略反对证据？

**判定标准**:
- ❌ 仅列出支持观点的证据 → 要求补充反方
- ❌ 选择性引用数据 → 硬否决

**输出要求**:
```
[确认偏误检查]
状态: 🟡 要求补充
问题: 仅提供了支持"市场增长"的证据
缺失: 未考虑经济衰退、政策变化等反方证据
建议: 补充至少2个反方观点或风险因素
```

---

### 8. 语言腐败 (Language Corruption) 🔴

**检查内容**: 是否存在定义模糊或偷换概念？

**判定标准**:
- ❌ 关键术语无定义 → 硬否决
- ❌ 同一术语在不同位置含义不同 → 硬否决
- ❌ 模糊表述（如"高质量发展"无指标）→ 要求定义

**输出要求**:
```
[语言腐败检查]
状态: ❌ 失败
问题: 核心术语"用户活跃度"未定义
阻断: 🔴 硬否决 - 必须明确定义（DAU/MAU/时长？）
```

---

### 9. 数学谬误 (Mathematical Error) 🔴

**检查内容**: 百分比变化等数学计算是否正确？

**判定标准**:
- ❌ 100→150是+50%（不是+150%） → 硬否决
- ❌ 百分比加减直接运算 → 硬否决
- ❌ 复合增长率计算错误 → 硬否决

**输出要求**:
```
[数学谬误检查]
状态: ❌ 失败
问题: "从100增长到150，增长150%"
正确: "从100增长到150，增长50%"
阻断: 🔴 硬否决 - 必须更正计算
```

---

### 10. 样本偏差 (Sampling Bias) 🟡

**检查内容**: 样本是否具有代表性？

**判定标准**:
- ❌ 仅调查愿意回答的人 → 标注偏差
- ❌ 样本来自特定群体却推广到全体 → 要求限定范围
- ❌ 样本量过小（N<30）却做统计推断 → 要求说明

**输出要求**:
```
[样本偏差检查]
状态: 🟡 要求说明
问题: 样本来自一线城市用户，但结论推广到全国
建议: 标注[基于一线城市样本]，或补充其他地区数据
```

## S7 反方观点（清单自身的质疑）

**反方观点1: 清单过于僵化**
- 观点: 强制10项检查可能导致形式主义审计
- 应对: 允许标注"不适用"，但必须有理由

**反方观点2: 检查项可能不完整**
- 观点: 10项无法覆盖所有认知偏差
- 应对: 定期更新清单，基于实际案例补充

**反方观点3: 阻断级别主观**
- 观点: 🔴/🟡/🟢 的划分可能不适用于所有场景
- 应对: 人类保留最终调整阻断级别的权力
