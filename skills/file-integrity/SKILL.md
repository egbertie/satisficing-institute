# file-integrity Skill V5标准版本

## S1: 全局考虑

### 输入
- 文件路径或目录
- 校验算法(MD5/SHA256等)
- 基准哈希值(可选)

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 系统管理员、安全人员 |
| **事** | 文件完整性校验、篡改检测、版本比对 |
| **物** | 文件、哈希值、校验记录 |
| **环境** | 文件系统权限、存储空间 |
| **外部集成** | 日志系统(可选) |
| **边界情况** | 权限不足、大文件、哈希碰撞 |

---

## S2: 系统考虑

### 处理流程
```
选择算法 → 计算哈希 → 比对基准 → 生成报告 → 记录日志
```

### 故障处理
- **权限不足**: 跳过并记录
- **大文件**: 分块哈希
- **哈希碰撞**: 使用SHA256替代MD5

---

## S3: 输出规范

### 校验报告
```json
{
  "file": "path/to/file",
  "algorithm": "sha256",
  "hash": "a3f5b2...",
  "expected": "a3f5b2...",
  "match": true,
  "timestamp": "2026-03-22T09:00:00+08:00"
}
```

---

## S4: 自动化集成

### 定时校验
- 每周全量校验
- 关键文件实时监控(可选)

---

## S5: 自我验证

### 质量指标
- 准确率: 100%
- 覆盖率: >95%

---

## S6: 认知谦逊

### 局限
- 只能检测内容变更，无法识别变更原因
- 哈希碰撞概率极低但存在
- 不加密，仅校验完整性

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 文件被篡改 | 哈希不匹配，告警 |
| 权限拒绝 | 跳过并记录 |
| 大文件 | 流式计算哈希 |
| 基准缺失 | 生成新基准 |
