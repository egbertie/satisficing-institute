# quality-assurance Skill V5标准版本

## S1: 全局考虑

### 输入
- 质量保证计划
- 检查清单
- 质量标准

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | QA工程师、团队成员 |
| **事** | 质量检查、过程监控、标准执行 |
| **物** | 检查清单、质量记录、审计报告 |
| **环境** | 开发流程、发布节奏 |
| **外部集成** | CI/CD、测试平台 |
| **边界情况** | 紧急发布、标准冲突、资源不足 |

---

## S2: 系统考虑

### QA流程
```
计划制定 → 标准培训 → 过程检查 → 问题跟踪 → 改进验证 → 报告输出
```

### 检查类型
- 代码审查
- 文档审查
- 流程合规
- 发布检查

---

## S3: 输出规范

### QA报告
```json
{
  "period": "2026-03",
  "items_checked": 50,
  "passed": 45,
  "failed": 5,
  "pass_rate": 90,
  "trends": "improving",
  "top_issues": [...]
}
```

---

## S4: 自动化集成

### 自动检查
- 提交规范
- 测试覆盖
- 安全扫描
- 依赖检查

---

## S5: 自我验证

### 质量指标
- 检查覆盖率: >95%
- 问题发现率: >85%
- 改进闭环率: >90%

---

## S6: 认知谦逊

### 局限
- QA不能替代自我负责
- 无法发现所有问题
- 标准可能滞后

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 紧急发布 | 快速检查，标记风险 |
| 资源不足 | 优先级排序，核心项必查 |
| 标准冲突 | 升级裁决，记录决策 |
