# error-handler Skill V5标准版本

## S1: 全局考虑

### 输入
- 错误类型（API/网络/文件/逻辑）
- 错误详情
- 上下文信息

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 开发者、运维人员 |
| **事** | 错误捕获、分类、处理、报告 |
| **物** | 错误日志、告警、恢复策略 |
| **环境** | 系统状态、网络环境 |
| **外部集成** | 通知渠道（可选） |
| **边界情况** | 级联错误、未知错误、资源耗尽 |

---

## S2: 系统考虑

### 处理流程
```
错误捕获 → 分类识别 → 策略选择 → 执行处理 → 记录日志 → 通知报告
```

### 故障处理
- **级联错误**: 防止无限重试
- **未知错误**: 记录并上报
- **资源耗尽**: 优雅降级

---

## S3: 输出规范

### 错误报告格式
```json
{
  "error_id": "err_xxx",
  "timestamp": "2026-03-22T09:00:00+08:00",
  "type": "api_timeout",
  "severity": "warning|error|critical",
  "message": "错误描述",
  "context": {},
  "action_taken": "自动重试3次",
  "status": "resolved|pending|escalated"
}
```

---

## S4: 自动化集成

### 错误分类
| 类型 | 处理策略 |
|------|----------|
| API超时 | 重试3次，退避策略 |
| 网络错误 | 立即重试1次 |
| 文件错误 | 记录并跳过 |
| 逻辑错误 | 上报并停止 |

---

## S5: 自我验证

### 质量指标
- 错误捕获率: 100%
- 误报率: <5%
- 恢复成功率: >80%

---

## S6: 认知谦逊

### 局限
- 无法预测所有错误
- 自动恢复可能不彻底
- 未知错误需人工介入

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 连续失败 | 退避策略，避免雪崩 |
| 未知错误类型 | 记录并上报 |
| 日志写入失败 | 内存缓存，稍后重试 |
| 通知渠道失效 | 本地记录，不阻塞 |
