## 二、关键认知修正

### ❗ 重要：Agent额度是共享池

**Agent功能包含**（共享同一个额度池）：
- ✅ Agent（智能体）
- ✅ 网站（Web解析）
- ✅ 文档（文档处理）
- ✅ 表格（数据分析）
- ❌ 深度研究（独立额度）
- ❌ PPT（独立额度）

**示例**（Allegretto 40次/月）：
```
你今天用了：
- 网站解析 5次
- 文档处理 3次
- 表格分析 2次
= 剩余Agent额度：40 - 10 = 30次

深度研究用了：2次（独立计算，剩余38次）
PPT用了：1次（独立计算，剩余39次）
```

## 三、Kimi Claw 特别说明

### 额度规则
- Kimi Claw 与 **Kimi Code 共享额度**
- 按会员档位下发对应额度
- **不支持同时运行两个及以上任务**

### 过期风险 ⚠️
- 会员过期后，**云资源保留7天**
- 7天后永久关停并释放资源
- **相关数据和资源都将清空**

## 四、满意解项目使用策略

### 基于Allegretto（40次/月）的分配建议

#### 每周分配（按4周计算）

| 功能 | 每周额度 | 使用场景 | 策略 |
|------|----------|----------|------|
| **深度研究** | 10次 | 每日研究主题 | 每天用1-2次，保留缓冲 |
| **PPT** | 10次 | 路演/讲座/汇报 | 重要场合使用，提前预约 |
| **Agent池** | 10次 | 网站/文档/表格 | 优先文档，次优网站 |

#### 具体使用计划

**每日例行（工作日）**:
```
09:00  深度研究 ×1（研究主题）
       ↓
18:00  文档处理 ×1-2（整理资料）
       ↓
按需   网站解析（政策查询）
```

**每周重点**:
```
周一: 深度研究 ×2（本周重点主题）
周三: PPT ×1（如有路演/汇报需求）
周五: 文档批量处理 ×3-5（整理一周资料）
```

**月度储备**:
```
深度研究: 40次 = 每日1次 × 30天 + 10次缓冲
PPT: 40次 = 每周1次 × 4周 + 36次缓冲（充足）
Agent: 40次 = 每日1-2次 × 30天（需精打细算）
```

## 五、关键限制与注意事项

### 使用限制
1. **消息上限**：高需求时段可能有限制
2. **并发限制**：
   - Andante: 不支持并发
   - Moderato及以上: 支持同时运行两个任务（同类型）
   - Kimi Claw: 始终不支持并发

### 额度消耗规则
- 不同功能因**任务复杂度**不同，消耗额度可能不同
- 具体以实际使用时的页面提示为准

### 失效规则 ⚠️
- 会员权益**按月周期发放**
- 有效期内未使用的额度**到期自动失效**
- **不予退款、结转、退换**

## 六、优化使用建议

### 1. Agent池优化（最紧缺）

**优先级排序**:
```
P0: 文档处理（学术研究必需）
P1: 表格分析（决策评估必需）
P2: 网站解析（可部分替代为手动搜索）
```

**节省策略**:
- 批量处理文档（一次上传多个）
- 复杂表格分析用Agent，简单表格手动处理
- 网站解析仅用于获取政策原文

### 2. 深度研究优化（核心）

**高质量使用**:
- 每天1个高质量主题 > 每天3个浅层主题
- 提前准备好详细的提示词
- 生成后立即发送给OpenClaw验证

### 3. PPT优化（充足）

**40次/月非常充足**:
- 投资人路演（重要，用1次）
- 专家讲座（中等，用1次）
- 研究汇报（频繁，可用多次）
- 内部讨论（可用简易版，节省额度）

## 七、与OpenClaw的协作优化

### Token节省策略

| 任务 | 执行方 | 消耗 |
|------|--------|------|
| 深度研究初稿 | Kimi深度研究 | Kimi额度（40次/月）|
| 论文验证优化 | OpenClaw | Claw Token（低）|
| 文档批量解析 | Kimi文档 | Kimi Agent额度 |
| 资料整理归档 | OpenClaw | Claw Token（极低）|
| PPT生成 | Kimi PPT | Kimi额度（40次/月）|
| PPT内容优化 | OpenClaw | Claw Token（中）|

### 最佳协作流程

```
1. 深度研究
   Kimi深度研究(消耗Kimi额度) 
   → OpenClaw验证(消耗Claw Token)
   → 2.0学术严谨版

2. 文档处理
   Kimi文档解析(消耗Kimi Agent额度)
   → OpenClaw整理归档(消耗Claw Token)
   → 知识图谱更新

3. PPT制作
   Kimi PPT生成(消耗Kimi额度)
   → OpenClaw内容优化(消耗Claw Token)
   → 最终演示文稿
```

## 八、升级建议

### 当前Allegretto是否足够？

**评估标准**:
- 深度研究 40次/月 = 每天1.3次 → **基本够用**
- PPT 40次/月 = 每周10次 → **非常充足**
- Agent 40次/月 = 每天1.3次 → **可能紧张**

**建议**:
1. **先使用1个月**，记录实际消耗
2. 如果Agent额度经常用完，考虑升级到Allegro（100次/月）
3. 如果深度研究不够用，优先优化提示词质量而非升级

### Allegro（100次/月）优势
- Agent额度充足（每天3次）
- 深度研究充足（每天3次）
- 支持更复杂的并发任务

## 九、紧急备案

### 额度用完怎么办？

1. **Agent额度用完**
   - 使用OpenClaw的web_fetch（免费但质量较低）
   - 手动搜索+整理
   - 等待下月额度重置

2. **深度研究额度用完**
   - 使用OpenClaw的kimi_search（免费但深度有限）
   - 使用Kimi普通对话（不消耗深度研究额度）
   - 等待下月额度重置

3. **PPT额度用完**
   - 使用OpenClaw生成Markdown大纲
   - 手动制作PPT
   - 等待下月额度重置（但40次/月很难用完）
