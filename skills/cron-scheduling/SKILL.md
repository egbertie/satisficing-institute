# cron-scheduling Skill V5标准版本

## S1: 全局考虑

### 输入
- 任务脚本路径
- 执行时间表（Cron表达式）
- 任务优先级

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 开发者配置任务、运维人员监控 |
| **事** | 定时任务调度、执行监控、失败重试 |
| **物** | Cron表达式、脚本、日志、状态 |
| **环境** | 时区、系统时间、资源限制 |
| **外部集成** | Cron服务、通知渠道 |
| **边界情况** | 任务重叠、执行超时、系统重启 |

---

## S2: 系统考虑

### 处理流程
```
配置解析 → 时间计算 → 触发执行 → 状态记录 → 结果通知
    ↑           ↓           ↓           ↓
    └──── 冲突检测 ← 超时处理 ← 失败重试 ←────┘
```

### 故障处理
- **执行超时**: 强制终止，标记失败
- **脚本错误**: 记录错误日志，发送告警
- **时间重叠**: 根据策略（跳过/排队/并行）

---

## S4: 自动化集成

### 调度策略
- 精确时间触发
- 时区感知（Asia/Shanghai）
- 失败重试（最多3次）
- 超时控制（可配置）

---

## S5: 自我验证

### 质量指标
- 调度准确率: >99.9%
- 执行成功率: >95%
- 告警及时性: <1分钟

---

## S6: 认知谦逊

### 局限
- 依赖系统时间准确
- 重启期间可能丢失任务
- 无法处理分布式调度
- 单点故障风险

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 系统时间回拨 | 检测异常，记录警告 |
| 任务执行超时 | 强制终止，标记timeout |
| 连续失败3次 | 暂停任务，发送告警 |
| 系统重启 | 恢复调度，检查错过的任务 |
| 表达式错误 | 启动时校验，拒绝配置 |
