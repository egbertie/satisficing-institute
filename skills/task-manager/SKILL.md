# task-manager Skill V5标准版本

## S1: 全局考虑

### 输入
- 任务描述
- 优先级（P0/P1/P2/P3）
- 截止时间
- 负责人

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 项目经理、执行人员 |
| **事** | 任务创建、分配、追踪、完成 |
| **物** | 任务项、状态、依赖关系 |
| **环境** | 项目阶段、资源约束 |
| **外部集成** | 日历、通知渠道 |
| **边界情况** | 任务逾期、依赖阻塞、资源冲突 |

---

## S2: 系统考虑

### 处理流程
```
创建任务 → 分配资源 → 状态追踪 → 进度更新 → 完成归档 → 复盘分析
```

### 故障处理
- **任务逾期**: 自动提醒，升级处理
- **资源冲突**: 提示调度问题
- **依赖阻塞**: 标记阻塞原因

---

## S3: 输出规范

### 任务状态
```
待办 → 进行中 → 已完成
   ↓      ↓         ↓
阻塞   逾期风险    已归档
```

### 任务报告
```json
{
  "task_id": "task_xxx",
  "title": "任务标题",
  "status": "in_progress",
  "priority": "P1",
  "assignee": "负责人",
  "created": "2026-03-22",
  "deadline": "2026-03-25",
  "progress": 60,
  "blockers": []
}
```

---

## S4: 自动化集成

### 提醒策略
- 截止前24小时: 提醒
- 截止前4小时: 紧急提醒
- 逾期: 升级告警

---

## S5: 自我验证

### 质量指标
- 任务完成率: >80%
- 逾期率: <20%
- 进度准确性: >90%

---

## S6: 认知谦逊

### 局限
- 无法强制任务完成
- 进度依赖人工更新
- 复杂依赖难以自动处理

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 任务逾期 | 升级告警，建议补救 |
| 资源不足 | 提示冲突，建议调整 |
| 连环阻塞 | 识别关键路径 |
| 大量任务 | 分页展示，优先排序 |
