# info-quality-guardian Skill V5标准版本

## S1: 全局考虑

### 输入
- 待验证信息
- 验证标准
- 可信源列表

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 信息使用者、审核人员 |
| **事** | 来源验证、可信度评估、风险标记、建议提供 |
| **物** | 信息内容、来源标识、验证记录 |
| **环境** | 网络环境、源站可信度 |
| **外部集成** | 可信数据库、事实核查API |
| **边界情况** | 来源不明、多方冲突、无法验证 |

---

## S2: 系统考虑

### 处理流程
```
信息接收 → 来源识别 → 可信度评估 → 风险标记 → 建议生成 → 输出结果
```

### 可信度等级
| 等级 | 描述 | 建议 |
|------|------|------|
| A | 权威来源，可信赖 | 直接使用 |
| B | 可信来源，需留意 | 建议核实 |
| C | 来源一般，需谨慎 | 建议交叉验证 |
| D | 来源可疑，风险高 | 不建议使用 |
| X | 无法验证 | 需人工判断 |

---

## S3: 输出规范

### 验证结果
```json
{
  "content": "...",
  "source": "...",
  "credibility": "B",
  "confidence": 0.75,
  "risks": ["来源非一手"],
  "suggestions": ["查找原始出处"],
  "verified_at": "2026-03-22T09:00:00+08:00"
}
```

---

## S4: 自动化集成

### 自动验证
- 来源域名检查
- 作者资质验证
- 发布时间核实
- 交叉引用检查

---

## S5: 自我验证

### 质量指标
- 验证覆盖率: >90%
- 误判率: <10%
- 响应时间: <2s

---

## S6: 认知谦逊

### 局限
- 无法100%准确判断
- 新来源可能误判
- 主观内容难客观评估
- 依赖可信源数据库

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 来源伪造 | 标记D级，告警 |
| 多方冲突 | 列出各方，标注争议 |
| 无法验证 | 标记X级，建议人工 |
| 权威来源 | 标记A级，快速通过 |
