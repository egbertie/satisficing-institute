## 功能1: 网站 (Web浏览/解析)

### 使用场景
- 快速获取政策原文、学术论文、行业报告
- 解析竞争对手官网、政府白皮书页面
- 提取关键信息，无需完整阅读

### 提示词模板

```
【满意解研究所 - 网站解析任务】

## 目标网页
[粘贴URL]

## 解析目标
请提取以下信息：
1. **核心观点** (3-5条，带原文引用)
2. **关键数据** (数字、百分比、时间点)
3. **政策条款** (如适用，原文摘录)
4. **与满意解关联点** (如何支持合伙人决策研究)

## 输出格式
| 类别 | 内容 | 原文位置 |
|------|------|----------|
| 核心观点1 | ... | 第X段 |
| 关键数据 | ... | 第X段 |
| ... | ... | ... |

## 特殊要求
- 不确定的内容标注[待核实]
- 无法找到的内容标注[未提及]
- 如有PDF下载链接，请提取
```

### 使用示例

```
【满意解研究所 - 网站解析任务】

## 目标网页
https://www.sz.gov.cn/cn/xxgk/zfxxgj/yjzj/content/post_11223344.html

## 解析目标
请提取以下信息：
1. **核心观点** (3-5条，带原文引用)
2. **关键数据** (数字、百分比、时间点)
3. **政策条款** (如适用，原文摘录)
4. **与满意解关联点** (如何支持合伙人决策研究)

## 输出格式
| 类别 | 内容 | 原文位置 |
|------|------|----------|
| 核心观点1 | ... | 第X段 |
| 关键数据 | ... | 第X段 |
| ... | ... | ... |

## 特殊要求
- 不确定的内容标注[待核实]
- 无法找到的内容标注[未提及]
- 如有PDF下载链接，请提取
```

```
## 目标文档

- 发布日期:

### 2. 核心观点 (5-7条)

### 3. 关键数据 (表格形式)
| 数据项 | 数值 | 时间 | 来源(文中位置) |
|--------|------|------|----------------|

### 4. 方法论/框架 (如有)
- 研究方法:
- 核心模型:
- 关键变量:

### 5. 与满意解关联点
- 可直接引用的内容:
- 需要验证的内容:
- 与现有研究的冲突/补充:

### 6. 需人工验证清单
- [ ] ...
- [ ] ...
```


## 功能3: PPT (演示文稿生成)

### 使用场景
- 快速生成投资人路演PPT
- 制作专家讲座/培训课件
- 整理研究汇报演示文稿

### 提示词模板

#### 版本A: 投资人路演PPT

```
【满意解研究所 - 投资人路演PPT】

## PPT主题
满意解研究所：硬科技初创企业合伙人选择决策教练

## 目标受众
天使投资人/VC/政府引导基金

## 核心信息
1. 市场规模：中国硬科技创业合伙人匹配服务市场
2. 独特价值：左脑逻辑+右脑直觉的综合决策方法论
3. 成功案例：首批客户验证
4. 商业模式：B2B服务+SaaS工具
5. 团队背景：专家网络+方法论创新
6. 融资需求：Pre-A轮XXX万，用于...

## PPT结构要求
1. 封面 - 一句话价值主张
2. 痛点 - 硬科技创业合伙人选择的高失败率
3. 解决方案 - 满意解方法论
4. 市场规模 - TAM/SAM/SOM数据
5. 商业模式 - 收入来源+定价策略
6. 竞争优势 - 五路图腾×专家网络
7. 里程碑 - 已达成+未来12个月计划
8. 团队 - 核心成员+专家顾问
9. 融资需求 - 资金用途
10. 结尾 - 行动号召

## 设计要求
- 每页不超过50字
- 多用图表/数据可视化
- 配色：专业稳重（深蓝+金色）
- 字体：清晰易读

## 输出要求
生成可下载的PPT文件，或提供完整大纲+每页内容
```

## 功能5: 深度研究 (优化配合版本)

### 使用场景
- 为满意解研究体系提供学术级研究支持
- 每日深度研究主题生成
- 与OpenClaw验证优化形成闭环

### 提示词模板 (已优化配合整体项目)

```
【满意解研究所 - 深度研究任务】

## 研究主题
[填写具体研究主题]

## 研究背景
满意解研究所专注于硬科技初创企业家合伙人选择决策教练，采用：
- 理论基础：赫伯特·西蒙满意解理论 + 前景理论 + 儒商哲学
- 方法论：左脑逻辑 + 右脑直觉（感知力）综合决策
- 框架：五路图腾（土水木金火）× 五大专家

## 研究目标
1. 为该主题建立学术级知识图谱
2. 提取可引用的核心观点和数据
3. 与满意解方法论建立关联点
4. 识别实践应用价值

## 执行指令

### 阶段1: 情报网格构建
1. 执行≥3个独立搜索查询，覆盖：
   - 学术数据库（arXiv/Google Scholar/JSTOR）
   - 灰色文献（政府白皮书/智库报告）
   - 行业数据源（SEC/Patents/统计局）
   - 多语种信源（英语+中文）

2. 对关键主张执行"5层溯源":
   原始数据→一手分析→二手综述→媒体解读→AI合成
   （禁止直接使用AI合成层）

3. 主动检索 contradictory evidence（反方证据）

### 阶段2: 证据链锻造
- 所有事实性声明采用"学术引用微格式":
  ^ID^ 作者. (年份). 标题. 期刊/出版社. DOI/URL. [检索日期: YYYY-MM-DD]

- 证据等级标注:
  [P1] 原始实验/官方统计/法律条文
  [P2] 系统综述/Meta分析/顶刊
  [P3] 工作论文/行业报告/专家访谈
  [P4] 新闻/百科（仅作背景）

### 阶段3: 五路图腾关联分析
从五个维度分析与满意解的关联：
1. **土（信实）**: 诚信度、价值观、风险评估
2. **水（变通）**: 政策环境、资源流动、地理优势
3. **木（生长）**: 成长潜力、协作能力、创新活力
4. **金（精准）**: 决策质量、神经机制、感知力
5. **火（能量）**: 内在动力、激情匹配、身心健康

### 阶段4: 逻辑架构
引言（Hook→Gap→Question→Significance）
    ↓
理论框架 + 概念操作化
    ↓
方法论 + 局限性声明
    ↓
多维度发现（链接证据链+五路图腾关联）
    ↓
讨论（Agree/Extend/Challenge/Synthesize）
    ↓
满意解应用点（明确标注）
    ↓
批判性反思（偏见/盲区/伦理）

### 阶段5: 迭代优化
第一轮: 信息饱和（检查核心数据库/近3个月预印本）
第二轮: 逻辑压力测试（魔鬼代言人审查）
第三轮: 元认知审计（引用密度≥3个/千字，原创性边界明确）

## 输出要求

### 1. 执行摘要（500字）
- 研究背景
- 核心发现（3-5条）
- 满意解应用价值

### 2. 详细研究报告（6000-8000字）
- 按上述逻辑架构组织
- 所有数据标注来源
- 所有案例标注真实性

### 3. 核心观点/金句库（10-15条）
每条包含：
- 金句原文
- 来源标注
- 满意解应用场景

### 4. 可引用数据清单（表格）
| 数据项 | 数值 | 时间 | 来源 | 证据等级 | 置信度 |
|--------|------|------|------|----------|--------|

### 5. 五路图腾关联矩阵
| 图腾 | 关联点 | 应用方式 | 优先级 |
|------|--------|----------|--------|

### 6. 需人工验证清单
- [ ] ...
- [ ] ...

## 禁忌
- ❌ "多项研究表明..."（不指明具体研究）
- ❌ "众所周知..."（必须提供溯源）
- ❌ 将推测包装为结论
- ❌ 编造不存在的引用

## 置信度编码
对所有结论标注置信度: High/Medium/Low/Speculative

## 知识截止
知识截止日期: 2026-03

## 后续流程
生成完成后，请将论文发送给OpenClaw进行验证优化，最终输出2.0学术严谨版。

请生成完整研究报告。
```
