# honesty-tagging-protocol Skill V5标准版本

## S1: 全局考虑

### 输入
- AI生成的内容
- 事实性声明
- 引用来源

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 内容创作者、信息消费者 |
| **事** | 不确定内容识别、诚实标注、置信度评估 |
| **物** | 文本内容、标注标签、置信度分数 |
| **环境** | 内容生成场景、发布渠道 |
| **外部集成** | 内容审核系统 |
| **边界情况** | 模糊表述、主观判断、时效性问题 |

---

## S2: 系统考虑

### 处理流程
```
内容分析 → 事实性判断 → 置信度评估 → 标签标注 → 输出呈现
```

### 标签类型
| 标签 | 含义 | 使用场景 |
|------|------|----------|
| [确定] | 高置信度事实 | 有可靠来源验证 |
| [推测] | 基于推理 | 逻辑推导但无直接证据 |
| [待验证] | 不确定 | 需要进一步核实 |
| [模拟] | 假设场景 | 虚构或假设性内容 |

### 故障处理
- **判断模糊**: 保守标注[待验证]
- **来源冲突**: 列出多方观点
- **时效过期**: 标注时间戳

---

## S3: 输出规范

### 标注格式
```
[确定] 2026年3月22日，5标准覆盖率达到63%。
[推测] 预计到本月底，覆盖率可能达到80%。
[待验证] 用户提到的某个具体数据。
[模拟] 假设Token预算增加50%的场景...
```

### 元数据
```json
{
  "tag": "待验证",
  "confidence": 0.3,
  "reason": "数据来源未确认",
  "suggestion": "请用户提供数据来源"
}
```

---

## S4: 自动化集成

### 触发场景
- 涉及具体数据时
- 引用外部信息时
- 时间敏感内容时
- 预测性表述时

---

## S5: 自我验证

### 质量指标
- 标注准确率: >90%
- 漏标率: <5%
- 误标率: <10%

---

## S6: 认知谦逊

### 局限
- 无法判断所有不确定性
- 依赖上下文理解
- 主观内容难以客观标注
- 标注本身可能有误

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 虚假数据 | 标注[待验证]或[推测] |
| 模糊时间 | 标注[待验证]，要求精确 |
| 个人观点 | 标注为观点，非事实 |
| 过时信息 | 标注时间戳，建议更新 |
