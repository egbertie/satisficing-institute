# 休眠协议 - 快速参考

## 核心规则（固化到系统）

### 规则1: 默认休眠 = 完全静默模式
- 除非用户明确指令，否则"休眠"=暂停所有Cron，零Token消耗
- 仅保留核心备份任务（每日1次）

### 规则2: 自动触发条件
- 最后对话10分钟内无新指令 → 自动进入完全静默
- 夜间22:00-08:00 → 更严格的休眠模式
- Token预算<30% → 建议休眠；<10% → 强制休眠

### 规则3: 夜间执行例外
- 仅当用户明确说"夜间执行XX"时，才执行该任务
- 任务完成后自动进入休眠
- 无明确指令 = 不执行任何任务

### 规则4: 唤醒条件
- 用户发送任何消息
- 或明确说"唤醒"/"开始"/"继续"

---

## 用户指令映射

| 用户说 | 我的行动 |
|--------|----------|
| "休眠"/"睡觉"/"暂停"/"休息" | 进入完全静默模式 |
| "完全静默" | 暂停所有任务，包括备份 |
| "休眠，但保留备份" | 保留每日备份，其他暂停 |
| "夜间执行XX，然后休眠" | 执行XX → 自动休眠 |
| "唤醒"/"开始"/"继续" | 恢复正常运行 |
| "保持清醒" | 暂停自动休眠2小时 |
