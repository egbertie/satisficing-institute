# Testing Limitations & Known Issues

本文件记录测试框架的局限性（S6标准），明确哪些场景无法通过自动化测试覆盖。

---

## 局限性清单

### LIM-001: AI模型输出的非确定性

| 属性 | 内容 |
|------|------|
| **Skill** | blue-sentinel |
| **类型** | 非确定性行为 |
| **描述** | 认知审计依赖AI模型输出，相同输入可能产生不同结果 |
| **影响** | 中 |
| **根本原因** | AI模型的内在随机性（temperature>0） |
| **缓解措施** | 1. 多次运行取平均 2. 设置随机种子 3. 验证输出格式而非内容 4. 统计一致性检查 |
| **手动测试** | 是，需人工验证审计质量 |

### LIM-002: 外部Token计数API依赖

| 属性 | 内容 |
|------|------|
| **Skill** | token-budget-enforcer |
| **类型** | 外部依赖 |
| **描述** | Token预算计算依赖外部API，网络/API变更会影响功能 |
| **影响** | 高 |
| **根本原因** | 无法直接访问内部Token计数器 |
| **缓解措施** | 1. Mock API响应 2. 契约测试 3. 本地缓存 4. 降级策略 |
| **手动测试** | 否，Mock足够覆盖 |

### LIM-003: 真实时间依赖

| 属性 | 内容 |
|------|------|
| **Skill** | zero-idle-enforcer, token-budget-enforcer |
| **类型** | 时间依赖 |
| **描述** | 涉及时间边界（跨日、时区）的功能难以在测试中快速验证 |
| **影响** | 中 |
| **根本原因** | 真实时间流逝无法加速 |
| **缓解措施** | 1. Mock时间 2. 时间旅行测试 3. 边界值测试 |
| **手动测试** | 否，时间Mock足够 |

### LIM-004: 多Skill并发竞争

| 属性 | 内容 |
|------|------|
| **Skill** | 所有Skill |
| **类型** | 并发竞争 |
| **描述** | 多个Skill同时读写状态文件可能产生竞争条件 |
| **影响** | 中 |
| **根本原因** | 文件系统非原子操作 |
| **缓解措施** | 1. 文件锁 2. 原子写入 3. 异常重试 4. 日志分析 |
| **手动测试** | 是，需压力测试验证 |

### LIM-005: 长时间运行稳定性

| 属性 | 内容 |
|------|------|
| **Skill** | 所有Skill |
| **类型** | 长时间行为 |
| **描述** | 内存泄漏、资源耗尽等问题需要长时间运行才能暴露 |
| **影响** | 低 |
| **根本原因** | 测试执行时间短 |
| **缓解措施** | 1. 资源监控 2. 夜间长时间测试 3. 静态分析 |
| **手动测试** | 是，24小时运行监控 |

### LIM-006: 大文件/大数据处理

| 属性 | 内容 |
|------|------|
| **Skill** | blue-sentinel |
| **类型** | 资源限制 |
| **描述** | 超大日志文件、大量历史数据处理无法在实际环境中测试 |
| **影响** | 低 |
| **根本原因** | 测试环境资源有限 |
| **缓解措施** | 1. 小数据代理 2. 流式处理 3. 性能测试 |
| **手动测试** | 是，生产环境监控 |

## 手动测试清单

## 无法测试的具体代码

以下代码路径在当前测试框架下无法覆盖：

```python
# zero-idle-enforcer/enforcer.py
# 无法测试：需要真实时间流逝
def wait_for_next_hour():
    """等待到下一个整点"""
    now = datetime.now()
    next_hour = (now + timedelta(hours=1)).replace(minute=0, second=0)
    time.sleep((next_hour - now).total_seconds())  # 无法测试

# token-budget-enforcer/enforcer.py  
# 无法测试：依赖真实API响应
def get_real_token_usage():
    """获取真实Token使用量"""
    response = requests.get(API_URL)  # 外部依赖
    return response.json()["usage"]
```

## 局限性管理流程

1. **发现**: 新发现的局限性记录到此文件
2. **评估**: 评估影响和缓解成本
3. **缓解**: 实施缓解措施
4. **验证**: 手动测试验证缓解效果
5. **关闭**: 局限性消除后标记为已解决
