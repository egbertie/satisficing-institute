## Skill: zero-idle-enforcer

### 基本信息
- **风险等级**: P0
- **版本**: 1.0.0
- **测试负责人**: Satisficing Institute
- **最后更新**: 2026-03-21

### 测试需求

#### 功能点清单

| ID | 功能点 | 优先级 | 测试类型 | 依赖 |
|----|--------|--------|----------|------|
| F1 | 读取用户活动时间 | P0 | 单元 | 无 |
| F2 | 判断Token级别（NORMAL/LOW/CRITICAL） | P0 | 单元 | F1 |
| F3 | 检查触发条件（空闲+Token+补位间隔） | P0 | 单元 | F1,F2 |
| F4 | 执行补位任务生成 | P0 | 单元 | F3 |
| F5 | 状态持久化（zero-idle-state.json） | P0 | 单元 | 无 |
| F6 | 日志记录功能 | P1 | 单元 | 无 |
| F7 | 与Heartbeat状态集成 | P1 | 集成 | F1 |
| F8 | 完整工作流（检测到执行） | P0 | E2E | F1-F5 |

#### 边界条件

| 边界ID | 场景 | 预期行为 |
|--------|------|----------|
| B1 | 正好2小时空闲（7200秒） | 触发补位 |
| B2 | 空闲时间>30天 | 视为异常，不触发 |
| B3 | Token剩余=15%（临界值） | 返回LOW（不触发） |
| B4 | Token剩余=14% | 返回CRITICAL（不触发） |
| B5 | 上次补位时间=2小时前 | 触发 |
| B6 | 上次补位时间=1小时前 | 不触发 |

#### 异常场景

| 异常ID | 场景 | 预期处理 |
|--------|------|----------|
| E1 | heartbeat-state.json损坏 | 回退到文件mtime |
| E2 | token-weekly-monitor.json缺失 | 默认NORMAL级别 |
| E3 | zero-idle-state.json损坏 | 重置状态 |
| E4 | 未来时间戳 | 忽略，使用回退 |
| E5 | 负数时间戳 | 视为异常，不触发 |

### 覆盖率目标
- 行覆盖率: ≥90%
- 分支覆盖率: ≥85%
- 函数覆盖率: ≥95%

## 测试计划汇总

### 测试资源分配

| Skill | 单元测试数 | 集成测试数 | E2E测试数 | 预计执行时间 |
|-------|-----------|-----------|-----------|-------------|
| zero-idle-enforcer | 30 | 5 | 3 | 45s |
| token-budget-enforcer | 30 | 5 | 3 | 45s |
| blue-sentinel | 20 | 3 | 2 | 30s |
| **总计** | **80** | **13** | **8** | **2min** |

### 测试环境要求

- Python 3.10+
- pytest 7.4+
- 内存: ≥2GB
- 磁盘: ≥1GB（用于报告）

### 测试数据需求

1. **Mock数据**: 所有外部依赖使用Mock
2. **临时文件**: 测试目录自动创建/清理
3. **状态文件**: 各Skill的JSON状态文件模板

### 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 测试环境不稳定 | 高 | 使用临时目录，隔离测试 |
| 测试执行时间过长 | 中 | 并行执行，分级触发 |
| 覆盖率不达标 | 高 | 代码审查，强制门禁 |
