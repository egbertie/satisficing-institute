# task-queue Skill V5标准版本

## S1: 全局考虑

### 输入
- 任务项
- 优先级
- 队列配置（容量/并发）

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 系统开发者 |
| **事** | 任务入队、出队、执行、重试 |
| **物** | 队列、任务、执行器 |
| **环境** | 系统负载、资源限制 |
| **外部集成** | 执行器、存储 |
| **边界情况** | 队列满、任务失败、死循环 |

---

## S2: 系统考虑

### 处理流程
```
任务入队 → 优先级排序 → 出队执行 → 结果处理 → 成功/重试/失败
```

### 故障处理
- **队列满**: 拒绝新任务或扩展容量
- **任务失败**: 重试3次，失败后移到死信队列
- **执行器崩溃**: 任务重新入队

---

## S4: 自动化集成

### 队列策略
| 策略 | 说明 |
|------|------|
| FIFO | 先进先出 |
| 优先级 | 高优先级优先 |
| 延迟 | 定时执行 |
| 重试 | 失败自动重试 |

---

## S5: 自我验证

### 质量指标
- 吞吐量: 满足需求
- 成功率: >95%
- 延迟: 可接受范围内

---

## S6: 认知谦逊

### 局限
- 依赖执行器稳定性
- 无法处理无限任务
- 分布式场景需额外支持

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 队列积压 | 告警，建议扩容 |
| 任务死循环 | 超时终止 |
| 执行器全部崩溃 | 暂停出队，恢复后继续 |
| 重复任务 | 去重或幂等处理 |
