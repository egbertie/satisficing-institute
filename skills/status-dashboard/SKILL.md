# status-dashboard Skill V5标准版本

## S1: 全局考虑

### 输入
- 系统状态数据源
- 监控指标配置
- 刷新频率

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 运维人员、项目经理 |
| **事** | 状态采集、可视化、告警 |
| **物** | 指标数据、仪表盘、报告 |
| **环境** | 系统负载、网络状态 |
| **外部集成** | 各类服务API |
| **边界情况** | 数据源失效、大量指标、刷新冲突 |

---

## S2: 系统考虑

### 处理流程
```
数据采集 → 指标计算 → 状态聚合 → 可视化渲染 → 告警判断 → 报告生成
```

### 故障处理
- **数据源失效**: 标记离线，使用缓存
- **采集超时**: 重试并记录
- **告警风暴**: 合并相似告警

---

## S3: 输出规范

### 仪表盘组件
| 组件 | 说明 |
|------|------|
| 健康状态灯 | 系统整体健康度 |
| 趋势图表 | 关键指标趋势 |
| 告警列表 | 当前活跃告警 |
| 状态卡片 | 各服务状态 |

---

## S4: 自动化集成

### 刷新策略
- 实时指标: 30秒
- 普通指标: 5分钟
- 日报: 每日09:00

---

## S5: 自我验证

### 质量指标
- 数据新鲜度: <1分钟
- 准确性: >99%
- 可用性: >99.9%

---

## S6: 认知谦逊

### 局限
- 依赖数据源可用性
- 无法预测所有故障
- 可视化定制需额外工作

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 数据源全部失效 | 显示缓存数据+告警 |
| 告警风暴 | 合并并分级 |
| 高并发访问 | 缓存+限流 |
| 长时间离线 | 记录缺口，恢复后补全 |
