# conversation-continuity Skill V5标准版本

## S1: 全局考虑

### 输入
- 对话上下文历史
- 当前用户输入
- 会话状态

### 覆盖维度
| 维度 | 考虑内容 |
|------|----------|
| **人** | 用户、AI助手 |
| **事** | 对话连续性维护、上下文压缩、关键信息保留 |
| **物** | 对话历史、摘要、关键决策、待办 |
| **环境** | Token限制、会话超时、上下文窗口 |
| **外部集成** | 存储服务（可选） |
| **边界情况** | 超长对话、主题切换、长时间间隔 |

---

## S2: 系统考虑

### 处理流程
```
对话进行 → Token检查 → 阈值触发 → 上下文压缩 → 摘要生成 → 继续对话
```

### 故障处理
- **Token超限**: 触发压缩，保留摘要
- **主题切换**: 检测并标记新主题
- **长时间间隔**: 主动提供上下文回顾

---

## S3: 输出规范

### 压缩输出
```
【对话摘要】
- 主题: [主题]
- 关键决策: [决策1], [决策2]
- 待办: [待办1], [待办2]
- 重要信息: [信息摘要]

【继续对话】
...
```

---

## S4: 自动化集成

### 触发条件
- Token使用量 > 80%
- 对话轮数 > 50轮
- 用户主动要求总结

---

## S5: 自我验证

### 质量指标
- 信息保留率: >95%
- 压缩后可用性: 用户可继续理解
- 关键信息无遗漏

---

## S6: 认知谦逊

### 局限
- 压缩可能丢失细节
- 无法判断用户记忆中的信息
- 长期对话仍可能超限

---

## S7: 对抗测试

| 场景 | 预期行为 |
|------|----------|
| 超长对话（>100轮） | 多级压缩，保留核心 |
| 多主题混合 | 按主题分段摘要 |
| 用户忘记上下文 | 主动提供回顾 |
