# 对话连续性管理 Skill

> 解决跨时段对话遗忘问题，确保长期项目上下文不丢失

---

## 背景问题

当对话间隔数小时或跨天时，AI 会：
- 遗忘已确认的 API Key
- 重复询问已完成的任务
- 丢失对话达成的共识
- 浪费双方时间

---

## 解决方案

### 核心机制：强制记忆写入

每次对话达成重要结论时，**立即**写入文件，而不是依赖会话记忆。

### 记忆层级

```
MEMORY.md          ← 长期记忆（关键决策、项目状态）
memory/YYYY-MM-DD.md  ← 每日日志（详细过程）
.env               ← API Key 等敏感配置
AGENTS.md          ← 工作规则更新
USER.md            ← 用户偏好更新
```

---

## 使用规则

### 触发条件

**必须立即记录的情况：**
1. API Key 提供并验证后
2. 重要决策达成后
3. 任务完成/状态变更后
4. 用户明确要求"记住"某事
5. 对话间隔 > 2 小时后的新会话

### 记录格式

```markdown
## [时间] 对话摘要

### 达成共识
- [ ] 事项1
- [ ] 事项2

### 提供的资源
- API Key: xxx（已验证/待验证）
- 文件路径: xxx

### 待办事项
- [ ] 任务1
- [ ] 任务2

### 下次对话需知
- 关键上下文1
- 关键上下文2
```

## 启动检查清单

每次新会话启动时，执行：

```markdown
## 会话启动检查

1. [ ] 读取 MEMORY.md
2. [ ] 读取最近 3 天的 memory/*.md
3. [ ] 检查 .env 中的 API Key
4. [ ] 检查待办事项状态
5. [ ] 确认用户当前上下文

## 关键信息速查
| 项目 | 状态 | 来源文件 |
|------|------|---------|
| API Keys | ___ | .env |
| 当前项目 | ___ | MEMORY.md |
| 待办事项 | ___ | memory/*.md |
| 上次对话时间 | ___ | 最近日志 |
```

## 自动化工具

### 1. 会话启动脚本
```bash
#!/bin/bash
# 启动时自动执行
cat MEMORY.md
cat memory/$(date +%Y-%m-%d).md 2>/dev/null || echo "今日无日志"
grep -E "API|Key|Token" .env | sed 's/=.*/=***/'
```

### 2. 记忆搜索工具
```bash
# 快速搜索历史记忆
grep -r "关键词" memory/ MEMORY.md
```

## 示例场景

### 场景1: API Key 提供后

**用户**: 这是 Jina Key: jina_xxx

**AI 必须执行:**
1. 立即验证 Key
2. 写入 .env 文件
3. 更新 memory/今日.md:
   ```markdown
   ## 10:15 - Jina API Key 提供
   - Key: jina_xxx (已验证)
   - 用途: 网页提取
   - 免费额度: 100万 tokens
   ```
4. 更新 API 清单

### 场景2: 跨天对话

**AI 启动时:**
1. 读取昨天日志
2. 向用户确认: "昨天我们完成了 X, Y, Z，今天要继续吗？"
3. 列出待办事项

```markdown
## 记忆原则

**绝不依赖会话记忆。**

每次对话达成结论，立即写入文件。
这是对用户的尊重，也是对自己的负责。

### 启动时必须做的事
1. 读取 MEMORY.md
2. 读取最近 3 天日志
3. 向用户确认上下文
```
