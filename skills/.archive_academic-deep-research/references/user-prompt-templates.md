## 模板A: 深度研究主题提交 (给Kimi深度研究)

### 使用场景
当你需要Kimi深度研究生成学术论文时使用

### 标准提示词

```
你是一位具备跨学科方法论训练的资深研究总监。

## 研究范式
- 认识论: 实证主义
- 方法论: 混合方法（定量+定性）
- 学术伦理: COPE指南

## 研究主题
[在此精确填写研究主题，例如：]
"硬科技初创企业合伙人选择机制：基于2015-2025年全球独角兽案例的实证分析"

## 执行指令

### 阶段1: 情报网格构建
1. 执行≥3个独立搜索查询，覆盖：
   - 学术数据库（arXiv/JSTOR/Google Scholar）
   - 灰色文献（政府白皮书/智库报告）
   - 行业数据源（SEC/Patents）
   - 多语种信源（英语+中文）

2. 对关键主张执行"5层溯源":
   原始数据→一手分析→二手综述→媒体解读→AI合成
   （禁止直接使用AI合成层）

3. 主动检索 contradictory evidence（反方证据）

### 阶段2: 证据链锻造
- 所有事实性声明采用"学术引用微格式":
  ^ID^ 作者. (年份). 标题. 期刊/出版社. DOI/URL. [检索日期: YYYY-MM-DD]

- 证据等级标注:
  [P1] 原始实验/官方统计
  [P2] 系统综述/Meta分析/顶刊
  [P3] 工作论文/行业报告
  [P4] 新闻/百科（仅作背景）

### 阶段3: 多维分析框架
从以下5个维度构建分析：
1. 时间维度: 历史演进→当下截面→未来推演（三情景）
2. 空间维度: 全球比较→区域差异→微观-中观-宏观
3. 学科维度: 主学科 + 2个交叉视角 → 涌现特性
4. 利益相关者: 政府/市场/公民社会/个体
5. 方法论三角: 定量 + 定性 + 计算

### 阶段4: 逻辑架构（钻石模型）
引言（Hook→Gap→Question→Significance）
    ↓
理论框架 + 概念操作化
    ↓
方法论 + 局限性声明
    ↓
多维度发现（链接证据链）
    ↓
讨论（Agree/Extend/Challenge/Synthesize）
    ↓
原创性声明（理论贡献+实践价值）
    ↓
批判性反思（偏见/盲区/伦理）

### 阶段5: 迭代优化（三轮）
第一轮: 信息饱和（检查核心数据库/近3个月预印本）
第二轮: 逻辑压力测试（魔鬼代言人审查）
第三轮: 元认知审计（引用密度≥3个/千字，原创性边界明确）

## 输出要求
- 总长度: 8000-12000字
- 结构化摘要: Background/Methods/Results/Conclusion
- 关键词: 5-7个
- 参考文献: ≥30条，近5年≥40%
- 格式: Markdown，标题层级清晰

## 禁忌
- ❌ "多项研究表明..."（不指明具体研究）
- ❌ "众所周知..."（必须提供溯源）
- ❌ 将推测包装为结论

## 置信度编码
对所有结论标注置信度: High/Medium/Low/Speculative

## 知识截止
知识截止日期: 2026-03

请生成完整研究报告。
```

## 模板B: 论文验证优化 (给OpenClaw)

### 使用场景
当你收到Kimi深度研究生成的论文，需要我进行验证和优化时使用

### 标准提示词

```
请对这份Kimi深度研究生成的论文执行academic-deep-research Skill：

## 处理要求
1. **真实性验证**: 区分真实内容与AI模拟内容
   - 机构验证（是否真实存在）
   - 数据验证（是否有可验证来源）
   - 案例验证（是否真实发生）
   - 专家验证（是否真实存在）

2. **引用标准化**: 将所有事实性声明补充为学术引用格式
   - 采用^ID^学术引用微格式
   - 标注证据等级[P1/P2/P3/P4]

3. **批判性反馈**: 红军/蓝军双向审查
   - 红军: 方法论/证据/逻辑/立场攻击
   - 蓝军: 承认局限/补充论证/重新框架/稳健性检验

4. **2.0版本整合**: 输出学术严谨、真实可靠的2.0版本
   - 真实性标注（✓/⚠️/🔶/🔷/❓）
   - 引用完整性（每段都有^ID^标注）
   - 批判性反思（红军/蓝军结果）
   - 局限性声明（明确研究边界）
   - 原创性声明（区分本研究vs前人vs推测）

## 论文内容
[粘贴Kimi深度研究生成的论文]

---

请分Phase输出：
- Phase 1: 真实性验证报告
- Phase 2: 引用标准化索引
- Phase 3: 批判性反馈（红军/蓝军）
- Phase 4: 2.0版本整合

如果内容较长，我会分批发送，请等待完整内容后再开始处理。
```

### 使用场景

```

[填写具体主题]
```
