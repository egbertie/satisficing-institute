# 测试用例

## 标准测试用例

### TC-001: 标准会议记录
**输入：**
```
产品周会 2026-03-21
参会: @Alice, @Bob

讨论内容:
1. Q2功能规划 - 确定了三个优先功能
2. 资源分配 - @Alice负责用户调研，下周三完成
3. 预算 - 50k已获批，@Bob负责确认

待办:
- @Carol需要评审技术方案，下周一前完成
- 团队联系供应商，待定
```

**预期输出：**
- 主题: 产品周会
- 日期: 2026-03-21
- 参会人员: Alice, Bob, Carol
- 行动项: 3+项
- 决策项: 2+项

---

### TC-002: 混乱笔记
**输入：**
```
营销会议 1/27
sarah - 需要finalize q1 campaign，她说周五
预算讨论 - mike认为需要50k，approved了
delay on产品发布 - 2周，等法务
john wed前发assets
还要figure out vendor情况
下次会议周四2pm
```

**预期输出：**
- 识别中英文混杂
- 提取4+项行动项
- 正确解析相对时间

---

### TC-003: VTT字幕文件
**输入：**
```vtt
WEBVTT

00:00:01.000 --> 00:00:05.000
主持人: 好，我们开始今天的周会

00:00:05.500 --> 00:00:12.000
Alice: 上周完成了用户调研，发现三个主要痛点

00:00:12.500 --> 00:00:18.000
Bob: 我们需要在下周三前完成设计方案
```

**预期输出：**
- 正确去除时间戳
- 提取发言内容
- 识别行动项

---

## 对抗测试用例

### ADV-001: 噪声注入
**输入：** 在标准输入中随机插入10%噪声字符

**预期：** 系统正常运行，结果质量略有下降但不崩溃

### ADV-002: 格式损坏
**输入：** 缺少标点的文本

**预期：** 仍能提取主要信息

### ADV-003: 语义混乱
**输入：** 倒序表达、多重否定

**预期：** 置信度降低，标注需要复核

### ADV-004: 超长输入
**输入：** 10万字无换行

**预期：** 分段处理，不内存溢出
