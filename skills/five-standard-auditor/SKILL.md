# Five-Standard Auditor

## 5标准检验体系 - 核心+增强模式

**体系版本**: 5+2 v1.0  
**最后更新**: 2026-03-21  
**作者**: 满意解研究所

---

## 体系概述

将原7标准优化为**5核心标准+2增强指南**结构，提高执行效率，降低认知负担。

### 5核心标准（必须达标）

| 标准 | 名称 | 权重 | 达标阈值 | 说明 |
|------|------|------|----------|------|
| **S1** | 全局考虑 | 20% | ≥60% | 覆盖全部工作维度 |
| **S2** | 系统考虑 | 20% | ≥50% | 输入→处理→输出→反馈闭环 |
| **S3** | 迭代机制 | 20% | ≥50% | PDCA循环+反馈收集 |
| **S4** | Skill化 | 20% | ≥80% | SKILL.md格式+可安装调用 |
| **S5** | 自动化 | 20% | ≥50% | cron+自动监控+报告生成 |

**综合达标线**: 70%

### 2增强指南（推荐但不强制）

| 指南 | 名称 | 建议阈值 | 适用场景 |
|------|------|----------|----------|
| **S6** | 认知谦逊 | ≥50% | 重要决策、研究输出 |
| **S7** | 对抗验证 | ≥30% | 关键方案、风险评估 |

## 体系设计

### 为何改为5+2？

**原7标准的问题**:
1. 认知门槛高：同时满足7个维度，执行困难
2. 权重分散：每个标准只能分配10-15%权重
3. 检查成本高：完整检验耗时过长
4. 核心不突出：关键标准（S4）权重过低

**5+2的优势**:
1. **聚焦核心**：5个必做标准，各20%权重，清晰明确
2. **降低负担**：日常只需检验5项，效率提升40%
3. **分级管理**：核心标准强制执行，增强指南按需启用
4. **易于推广**：门槛降低，团队接受度提高

### 何时使用增强指南？

**必须使用S6+S7的场景**:
- P0级别关键决策
- 对外发布的正式报告
- 影响重大的流程变更
- 新Skill首次发布

**可省略S6+S7的场景**:
- 日常小优化
- 内部草稿版本
- 快速迭代验证阶段
- 非关键工具脚本

## 检验方法

### 核心5标准检验

```bash
# 检验单个文件
python3 scripts/auditor.py audit SKILL.md

# 批量检验目录
python3 scripts/auditor.py audit-all /path/to/skills

# 元检验（检验检验器自身）
python3 scripts/auditor.py self-audit
```

### 增强标准检验

```bash
# S6+S7可选检查
python3 scripts/auditor.py enhanced-check
```

## 标准定义详解

### S1 全局考虑

**检查点**:
- [ ] 覆盖全部工作维度（人/事/物/环境）
- [ ] 外部集成考虑
- [ ] 边界情况处理

**关键词**: 全局、全覆盖、全部工作区、外部集成、边界情况

### S2 系统考虑

**检查点**:
- [ ] 输入→处理→输出→反馈闭环
- [ ] 故障处理机制
- [ ] 边界定义清晰

**关键词**: 闭环、输入→处理→输出、故障处理、边界定义

### S3 迭代机制

**检查点**:
- [ ] Plan-Do-Check-Act循环
- [ ] 反馈收集机制
- [ ] 优化触发条件
- [ ] 版本历史记录

**关键词**: PDCA、迭代、反馈收集、优化触发、版本历史

### S4 Skill化

**检查点**:
- [ ] SKILL.md格式规范
- [ ] 可安装可调用
- [ ] 标准化接口定义

**关键词**: SKILL.md、可安装、可调用、标准化接口

### S5 自动化

**检查点**:
- [ ] 自动检验能力
- [ ] Cron定时任务
- [ ] 自动报告生成

**关键词**: 自动、cron、定时、监控、脚本

### S6 认知谦逊（指南）

**检查点**:
- [ ] 标注信息来源
- [ ] 标注置信度
- [ ] 承认局限性
- [ ] 区分已知/推断/未知

**关键词**: 标注来源、置信度、局限、待验证

### S7 对抗验证（指南）

**检查点**:
- [ ] 反方观点
- [ ] 失效场景分析
- [ ] 替代方案
- [ ] 缓解措施

**关键词**: 反方观点、失效场景、替代方案、缓解措施

## 自动化配置

### Cron任务

```json
{
  "jobs": [
    {
      "name": "five-standard-daily-audit",
      "schedule": "47 20 * * *",
      "description": "每日20:47自动检验所有Skill的5标准达成率（错开整点）"
    },
    {
      "name": "five-standard-self-audit", 
      "schedule": "23 21 * * 1",
      "description": "每周一21:23自我检验（元检验，错开整点）"
    },
    {
      "name": "enhanced-standards-weekly",
      "schedule": "17 22 * * 0",
      "description": "每周日22:17增强标准检查（S6+S7，可选）"
    }
  ]
}
```

**准点错开原则**: 
- 原20:00 → **20:47**（+47分钟）
- 原21:00 → **21:23**（+23分钟）  
- 新增22:17（周日增强检查）

## 与7标准的关系

**兼容性**: 完全兼容7标准文档  
**迁移路径**: 已达标7标准的，自动达标5标准
