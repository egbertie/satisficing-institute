# 商业计划沙盘交付计划（Token优化版）

> 目标：最小Token消耗，最高质量交付  
> 策略：并行协作+精准分工+MVP优先  > 交付时间：确定后严格执行

---

## 一、商业计划沙盘内容框架

### 1.1 沙盘必须包含的模块

```
商业计划沙盘（第一稿）
├── 1. 执行摘要（1页）
├── 2. 公司/项目概述
├── 3. 市场分析（目标客户、市场规模、竞争格局）
├── 4. 产品与服务（核心产品、服务模式、定价）
├── 5. 商业模式（收入来源、成本结构、盈利模型）
├── 6. 营销策略（获客渠道、品牌定位、推广计划）
├── 7. 运营计划（团队、流程、里程碑）
├── 8. 财务预测（3-5年收入、成本、现金流）
├── 9. 融资需求（融资金额、用途、退出机制）
└── 10. 风险与对策
```

### 1.2 第一稿重点

**不是完美版，是可用版**：
- 核心数据准确（市场规模、定价、成本）
- 逻辑清晰完整（商业模式闭环）
- 关键假设明确（待验证的假设标注）
- 可后续迭代（根据反馈快速优化）

## 二、Token优化策略

### 2.1 减少Token消耗的核心原则

| 策略 | 说明 | 预期节省 |
|------|------|---------|
| **精准分工** | 每个角色只做最擅长的事，不重复 | 节省40% |
| **MVP优先** | 先交付核心模块，非核心简化 | 节省30% |
| **数据复用** | 已有数据直接引用，不重新生成 | 节省20% |
| **结构化输出** | 用表格、列表代替长文本 | 节省10% |

### 2.3 Token节省具体措施

**1. 数据复用（节省20K tokens）**
- FIN：直接引用定价审核报告、成本核算模型
- PROD：引用14个工具介绍、服务流程
- NARR：引用品牌定位、五图腾金句
- GAME：引用大富翁游戏设计
- LEGAL：引用法律风险评估

**2. 结构化输出（节省5K tokens）**
- 用表格展示财务数据
- 用列表展示里程碑
- 用图示代替长文本描述

**3. 并行协作（节省10K tokens）**
- 各角色同时工作，不等待
- 避免串行传递造成的重复

### 5.2 第一稿质量标准

**必须有**：
- [ ] 准确的核心数据（市场、定价、成本）
- [ ] 清晰的商业模式（收入来源、成本结构）
