## 二、晨会机制（Daily Standup）

### 2.1 基本信息
- **频率**: 每日（工作日）
- **时长**: 5-10分钟
- **形式**: 文字/语音（飞书）
- **参与者**: Egbertie + 满意妞

### 2.2 标准话术模板
```
📅 晨会 [日期]

昨日完成：
• [任务1] - 状态
• [任务2] - 状态

今日计划：
• [优先级P0任务]
• [优先级P1任务]

阻塞/风险：
• [需要用户决策的事项]
• [等待外部反馈的事项]

需要支持：
• [需要Egbertie协助的事项]
```

### 2.3 触发条件
- 自动触发: 每日09:00（如工作日开始）
- 手动触发: 用户发送"晨会"或"今天有什么"
- 静默规则: 如无重要事项可合并到周会

## 三、周会机制（Weekly Review）

### 3.1 基本信息
- **频率**: 每周五 18:00-19:00
- **时长**: 60分钟
- **形式**: 飞书语音/文档
- **参与者**: Egbertie + 满意妞 + 可选专家

### 3.2 标准议程

| 时间 | 议题 | 负责人 | 输出 |
|------|------|--------|------|
| 0-10min | 本周完成回顾 | 满意妞 | 完成清单 |
| 10-30min | 进行中任务状态 | 满意妞 | 进度更新 |
| 30-45min | 下周计划确认 | 共同 | 下周TODO |
| 45-55min | 风险与阻塞讨论 | 共同 | 解决方案 |
| 55-60min | 其他事项 | 共同 | 行动项 |

### 3.3 周会报告模板
```markdown
# 周会报告 [第X周] - 日期

## 本周完成 ✅
| 任务 | 负责人 | 状态 |
|------|--------|------|

## 进行中 🔄
| 任务 | 进度 | 预计完成 |
|------|------|----------|

## 下周计划 📋
| 任务 | 优先级 | 负责人 |
|------|--------|--------|

## 风险与阻塞 ⚠️
| 事项 | 影响 | 应对方案 |
|------|------|----------|

## 决策待确认 ❓
| 事项 | 建议方案 | 截止日期 |
|------|----------|----------|
```

## 四、复盘会机制（Monthly Retrospective）

### 4.1 基本信息
- **频率**: 每月最后一个周五
- **时长**: 90-120分钟
- **形式**: 深度对话 + 文档
- **参与者**: Egbertie + 满意妞 + PEO/EEO（可选）

### 4.2 复盘框架（4L模型）

```
┌────────────────────────────────────────┐
│           月度复盘 - 4L模型             │
├────────────────────────────────────────┤
│                                        │
│  🎯 Liked（喜欢的）                     │
│     本月做得好的，值得继续保持的         │
│                                        │
│  📚 Learned（学到的）                   │
│     本月的新认知、新发现                 │
│                                        │
│  😕 Lacked（缺失的）                    │
│     本月做得不够好的，需要改进的         │
│                                        │
│  🌱 Longed for（渴望的）                │
│     下月希望实现的新目标                 │
│                                        │
└────────────────────────────────────────┘
```

## 五、紧急会议触发机制

### 5.1 触发条件
| 场景 | 响应时间 | 会议形式 | 参与者 |
|------|----------|----------|--------|
| 系统故障/数据丢失 | < 15分钟 | 立即语音 | 全员 |
| 关键决策待确认 | < 2小时 | 快速对齐 | 相关方 |
| 重大风险发现 | < 4小时 | 专题讨论 | 相关方 |
| 用户明确要求 | 立即 | 按需 | 按需 |

### 5.2 紧急会议话术
```
🚨 紧急会议召集

主题: [简明描述]
时间: [立即/具体时间点]
形式: [语音/文字/文档]
预计时长: [X分钟]

议程:
1. [议题1]
2. [议题2]

请确认可参加时间。
```
